### 1.2 Accessibility là gì?

**Web Accessibility (A11y)** — con số 11 là số chữ cái giữa "a" và "y" trong "accessibility" — là practice thiết kế và xây dựng web sao cho **mọi người đều có thể sử dụng**, bao gồm:

- Người **mù** hoặc **khiếm thị** → dùng screen readers (NVDA, JAWS, VoiceOver)
- Người **điếc** hoặc **khiếm thính** → cần captions, transcripts
- Người **khiếm khuyết vận động** → dùng bàn phím, switch access, eye tracking
- Người **nhận thức** → cần UI rõ ràng, ngôn ngữ đơn giản, không distraction
- Người **cao tuổi** → font lớn hơn, contrast cao hơn
- Người **tạm thời bị hạn chế** → gãy tay, đeo kính mới, dùng điện thoại ngoài trời

**WCAG (Web Content Accessibility Guidelines)** là tiêu chuẩn quốc tế do W3C ban hành. Ba mức độ:

- **Level A** — Bắt buộc tối thiểu
- **Level AA** — Tiêu chuẩn cho hầu hết tổ chức (bạn nên đạt level này)
- **Level AAA** — Cao nhất, không bắt buộc hoàn toàn

**Bốn nguyên tắc POUR:**

```
P — Perceivable   (Có thể nhận biết được)
    → Alt text cho ảnh, captions cho video, đủ color contrast

O — Operable      (Có thể thao tác được)
    → Keyboard navigation, đủ thời gian, không gây seizures

U — Understandable (Có thể hiểu được)
    → Ngôn ngữ rõ ràng, form errors descriptive, consistent UI

R — Robust        (Vững chắc)
    → Hoạt động với nhiều assistive technologies khác nhau
```

### 1.3 Mental Model: Accessibility như "Clear Communication"

```
Bạn viết code → Browser render HTML → Accessibility Tree → Screen Reader → User

                                            ↑
                              Đây là nơi A11y sống
```

Browser tạo **Accessibility Tree** — một representation của page dành riêng cho assistive technologies. Screen readers không đọc visual CSS, chúng đọc accessibility tree.

```
Visual HTML:                    Accessibility Tree:
<div class="card">              Role: generic
  <img src="..." />             → Role: image, Name: "Ảnh sản phẩm áo thun"
  <span>Áo thun</span>         → Role: text, Name: "Áo thun"
  <div class="btn">             → Role: generic ← SCREEN READER BỊ MÙ!
    Mua ngay
  </div>
</div>

→ Semantic HTML:                Accessibility Tree (đúng):
<article>                       Role: article
  <img alt="Ảnh sản phẩm áo thun" />  → Role: image, Name: "Ảnh sản phẩm áo thun"
  <h3>Áo thun</h3>             → Role: heading level 3, Name: "Áo thun"
  <button>                      → Role: button, Name: "Mua ngay" ← SCREEN READER HIỂU!
    Mua ngay
  </button>
</article>
```

**Analogy:** Accessibility Tree giống như bản mô tả bằng lời của một bức tranh. Nếu bạn vẽ đẹp nhưng không describe rõ, người mù không biết bức tranh nói gì.

### 1.4 Hiểu Lầm Phổ Biến

**❌ Hiểu lầm 1: "Accessibility chỉ cho người khuyết tật"**

Keyboard navigation → ai cũng dùng Tab. Alt text → SEO. Color contrast → màn hình ngoài trời. A11y = good UX cho tất cả mọi người.

**❌ Hiểu lầm 2: "Thêm ARIA vào là xong"**

```jsx
// ❌ ARIA không thay thế được semantic HTML
// "No ARIA is better than bad ARIA" — W3C
<div role="button" aria-label="Submit">Submit</div>
// Thiếu: keyboard focus, Enter/Space key support, disabled state, etc.

// ✅ Dùng đúng element
<button type="submit">Submit</button>
// Tự động có: focusable, Enter/Space support, role="button", etc.
```

**❌ Hiểu lầm 3: "Screen reader users không phải target user của chúng tôi"**

Bạn không biết ai dùng sản phẩm của bạn. Và ngay cả khi không có user nào bị khuyết tật hôm nay, họ có thể bị tai nạn ngày mai.

**❌ Hiểu lầm 4: "Accessible web trông xấu hơn"**

Không đúng. BBC, Apple, Google — tất cả đều accessible và đều đẹp. A11y là về structure và interaction, không phải về aesthetics.

```tsx
          <article className='product-card'>
            <img

            />

            <p>
              <span className='visually-hidden'>Giá:</span>
              29,990,000 đ
            </p>
            {/* button, không phải div! */}
            <button
              type='button'
              onClick={handleAddToCart}
              aria-label='Thêm iPhone 15 vào giỏ hàng'
            >
              Thêm vào giỏ
```

**Bảng Semantic HTML Elements Quan Trọng:**

```
Element         | Dùng khi nào
----------------|--------------------------------------------------
<header>        | Header của page hoặc section
<nav>           | Navigation links
<main>          | Nội dung chính (chỉ 1 <main> per page)
<section>       | Nhóm nội dung có heading riêng
<article>       | Self-contained content (blog post, product card)
<aside>         | Nội dung phụ (sidebar, related links)
<footer>        | Footer của page hoặc section
<h1>-<h6>       | Headings theo hierarchy (h1 > h2 > h3...)
<button>        | Clickable actions
<a href>        | Navigation/links
<label>         | Label cho form inputs
<fieldset>      | Group form inputs liên quan
<legend>        | Title cho fieldset
<ul>/<ol>       | Lists (nav items, product lists, etc.)
<table>         | Tabular data (có header, không phải layout)
```

> **Quy tắc vàng:** Nếu nó trông như button → dùng `<button>`. Nếu nó navigate → dùng `<a>`. Nếu nó là form input → dùng input elements tương ứng.

### Demo 2: ARIA Attributes ⭐⭐

ARIA (Accessible Rich Internet Applications) cung cấp thêm semantic information khi HTML thuần không đủ:

```tsx
// ── aria-label ──────────────────────────────────────────────────
// Dùng khi element không có visible text label

// ❌ Không có accessible name
<button onClick={handleClose}>✕</button>
// Screen reader đọc: "Button" hoặc "✕" — không có nghĩa

// ✅ Có aria-label
<button onClick={handleClose} aria-label="Đóng modal">✕</button>
// Screen reader đọc: "Đóng modal, Button"

// Icon buttons phổ biến:
<button aria-label="Tìm kiếm"><SearchIcon /></button>
<button aria-label="Thêm vào yêu thích"><HeartIcon /></button>
<button aria-label="Mở menu"><HamburgerIcon /></button>

// ── aria-labelledby ─────────────────────────────────────────────
// Dùng khi label là element riêng biệt trên page

<h2 id="cart-title">Giỏ hàng của bạn</h2>
<section aria-labelledby="cart-title">
  {/* Section này được label bởi element có id="cart-title" */}
  ...
</section>

// ── aria-describedby ────────────────────────────────────────────
// Thêm description (không phải label) cho element

<label htmlFor="password">Mật khẩu</label>
<input
  id="password"
  type="password"
  aria-describedby="password-hint"
/>
<p id="password-hint">Ít nhất 8 ký tự, bao gồm chữ và số</p>
// Screen reader đọc: "Mật khẩu, Edit text. Ít nhất 8 ký tự..."

// ── aria-required ───────────────────────────────────────────────
<input
  type="email"
  required           // HTML attribute
  aria-required="true"  // Cho screen readers cũ
/>

// ── aria-invalid + aria-errormessage ────────────────────────────
// Thông báo validation errors cho screen readers

interface FormFieldProps {
  id: string;
  label: string;
  error?: string;
  value: string;
  onChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
}

function FormField({ id, label, error, value, onChange }: FormFieldProps) {
  const errorId = `${id}-error`;

  return (
    <div className="form-field">
      <label htmlFor={id}>{label}</label>
      <input
        id={id}
        value={value}
        onChange={onChange}
        aria-invalid={error ? 'true' : 'false'}
        aria-describedby={error ? errorId : undefined}
      />
      {error && (
        <p id={errorId} className="error-message" role="alert">
          {/* role="alert" → screen reader thông báo ngay lập tức */}
          {error}
        </p>
      )}
    </div>
  );
}

// ── aria-live ───────────────────────────────────────────────────
// Thông báo dynamic content changes (không cần focus vào element)

function NotificationArea() {
  const [message, setMessage] = React.useState('');

  return (
    <>
      <button onClick={() => setMessage('Đã thêm vào giỏ hàng!')}>
        Thêm vào giỏ
      </button>

      {/* aria-live="polite" → đọc sau khi user xong việc đang làm */}
      {/* aria-live="assertive" → đọc ngay lập tức (dùng cho errors khẩn cấp) */}
      <div aria-live="polite" aria-atomic="true" className="visually-hidden">
        {message}
      </div>
    </>
  );
}

// ── aria-expanded ───────────────────────────────────────────────
// Cho biết một expandable element đang open hay close

interface AccordionProps {
  title: string;
  children: React.ReactNode;
}

function Accordion({ title, children }: AccordionProps) {
  const [isOpen, setIsOpen] = React.useState(false);
  const contentId = React.useId(); // React 18 hook tạo unique ID

  return (
    <div>
      <button
        type="button"
        onClick={() => setIsOpen(!isOpen)}
        aria-expanded={isOpen}  // "true" hoặc "false"
        aria-controls={contentId}  // ID của element được control
      >
        {title}
        <span aria-hidden="true">{isOpen ? '▲' : '▼'}</span>
        {/* aria-hidden="true" → icon decorative, screen reader bỏ qua */}
      </button>

      <div
        id={contentId}
        hidden={!isOpen}  // hidden attribute → accessible hide
      >
        {children}
      </div>
    </div>
  );
}
```

### Demo 3: Keyboard Navigation & Focus Management ⭐⭐⭐

Đây là phần phức tạp nhất nhưng cũng quan trọng nhất trong React:

```tsx
// ── Focus Trap trong Modal ───────────────────────────────────────
// Khi modal mở, focus phải "bị kẹt" bên trong modal
// User không thể Tab ra ngoài modal

import React, { useEffect, useRef } from 'react';

interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  title: string;
  children: React.ReactNode;
}

function AccessibleModal({ isOpen, onClose, title, children }: ModalProps) {
  const modalRef = useRef<HTMLDivElement>(null);
  const titleId = React.useId();

  // ── Focus Management ──────────────────────────────────────────
  useEffect(() => {
    if (!isOpen) return;

    // Move focus vào modal khi mở
    // (Không dùng useEffect ngày 58 trong exercises — nhưng demo này
    // là code đã có sẵn để học, không phải exercise)
    const modal = modalRef.current;
    if (!modal) return;

    // Tìm element có thể focus đầu tiên trong modal
    const focusableElements = modal.querySelectorAll<HTMLElement>(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])',
    );
    const firstElement = focusableElements[0];
    const lastElement = focusableElements[focusableElements.length - 1];

    // Focus vào element đầu tiên
    firstElement?.focus();

    // ── Focus Trap: bắt Tab key ────────────────────────────────
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === 'Escape') {
        onClose(); // Đóng modal khi nhấn Escape
        return;
      }

      if (e.key !== 'Tab') return;

      if (e.shiftKey) {
        // Shift + Tab: đi ngược
        if (document.activeElement === firstElement) {
          e.preventDefault();
          lastElement?.focus(); // Wrap về cuối
        }
      } else {
        // Tab: đi xuôi
        if (document.activeElement === lastElement) {
          e.preventDefault();
          firstElement?.focus(); // Wrap về đầu
        }
      }
    };

    modal.addEventListener('keydown', handleKeyDown);
    return () => modal.removeEventListener('keydown', handleKeyDown);
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    // Overlay: click để đóng
    <div
      className='modal-overlay'
      onClick={onClose}
      // aria-hidden overlay, không phải modal content
    >
      {/* Modal content: ngăn click bubble lên overlay */}
      <div
        ref={modalRef}
        className='modal-content'
        role='dialog' // Semantic role
        aria-modal='true' // Nói với screen reader đây là modal
        aria-labelledby={titleId} // Title của modal
        onClick={(e) => e.stopPropagation()}
      >
        <div className='modal-header'>
          <h2 id={titleId}>{title}</h2>
          <button
            type='button'
            onClick={onClose}
            aria-label='Đóng hộp thoại'
          >
            ✕
          </button>
        </div>
        <div className='modal-body'>{children}</div>
      </div>
    </div>
  );
}

// ── Restore Focus khi đóng Modal ────────────────────────────────
// Sau khi đóng modal, focus phải return về element đã trigger modal

function PageWithModal() {
  const [isOpen, setIsOpen] = React.useState(false);
  const triggerButtonRef = useRef<HTMLButtonElement>(null);

  const handleOpen = () => setIsOpen(true);

  const handleClose = () => {
    setIsOpen(false);
    // Restore focus về button đã mở modal
    triggerButtonRef.current?.focus();
  };

  return (
    <>
      <button
        ref={triggerButtonRef}
        type='button'
        onClick={handleOpen}
      >
        Mở hộp thoại
      </button>

      <AccessibleModal
        isOpen={isOpen}
        onClose={handleClose}
        title='Xác nhận đặt hàng'
      >
        <p>Bạn có chắc muốn đặt hàng không?</p>
        <button onClick={handleClose}>Hủy</button>
        <button
          onClick={() => {
            /* ... */ handleClose();
          }}
        >
          Xác nhận
        </button>
      </AccessibleModal>
    </>
  );
}
```

### Demo 4: Color Contrast & Visual A11y ⭐⭐

```tsx
// ── Color Contrast Requirements (WCAG AA) ───────────────────────
// Normal text: contrast ratio ≥ 4.5:1
// Large text (18px+ hoặc 14px+ bold): contrast ratio ≥ 3:1
// UI components và graphics: contrast ratio ≥ 3:1

// ❌ Cách SAI: Chỉ dùng màu để convey information
function OrderStatus({ status }: { status: 'success' | 'error' | 'pending' }) {
  const colors = {
    success: 'green',

function useReducedMotion(): boolean {
  const [reducedMotion, setReducedMotion] = React.useState(false);

  React.useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setReducedMotion(mediaQuery.matches);

    const handler = (e: MediaQueryListEvent) => setReducedMotion(e.matches);
    mediaQuery.addEventListener('change', handler);
    return () => mediaQuery.removeEventListener('change', handler);
  }, []);

  return reducedMotion;
}
```

### Bảng So Sánh — "Ẩn" Element Với Screen Readers

| Method                  | Visual | Screen Reader | Dùng khi nào                                  |
| ----------------------- | ------ | ------------- | --------------------------------------------- |
| `display: none`         | Ẩn     | Ẩn            | Ẩn hoàn toàn (modal đóng, tab inactive)       |
| `visibility: hidden`    | Ẩn     | Ẩn            | Ẩn nhưng giữ layout space                     |
| `opacity: 0`            | Ẩn     | **Đọc được**  | Animation (phải kết hợp pointer-events: none) |
| `aria-hidden="true"`    | Hiện   | Ẩn            | Icon decorative, duplicate text               |
| `.visually-hidden` CSS  | Ẩn     | **Đọc được**  | Screen reader only text                       |
| `hidden` HTML attribute | Ẩn     | Ẩn            | Native hide, tương đương `display:none`       |

### Bảng So Sánh — Khi Nào Dùng ARIA Role Nào

| Role       | Dùng khi                                       | Native alternative                    |
| ---------- | ---------------------------------------------- | ------------------------------------- |
| `button`   | Element clickable không phải button            | `<button>` (luôn prefer)              |
| `link`     | Element navigate không phải anchor             | `<a href>` (luôn prefer)              |
| `dialog`   | Modal/popup overlay                            | HTML `<dialog>` (browser support tốt) |
| `alert`    | Error/important message cần đọc ngay           | Không có native                       |
| `status`   | Success/info message không khẩn cấp            | Không có native                       |
| `menu`     | Dropdown menu (actions, không phải navigation) | Không có native                       |
| `listbox`  | Select-like dropdown                           | `<select>`                            |
| `tab`      | Tab trong tablist                              | Không có native                       |
| `tabpanel` | Content của tab                                | Không có native                       |
| `group`    | Group inputs liên quan                         | `<fieldset>`                          |
| `search`   | Search form hoặc region                        | `<search>` (HTML5.3)                  |

> **Quy tắc vàng:** **"First rule of ARIA: Don't use ARIA"** — W3C. Luôn ưu tiên HTML semantic elements. Chỉ dùng ARIA khi HTML không có native solution.

### Bug 1: Focus Không Return Sau Khi Đóng Modal

```tsx
// ❌ Code có bug:
function BrokenModal() {
  const [isOpen, setIsOpen] = React.useState(false);

  return (
    <>
      <button onClick={() => setIsOpen(true)}>Mở modal</button>

      {isOpen && (
        <div
          role='dialog'
          aria-modal='true'
          aria-label='Settings'
        >
          <h2>Cài đặt</h2>
          <button onClick={() => setIsOpen(false)}>Đóng</button>
          <input
            type='text'
            placeholder='Username'
          />
        </div>
      )}
    </>
  );
}

// ❓ Câu hỏi:
// 1. Bug gì xảy ra với keyboard user khi đóng modal?
// 2. Bug gì xảy ra khi modal mở, focus ở đâu?
// 3. Sửa như thế nào?
```

**Giải thích và fix:**

```tsx
// 🔍 VẤN ĐỀ 1: Khi modal đóng, focus "rơi" về document.body
// User mất orientation — họ không biết focus đang ở đâu

// 🔍 VẤN ĐỀ 2: Khi modal mở, focus vẫn ở button trigger
// User phải Tab nhiều lần để vào modal

// ✅ Fix:
function FixedModal() {
  const [isOpen, setIsOpen] = React.useState(false);
  const triggerRef = React.useRef<HTMLButtonElement>(null);
  const firstFocusRef = React.useRef<HTMLButtonElement>(null);

  const handleOpen = () => {
    setIsOpen(true);
    // Focus sẽ được set sau khi modal render
  };

  const handleClose = () => {
    setIsOpen(false);
    // Return focus về trigger
    triggerRef.current?.focus();
  };

  React.useEffect(() => {
    if (isOpen) {
      // Focus vào element đầu tiên sau khi modal render
      firstFocusRef.current?.focus();
    }
  }, [isOpen]);

  return (
    <>
      <button
        ref={triggerRef}
        onClick={handleOpen}
      >
        Mở modal
      </button>

      {isOpen && (
        <div
          role='dialog'
          aria-modal='true'
          aria-label='Settings'
        >
          <h2>Cài đặt</h2>
          {/* ref để focus vào sau khi open */}
          <button
            ref={firstFocusRef}
            onClick={handleClose}
          >
            Đóng
          </button>
          <input
            type='text'
            placeholder='Username'
          />
        </div>
      )}
    </>
  );
}
```

```tsx
// ❌ Code có bug:
function BrokenForm() {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');

        </p>

      <button type='submit'>Gửi</button>
    </form>
  );
}

// ❓ Câu hỏi:
// 1. Tại sao screen reader không thông báo error?
// 2. Có ít nhất 3 vấn đề A11y trong code này. Tìm hết.
// 3. Fix tất cả.
```

**Giải thích và fix:**

```tsx
// 🔍 VẤN ĐỀ 1: Error element xuất hiện động nhưng không có aria-live
// Screen reader chỉ đọc những gì đang trong DOM khi user focus vào
// Dynamic content cần aria-live để thông báo changes

// 🔍 VẤN ĐỀ 2: Chỉ dùng màu đỏ để indicate error
// Người mù màu không phân biệt được

// 🔍 VẤN ĐỀ 3: Input không có aria-invalid khi có error
// Screen reader không biết input đang có error state

// 🔍 VẤN ĐỀ 4: Error message không được kết nối với input
// Không có aria-describedby

// ✅ Fix:
function FixedForm() {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');
  const errorId = React.useId();

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    if (!email.includes('@')) {
      setError(
        'Email không hợp lệ. Vui lòng nhập đúng định dạng: ten@domain.com',
      );
    } else {
      setError('');
    }
  };

  return (
    <form
      onSubmit={handleSubmit}
      noValidate
    >
      <label htmlFor='email'>
        Email
        <span aria-hidden='true'> *</span>
        <span className='visually-hidden'> (bắt buộc)</span>
      </label>
      <input
        id='email'
        type='email'
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          if (error) setError(''); // Clear error khi user sửa
        }}
        aria-required='true'
        aria-invalid={error ? 'true' : 'false'}
        aria-describedby={error ? errorId : undefined}
      />
      {/* role="alert" → screen reader đọc ngay khi content thay đổi */}
      {/* aria-live="assertive" cũng được nhưng role="alert" là conventional */}
      <div
        id={errorId}
        role='alert'
        className='error-message'
      >
        {error && (
          <>
            {/* Icon + text, không chỉ màu */}
            <span aria-hidden='true'>⚠ </span>
            {error}
          </>
        )}
      </div>
      <button type='submit'>Gửi</button>
    </form>
  );
}
```

### Câu Hỏi Phỏng Vấn

**Junior:**

> "alt text trong thẻ img dùng để làm gì? Khi nào nên để alt rỗng (`alt=""`)?"

_Expected answer:_ alt text là mô tả thay thế cho image, được screen readers đọc và hiển thị khi ảnh không load. alt="" (rỗng) cho images có tính decorative — ảnh chỉ để đẹp, không mang thông tin — để screen readers bỏ qua.

**Mid-level:**

> "Giải thích sự khác biệt giữa aria-label, aria-labelledby, và aria-describedby. Cho ví dụ khi dùng từng cái."

_Expected answer:_ aria-label cung cấp label string trực tiếp (dùng khi không có visible label). aria-labelledby reference ID của element khác đang là label (dùng khi label đã visible trên page). aria-describedby cung cấp additional description, không phải label (dùng cho hints, error messages).

**Senior:**

> "Bạn nhận được yêu cầu thêm một combobox (searchable select) custom vào design system. Hãy mô tả approach của bạn để đảm bảo component này accessible."

_Expected answer:_ Refer đến WAI-ARIA Combobox pattern. Cần: input có role="combobox", aria-expanded, aria-autocomplete, aria-controls trỏ đến listbox; listbox có role="listbox"; options có role="option", aria-selected; keyboard: Arrow keys navigate list, Enter select, Escape close; focus management; aria-activedescendant để indicate focused option. Testing với VoiceOver và NVDA.
