## 🤔 Kiểm tra đầu vào (5 phút)

1. Bạn có thể điều hướng toàn bộ website chỉ bằng bàn phím không? Thử ngay trên một trang web bạn hay dùng — nhấn Tab liên tục và xem điều gì xảy ra.
   - _Đáp án: Có thể nếu website được xây dựng đúng chuẩn (semantic HTML, focusable elements). Nếu dùng `<div>` thay `<button>`, focus sẽ bị skip. Thử trên Google hoặc GitHub — Tab hoạt động mượt; thử trên site cũ/kém chuẩn — focus mất hướng hoặc bị mắc kẹt._

2. `<button>` và `<div onClick={...}>` trông giống nhau khi render, nhưng chúng khác nhau thế nào với người dùng bàn phím và screen reader?
   - _Đáp án: `<button>` tự động focusable (Tab), hỗ trợ Enter/Space, có role="button" trong accessibility tree. `<div onClick>` không focusable, không phản hồi keyboard, screen reader đọc "generic" thay vì "button" — người dùng bàn phím và screen reader hoàn toàn không tương tác được._

3. `aria-label` và `aria-labelledby` đều dùng để "đặt tên" cho element — bạn đoán sự khác biệt là gì?
   - _Đáp án: `aria-label` đặt tên trực tiếp bằng string (`aria-label="Đóng"`). `aria-labelledby` reference ID của element khác trên page (`aria-labelledby="title-id"`) — dùng khi label đã visible, tránh duplicate text._

### 1.2 Accessibility là gì?

**Web Accessibility (A11y)** — con số 11 là số chữ cái giữa "a" và "y" trong "accessibility" — là practice thiết kế và xây dựng web sao cho **mọi người đều có thể sử dụng**, bao gồm:

- Người **mù** hoặc **khiếm thị** → dùng screen readers (NVDA, JAWS, VoiceOver)
- Người **điếc** hoặc **khiếm thính** → cần captions, transcripts
- Người **khiếm khuyết vận động** → dùng bàn phím, switch access, eye tracking
- Người **nhận thức** → cần UI rõ ràng, ngôn ngữ đơn giản, không distraction
- Người **cao tuổi** → font lớn hơn, contrast cao hơn
- Người **tạm thời bị hạn chế** → gãy tay, đeo kính mới, dùng điện thoại ngoài trời

**WCAG (Web Content Accessibility Guidelines)** là tiêu chuẩn quốc tế do W3C ban hành. Ba mức độ:

- **Level A** — Bắt buộc tối thiểu
- **Level AA** — Tiêu chuẩn cho hầu hết tổ chức (bạn nên đạt level này)
- **Level AAA** — Cao nhất, không bắt buộc hoàn toàn

**Bốn nguyên tắc POUR:**

```
P — Perceivable   (Có thể nhận biết được)
    → Alt text cho ảnh, captions cho video, đủ color contrast

O — Operable      (Có thể thao tác được)
    → Keyboard navigation, đủ thời gian, không gây seizures

U — Understandable (Có thể hiểu được)
    → Ngôn ngữ rõ ràng, form errors descriptive, consistent UI

R — Robust        (Vững chắc)
    → Hoạt động với nhiều assistive technologies khác nhau
```

### 1.3 Mental Model: Accessibility như "Clear Communication"

```
Bạn viết code → Browser render HTML → Accessibility Tree → Screen Reader → User

                                            ↑
                              Đây là nơi A11y sống
```

Browser tạo **Accessibility Tree** — một representation của page dành riêng cho assistive technologies. Screen readers không đọc visual CSS, chúng đọc accessibility tree.

```
Visual HTML:                    Accessibility Tree:
<div class="card">              Role: generic
  <img src="..." />             → Role: image, Name: "Ảnh sản phẩm áo thun"
  <span>Áo thun</span>         → Role: text, Name: "Áo thun"
  <div class="btn">             → Role: generic ← SCREEN READER BỊ MÙ!
    Mua ngay
  </div>
</div>

→ Semantic HTML:                Accessibility Tree (đúng):
<article>                       Role: article
  <img alt="Ảnh sản phẩm áo thun" />  → Role: image, Name: "Ảnh sản phẩm áo thun"
  <h3>Áo thun</h3>             → Role: heading level 3, Name: "Áo thun"
  <button>                      → Role: button, Name: "Mua ngay" ← SCREEN READER HIỂU!
    Mua ngay
  </button>
</article>
```

**Analogy:** Accessibility Tree giống như bản mô tả bằng lời của một bức tranh. Nếu bạn vẽ đẹp nhưng không describe rõ, người mù không biết bức tranh nói gì.

### 1.4 Hiểu Lầm Phổ Biến

**❌ Hiểu lầm 1: "Accessibility chỉ cho người khuyết tật"**

Keyboard navigation → ai cũng dùng Tab. Alt text → SEO. Color contrast → màn hình ngoài trời. A11y = good UX cho tất cả mọi người.

**❌ Hiểu lầm 2: "Thêm ARIA vào là xong"**

```jsx
// ❌ ARIA không thay thế được semantic HTML
// "No ARIA is better than bad ARIA" — W3C
<div role="button" aria-label="Submit">Submit</div>
// Thiếu: keyboard focus, Enter/Space key support, disabled state, etc.

// ✅ Dùng đúng element
<button type="submit">Submit</button>
// Tự động có: focusable, Enter/Space support, role="button", etc.
```

**❌ Hiểu lầm 3: "Screen reader users không phải target user của chúng tôi"**

Bạn không biết ai dùng sản phẩm của bạn. Và ngay cả khi không có user nào bị khuyết tật hôm nay, họ có thể bị tai nạn ngày mai.

**❌ Hiểu lầm 4: "Accessible web trông xấu hơn"**

Không đúng. BBC, Apple, Google — tất cả đều accessible và đều đẹp. A11y là về structure và interaction, không phải về aesthetics.

```tsx
          <article className='product-card'>
            <img

            />
            <h3>iPhone 15</h3>
            <p>
              <span className='visually-hidden'>Giá:</span>
              29,990,000 đ
            </p>
            {/* button, không phải div! */}
            <button
              type='button'
              onClick={handleAddToCart}
              aria-label='Thêm iPhone 15 vào giỏ hàng'
            >
              Thêm vào giỏ
            </button>
          </article>
        </section>

        <p>© 2024 ShopVN</p>

```

**Bảng Semantic HTML Elements Quan Trọng:**

```
Element         | Dùng khi nào
----------------|--------------------------------------------------
<header>        | Header của page hoặc section
<nav>           | Navigation links
<main>          | Nội dung chính (chỉ 1 <main> per page)
<section>       | Nhóm nội dung có heading riêng
<article>       | Self-contained content (blog post, product card)
<aside>         | Nội dung phụ (sidebar, related links)
<footer>        | Footer của page hoặc section
<h1>-<h6>       | Headings theo hierarchy (h1 > h2 > h3...)
<button>        | Clickable actions
<a href>        | Navigation/links
<label>         | Label cho form inputs
<fieldset>      | Group form inputs liên quan
<legend>        | Title cho fieldset
<ul>/<ol>       | Lists (nav items, product lists, etc.)
<table>         | Tabular data (có header, không phải layout)
```

> **Quy tắc vàng:** Nếu nó trông như button → dùng `<button>`. Nếu nó navigate → dùng `<a>`. Nếu nó là form input → dùng input elements tương ứng.

### Demo 2: ARIA Attributes ⭐⭐

ARIA (Accessible Rich Internet Applications) cung cấp thêm semantic information khi HTML thuần không đủ:

```tsx
// ── aria-label ──────────────────────────────────────────────────
// Dùng khi element không có visible text label

// ❌ Không có accessible name
<button onClick={handleClose}>✕</button>
// Screen reader đọc: "Button" hoặc "✕" — không có nghĩa

// ✅ Có aria-label
<button onClick={handleClose} aria-label="Đóng modal">✕</button>
// Screen reader đọc: "Đóng modal, Button"

// Icon buttons phổ biến:
<button aria-label="Tìm kiếm"><SearchIcon /></button>
<button aria-label="Thêm vào yêu thích"><HeartIcon /></button>
<button aria-label="Mở menu"><HamburgerIcon /></button>

// ── aria-labelledby ─────────────────────────────────────────────
// Dùng khi label là element riêng biệt trên page

<h2 id="cart-title">Giỏ hàng của bạn</h2>
<section aria-labelledby="cart-title">
  {/* Section này được label bởi element có id="cart-title" */}
  ...
</section>

// ── aria-describedby ────────────────────────────────────────────
// Thêm description (không phải label) cho element

<label htmlFor="password">Mật khẩu</label>
<input
  id="password"
  type="password"
  aria-describedby="password-hint"
/>
<p id="password-hint">Ít nhất 8 ký tự, bao gồm chữ và số</p>
// Screen reader đọc: "Mật khẩu, Edit text. Ít nhất 8 ký tự..."

// ── aria-required ───────────────────────────────────────────────
<input
  type="email"
  required           // HTML attribute
  aria-required="true"  // Cho screen readers cũ
/>

// ── aria-invalid + aria-errormessage ────────────────────────────
// Thông báo validation errors cho screen readers

interface FormFieldProps {
  id: string;
  label: string;
  error?: string;
  value: string;
  onChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
}

function FormField({ id, label, error, value, onChange }: FormFieldProps) {
  const errorId = `${id}-error`;

  return (
    <div className="form-field">
      <label htmlFor={id}>{label}</label>
      <input
        id={id}
        value={value}
        onChange={onChange}
        aria-invalid={error ? 'true' : 'false'}
        aria-describedby={error ? errorId : undefined}
      />
      {error && (
        <p id={errorId} className="error-message" role="alert">
          {/* role="alert" → screen reader thông báo ngay lập tức */}
          {error}
        </p>
      )}
    </div>
  );
}

// ── aria-live ───────────────────────────────────────────────────
// Thông báo dynamic content changes (không cần focus vào element)

function NotificationArea() {
  const [message, setMessage] = React.useState('');

  return (
    <>
      <button onClick={() => setMessage('Đã thêm vào giỏ hàng!')}>
        Thêm vào giỏ
      </button>

      {/* aria-live="polite" → đọc sau khi user xong việc đang làm */}
      {/* aria-atomic="true" → khi nội dung thay đổi, screen reader sẽ đọc lại toàn bộ vùng thay vì chỉ phần thay đổi */}
      {/* aria-live="assertive" → đọc ngay lập tức (dùng cho errors khẩn cấp) */}
      <div aria-live="polite" aria-atomic="true" className="visually-hidden">
        {message}
      </div>
    </>
  );
}

// ── aria-expanded ───────────────────────────────────────────────
// Cho biết một expandable element đang open hay close

interface AccordionProps {
  title: string;
  children: React.ReactNode;
}

function Accordion({ title, children }: AccordionProps) {
  const [isOpen, setIsOpen] = React.useState(false);
  const contentId = React.useId(); // React 18 hook tạo unique ID

  return (
    <div>
      <button
        type="button"
        onClick={() => setIsOpen(!isOpen)}
        aria-expanded={isOpen}  // "true" hoặc "false"
        aria-controls={contentId}  // ID của element được control
      >
        {title}
        <span aria-hidden="true">{isOpen ? '▲' : '▼'}</span>
        {/* aria-hidden="true" → icon decorative, screen reader bỏ qua */}
      </button>

      <div
        id={contentId}
        hidden={!isOpen}  // hidden attribute → accessible hide
      >
        {children}
      </div>
    </div>
  );
}
```

### Demo 3: Keyboard Navigation & Focus Management ⭐⭐⭐

Đây là phần phức tạp nhất nhưng cũng quan trọng nhất trong React:

```tsx
// ── Focus Trap trong Modal ───────────────────────────────────────
// Khi modal mở, focus phải "bị kẹt" bên trong modal
// User không thể Tab ra ngoài modal

import React, { useEffect, useRef } from 'react';

interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  title: string;
  children: React.ReactNode;
}

function AccessibleModal({ isOpen, onClose, title, children }: ModalProps) {
  const modalRef = useRef<HTMLDivElement>(null);
  const titleId = React.useId();

  // ── Focus Management ──────────────────────────────────────────
  useEffect(() => {
    if (!isOpen) return;

    // Move focus vào modal khi mở
    const modal = modalRef.current;
    if (!modal) return;

    // Tìm element có thể focus đầu tiên trong modal
    const focusableElements = modal.querySelectorAll<HTMLElement>(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])',
    );
    const firstElement = focusableElements[0];
    const lastElement = focusableElements[focusableElements.length - 1];

    // Focus vào element đầu tiên
    firstElement?.focus();

    // ── Focus Trap: bắt Tab key ────────────────────────────────
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === 'Escape') {
        onClose(); // Đóng modal khi nhấn Escape
        return;
      }

      if (e.key !== 'Tab') return;

      if (e.shiftKey) {
        // Shift + Tab: đi ngược
        if (document.activeElement === firstElement) {
          e.preventDefault();
          lastElement?.focus(); // Wrap về cuối
        }
      } else {
        // Tab: đi xuôi
        if (document.activeElement === lastElement) {
          e.preventDefault();
          firstElement?.focus(); // Wrap về đầu
        }
      }
    };

    modal.addEventListener('keydown', handleKeyDown);
    return () => modal.removeEventListener('keydown', handleKeyDown);
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    // Overlay: click để đóng
    <div
      className='modal-overlay'
      onClick={onClose}
      // aria-hidden overlay, không phải modal content
    >
      {/* Modal content: ngăn click bubble lên overlay */}
      <div
        ref={modalRef}
        className='modal-content'
        role='dialog' // Semantic role
        aria-modal='true' // Nói với screen reader đây là modal
        aria-labelledby={titleId} // Title của modal
        onClick={(e) => e.stopPropagation()}
      >
        <div className='modal-header'>
          <h2 id={titleId}>{title}</h2>
          <button
            type='button'
            onClick={onClose}
            aria-label='Đóng hộp thoại'
          >
            ✕
          </button>
        </div>
        <div className='modal-body'>{children}</div>
      </div>
    </div>
  );
}

// ── Restore Focus khi đóng Modal ────────────────────────────────
// Sau khi đóng modal, focus phải return về element đã trigger modal

function PageWithModal() {
  const [isOpen, setIsOpen] = React.useState(false);
  const triggerButtonRef = useRef<HTMLButtonElement>(null);

  const handleOpen = () => setIsOpen(true);

  const handleClose = () => {
    setIsOpen(false);
    // Restore focus về button đã mở modal
    triggerButtonRef.current?.focus();
  };

  return (
    <>
      <button
        ref={triggerButtonRef}
        type='button'
        onClick={handleOpen}
      >
        Mở hộp thoại
      </button>

      <AccessibleModal
        isOpen={isOpen}
        onClose={handleClose}
        title='Xác nhận đặt hàng'
      >
        <p>Bạn có chắc muốn đặt hàng không?</p>
        <button onClick={handleClose}>Hủy</button>
        <button
          onClick={() => {
            /* ... */ handleClose();
          }}
        >
          Xác nhận
        </button>
      </AccessibleModal>
    </>
  );
}
```

```tsx
function useReducedMotion(): boolean {
  const [reducedMotion, setReducedMotion] = React.useState(false);

  React.useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setReducedMotion(mediaQuery.matches);

    const handler = (e: MediaQueryListEvent) => setReducedMotion(e.matches);
    mediaQuery.addEventListener('change', handler);
    return () => mediaQuery.removeEventListener('change', handler);
  }, []);

  return reducedMotion;
}
```

```tsx
interface SearchInputProps {
  value: string;
  onChange: (value: string) => void;
  onSearch: (value: string) => void;
  placeholder?: string;
}

function SearchInput({
  value,
  onChange,
  onSearch,
  placeholder = 'Nhập từ khóa...',
}: SearchInputProps) {
  const inputId = React.useId();
  const hintId = React.useId();

  const handleSubmit = (event: React.FormEvent<HTMLFormElement>) => {
    event.preventDefault();
    onSearch(value);
  };

  return (
    /**
     * Quyết định A11y:
     * - Dùng <form role="search"> để đây là vùng tìm kiếm có ý nghĩa.
     * - Input có label thật bằng <label htmlFor>, nhưng label được visually-hidden.
     * - Dùng aria-describedby để screen reader đọc thêm hướng dẫn.
     * - Button "✕" có aria-label vì icon không đủ nghĩa.
     * - Button "🔍" có aria-label vì icon không đủ nghĩa.
     */
    <form
      className='search-wrapper'
      role='search'
      onSubmit={handleSubmit}
    >
      <label
        htmlFor={inputId}
        className='visually-hidden'
      >
        Tìm kiếm bài viết
      </label>

      <input
        id={inputId}
        className='search-input'
        type='search'
        value={value}
        onChange={(event) => onChange(event.target.value)}
        placeholder={placeholder}
        aria-describedby={hintId}
      />

      <p
        id={hintId}
        className='visually-hidden'
      >
        Nhập từ khóa rồi nhấn Enter hoặc nút tìm kiếm để tìm bài viết.
        {value ? ' Có thể dùng nút xóa để xóa nội dung tìm kiếm hiện tại.' : ''}
      </p>

      {value.length > 0 && (
        <button
          className='icon-button'
          type='button'
          onClick={() => onChange('')}
          aria-label='Xóa nội dung tìm kiếm'
        >
          <span aria-hidden='true'>✕</span>
        </button>
      )}

      <button
        className='icon-button'
        type='submit'
        aria-label='Tìm kiếm'
      >
        <span aria-hidden='true'>🔍</span>
      </button>
    </form>
  );
}

interface StarRatingProps {
  value: number;
  onChange: (rating: number) => void;
  label: string;
}

function StarRating({ value, onChange, label }: StarRatingProps) {
  const groupId = React.useId();
  const statusId = React.useId();

  return (
    /**
     * Quyết định A11y:
     * - Dùng role="group" để gom 5 button rating thành một nhóm logic.
     * - aria-labelledby trỏ đến text label đang hiển thị.
     * - Mỗi button có aria-label riêng: "Chọn 3 trên 5 sao".
     * - aria-pressed cho biết button rating hiện tại có đang được chọn không.
     * - aria-describedby giúp mỗi button biết trạng thái rating hiện tại.
     * - Vùng status có aria-live="polite" để thông báo khi rating thay đổi.
     */
    <div
      className='rating-group'
      role='group'
      aria-labelledby={groupId}
    >
      <span
        id={groupId}
        className='rating-label'
      >
        {label}
      </span>

      <div className='rating-buttons'>
        {[1, 2, 3, 4, 5].map((star) => {
          const isSelected = star === value;
          const isFilled = star <= value;

          return (
            <button
              key={star}
              className={`rating-button ${isSelected ? 'is-selected' : ''}`}
              type='button'
              onClick={() => onChange(star)}
              aria-label={`Chọn ${star} trên 5 sao`}
              aria-pressed={isSelected}
              aria-describedby={statusId}
              title={`${star} trên 5 sao`}
            >
              <span aria-hidden='true'>{isFilled ? '★' : '☆'}</span>
            </button>
          );
        })}
      </div>

      <p
        id={statusId}
        className='rating-value'
        aria-live='polite'
        aria-atomic='true'
      >
        {value > 0
          ? `Đánh giá hiện tại: ${value} trên 5 sao.`
          : 'Chưa có đánh giá.'}
      </p>
    </div>
  );
}

type ToastType = 'success' | 'error' | 'info' | 'warning';

interface ToastProps {
  message: string;
  type: ToastType;
  onDismiss: () => void;
}

function Toast({ message, type, onDismiss }: ToastProps) {
  const icons: Record<ToastType, string> = {
    success: '✓',
    error: '✕',
    info: 'ℹ',
    warning: '⚠',
  };

  const labels: Record<ToastType, string> = {
    success: 'Thành công',
    error: 'Lỗi',
    info: 'Thông báo',
    warning: 'Cảnh báo',
  };

  const isError = type === 'error';

  return (
    /**
     * Quyết định A11y:
     * - Toast xuất hiện động, user có thể đang focus ở nơi khác.
     * - Với error: dùng role="alert" + aria-live="assertive" vì lỗi cần được đọc ngay.
     * - Với success/info/warning: dùng role="status" + aria-live="polite" để không cắt ngang user.
     * - Icon chỉ là trang trí nên aria-hidden="true".
     * - Button đóng có aria-label rõ nghĩa.
     */
    <div
      className={`toast toast-${type}`}
      role={isError ? 'alert' : 'status'}
      aria-live={isError ? 'assertive' : 'polite'}
      aria-atomic='true'
    >
      <span aria-hidden='true'>{icons[type]}</span>

      <p>
        <span className='visually-hidden'>{labels[type]}: </span>
        {message}
      </p>

      <button
        className='toast-dismiss'
        type='button'
        onClick={onDismiss}
        aria-label={`Đóng thông báo: ${message}`}
      >
        <span aria-hidden='true'>✕</span>
      </button>
    </div>
  );
}
```

```tsx
/**
 * 🎯 Mục tiêu: Xây dựng Dropdown Menu accessible hoàn chỉnh
 * ⏱️ Thời gian: 40 phút
 *
 * 📋 Product Requirements:
 * User Story: "Là keyboard user, tôi muốn điều hướng dropdown menu
 *              bằng bàn phím để tôi không cần dùng chuột."
 *
 * ✅ Acceptance Criteria:
 * - [ ] Enter hoặc Space mở dropdown
 * - [ ] Escape đóng dropdown và trả focus về trigger button
 * - [ ] Arrow Down/Up điều hướng giữa các items
 * - [ ] Enter chọn item đang focus
 * - [ ] Tab đóng dropdown (không trap focus trong dropdown)
 * - [ ] aria-expanded trên trigger button
 * - [ ] aria-haspopup="listbox" hoặc "menu" trên trigger
 * - [ ] role="menu" trên list
 * - [ ] role="menuitem" trên từng item
 * - [ ] aria-activedescendant hoặc focus management đúng
 * - [ ] Click outside đóng dropdown
 * 
 * Ý nghĩa ngắn gọn của từng ARIA trong dropdown:
 *  `aria-expanded`
 *  - Cho biết dropdown đang mở (`true`) hay đóng (`false`).
 *  - Đặt trên nút trigger.
 *
 *  `aria-haspopup="menu"`
 *  - Cho biết khi tương tác với trigger sẽ xuất hiện một popup menu
 *  - Giúp screen reader hiểu đây không phải button thông thường.
 *
 * `role="menu"`
 *
 *  - Đánh dấu phần chứa danh sách là một menu.
 *  - Screen reader sẽ xử lý các item bên trong theo ngữ nghĩa menu.
 *
 * `role="menuitem"`
 *
 *  - Đánh dấu từng phần tử là một mục trong menu.
```

```tsx
/**
 * Tìm index của item enabled đầu tiên.
 * Nếu tất cả item đều disabled hoặc danh sách rỗng, trả về -1.
 */
function getFirstEnabledIndex(items: MenuItem[]) {
  return items.findIndex((item) => !item.disabled);
}

/**
 * Tìm index của item enabled cuối cùng.
 * Nếu tất cả item đều disabled hoặc danh sách rỗng, trả về -1.
 */
function getLastEnabledIndex(items: MenuItem[]) {
  for (let index = items.length - 1; index >= 0; index -= 1) {
    if (!items[index].disabled) {
      return index;
    }
  }

  return -1;
}

/**
 * Tìm item enabled tiếp theo.
 * Có wrap-around:
 * - Nếu đang ở item cuối, ArrowDown quay về item đầu.
 * - Nếu đang ở item đầu, ArrowUp quay về item cuối.
 */
function getNextEnabledIndex(
  items: MenuItem[],
  currentIndex: number,
  direction: 'next' | 'previous',
) {
  if (items.length === 0) {
    return -1;
  }

  const enabledItems = items.filter((item) => !item.disabled);

  if (enabledItems.length === 0) {
    return -1;
  }

  let nextIndex = currentIndex;

  for (let count = 0; count < items.length; count += 1) {
    nextIndex =
      direction === 'next'
        ? (nextIndex + 1) % items.length
        : (nextIndex - 1 + items.length) % items.length;

    if (!items[nextIndex].disabled) {
      return nextIndex;
    }
  }

  return -1;
}

function DropdownMenu({ label, items }: DropdownMenuProps) {
  const [isOpen, setIsOpen] = React.useState(false);
  const [focusedIndex, setFocusedIndex] = React.useState(-1);

  const wrapperRef = React.useRef<HTMLDivElement>(null);
  const triggerRef = React.useRef<HTMLButtonElement>(null);
  const menuRef = React.useRef<HTMLUListElement>(null);
  const itemRefs = React.useRef<Array<HTMLButtonElement | null>>([]);

  const menuId = React.useId();
  const hasEnabledItems = items.some((item) => !item.disabled);

  /**
   * Mở menu và focus vào item phù hợp.
   * - Nếu không truyền index, focus item enabled đầu tiên.
   * - Nếu danh sách rỗng hoặc toàn disabled, chỉ mở menu để user biết không có lựa chọn.
   */
  const openMenu = (preferredIndex?: number) => {
    const firstIndex = getFirstEnabledIndex(items);
    const nextIndex = preferredIndex ?? firstIndex;

    setIsOpen(true);
    setFocusedIndex(nextIndex);
  };

  /**
   * Đóng menu.
   * shouldRestoreFocus = true dùng khi Escape hoặc chọn item.
   * Với Tab/click outside, không ép focus về trigger để tránh phá flow tự nhiên.
   */
  const closeMenu = (shouldRestoreFocus: boolean) => {
    setIsOpen(false);
    setFocusedIndex(-1);

    if (shouldRestoreFocus) {
      triggerRef.current?.focus();
    }
  };

  /**
   * Focus item khi focusedIndex thay đổi.
   * Dùng requestAnimationFrame để đảm bảo menu đã render xong trước khi focus.
   */
  React.useEffect(() => {
    if (!isOpen || focusedIndex < 0) {
      return;
    }

    const frameId = window.requestAnimationFrame(() => {
      itemRefs.current[focusedIndex]?.focus();
    });

    return () => window.cancelAnimationFrame(frameId);
  }, [isOpen, focusedIndex]);

  /**
   * Click outside:
   * Nếu click ra ngoài dropdown thì đóng menu.
   */
  React.useEffect(() => {
    if (!isOpen) {
      return;
    }

    const handlePointerDown = (event: PointerEvent) => {
      const target = event.target as Node;

      if (!wrapperRef.current?.contains(target)) {
        closeMenu(false);
      }
    };

    document.addEventListener('pointerdown', handlePointerDown);

    return () => {
      document.removeEventListener('pointerdown', handlePointerDown);
    };
  }, [isOpen]);

  /**
   * Khi items thay đổi, reset refs để tránh giữ reference cũ.
   */
  React.useEffect(() => {
    itemRefs.current = itemRefs.current.slice(0, items.length);
  }, [items.length]);

  const handleTriggerClick = () => {
    if (isOpen) {
      closeMenu(false);
      return;
    }

    openMenu();
  };

  const handleTriggerKeyDown = (
    event: React.KeyboardEvent<HTMLButtonElement>,
  ) => {
    /**
     * Enter hoặc Space:
     * - Mở dropdown.
     * - Focus item enabled đầu tiên.
     */
    if (event.key === 'Enter' || event.key === ' ') {
      event.preventDefault();
      openMenu();
      return;
    }

    /**
     * ArrowDown:
     * - Mở dropdown.
     * - Focus item enabled đầu tiên.
     */
    if (event.key === 'ArrowDown') {
      event.preventDefault();
      openMenu(getFirstEnabledIndex(items));
      return;
    }

    /**
     * ArrowUp:
     * - Mở dropdown.
     * - Focus item enabled cuối cùng.
     */
    if (event.key === 'ArrowUp') {
      event.preventDefault();
      openMenu(getLastEnabledIndex(items));
    }
  };

  const handleMenuKeyDown = (event: React.KeyboardEvent<HTMLUListElement>) => {
    /**
     * Escape:
     * - Đóng dropdown.
     * - Trả focus về trigger button.
     */
    if (event.key === 'Escape') {
      event.preventDefault();
      closeMenu(true);
      return;
    }

    /**
     * Tab:
     * - Đóng dropdown.
     * - Không trap focus.
     * - Để browser tiếp tục tab sang element tiếp theo tự nhiên.
     */
    if (event.key === 'Tab') {
      closeMenu(false);
      return;
    }

    /**
     * ArrowDown:
     * - Đi đến item enabled tiếp theo.
     * - Bỏ qua item disabled.
     * - Có wrap-around.
     */
    if (event.key === 'ArrowDown') {
      event.preventDefault();

      const nextIndex = getNextEnabledIndex(items, focusedIndex, 'next');
      setFocusedIndex(nextIndex);
      return;
    }

    /**
     * ArrowUp:
     * - Đi đến item enabled phía trước.
     * - Bỏ qua item disabled.
     * - Có wrap-around.
     */
    if (event.key === 'ArrowUp') {
      event.preventDefault();

      const previousIndex = getNextEnabledIndex(
        items,
        focusedIndex,
        'previous',
      );
      setFocusedIndex(previousIndex);
      return;
    }

    /**
     * Home:
     * - Bonus UX tốt hơn.
     * - Focus item enabled đầu tiên.
     */
    if (event.key === 'Home') {
      event.preventDefault();
      setFocusedIndex(getFirstEnabledIndex(items));
      return;
    }

    /**
     * End:
     * - Bonus UX tốt hơn.
     * - Focus item enabled cuối cùng.
     */
    if (event.key === 'End') {
      event.preventDefault();
      setFocusedIndex(getLastEnabledIndex(items));
      return;
    }

    /**
     * Enter hoặc Space:
     * - Chọn item đang focus.
     */
    if (event.key === 'Enter' || event.key === ' ') {
      event.preventDefault();

      const currentItem = items[focusedIndex];

      if (!currentItem || currentItem.disabled) {
        return;
      }

      currentItem.onClick();
      closeMenu(true);
    }
  };

  const handleItemClick = (item: MenuItem) => {
    if (item.disabled) {
      return;
    }

    item.onClick();
    closeMenu(true);
  };

  return (
    <div
      ref={wrapperRef}
      className='dropdown'
    >
      <button
        ref={triggerRef}
        className='dropdown-trigger'
        type='button'
        aria-haspopup='menu'
        aria-expanded={isOpen}
        aria-controls={menuId}
        disabled={items.length === 0}
        onClick={handleTriggerClick}
        onKeyDown={handleTriggerKeyDown}
      >
        {label}
        <span aria-hidden='true'>{isOpen ? '▲' : '▼'}</span>
      </button>

      {isOpen && (
        <ul
          ref={menuRef}
          id={menuId}
          className='dropdown-menu'
          role='menu'
          aria-label={label}
          onKeyDown={handleMenuKeyDown}
        >
          {!hasEnabledItems && (
            <li
              className='empty-menu'
              role='none'
            >
              Không có lựa chọn khả dụng
            </li>
          )}

          {items.map((item, index) => (
            <li
              key={item.id}
              role='none'
            >
              <button
                ref={(element) => {
                  itemRefs.current[index] = element;
                }}
                className={`dropdown-menu-item ${
                  focusedIndex === index ? 'is-focused' : ''
                }`}
                type='button'
                role='menuitem'
                aria-disabled={item.disabled ? 'true' : undefined}
                disabled={item.disabled}
                tabIndex={focusedIndex === index ? 0 : -1}
                onClick={() => handleItemClick(item)}
                onFocus={() => setFocusedIndex(index)}
              >
                {item.icon && <span aria-hidden='true'>{item.icon}</span>}
                <span>{item.label}</span>
              </button>
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}
```

```tsx
function getSortLabel(column: SortColumn) {
  const labels: Record<SortColumn, string> = {
    name: 'Họ tên',
    email: 'Email',
    role: 'Vai trò',
    status: 'Trạng thái',
  };

  return labels[column];
}

function getAriaSort(
  column: SortColumn,
  sortColumn: SortColumn,
  sortDirection: 'asc' | 'desc',
): 'ascending' | 'descending' | 'none' {
  if (column !== sortColumn) {
    return 'none';
  }

  return sortDirection === 'asc' ? 'ascending' : 'descending';
}

function getSortIcon(
  column: SortColumn,
  sortColumn: SortColumn,
  sortDirection: 'asc' | 'desc',
) {
  if (column !== sortColumn) {
    return '↕';
  }

  return sortDirection === 'asc' ? '↑' : '↓';
}

function getNextSortDirection(
  column: SortColumn,
  sortColumn: SortColumn,
  sortDirection: 'asc' | 'desc',
) {
  if (column !== sortColumn) {
    return 'tăng dần';
  }

  return sortDirection === 'asc' ? 'giảm dần' : 'tăng dần';
}

function sortData(
  data: TableData[],
  sortColumn: SortColumn,
  sortDirection: 'asc' | 'desc',
) {
  return [...data].sort((a, b) => {
    const firstValue = a[sortColumn];
    const secondValue = b[sortColumn];

    if (typeof firstValue === 'boolean' && typeof secondValue === 'boolean') {
      return sortDirection === 'asc'
        ? Number(firstValue) - Number(secondValue)
        : Number(secondValue) - Number(firstValue);
    }

    return sortDirection === 'asc'
      ? String(firstValue).localeCompare(String(secondValue), 'vi')
      : String(secondValue).localeCompare(String(firstValue), 'vi');
  });
}
```

```tsx
   * - Khi user qua bước mới, focus vào fieldset của bước đó.
   * - Điều này giúp keyboard user và screen reader user biết context đã thay đổi.
   */
  React.useEffect(() => {
    stepRef.current?.focus();
  }, [state.currentStep]);

  /**
   * Focus management khi có lỗi:
   * - Focus vào error summary trước để user nghe toàn bộ lỗi.
   * - Lưu lại field lỗi đầu tiên để user có thể click link hoặc Tab tới field tương ứng.
   */
  React.useEffect(() => {
    if (Object.keys(state.errors).length > 0) {
      window.requestAnimationFrame(() => {
        errorSummaryRef.current?.focus();
      });
    }
  }, [state.errors]);

  const updatePersonalInfo = (field: keyof PersonalInfo, value: string) => {
    setState((currentState) => ({
      ...currentState,
      personalInfo: {
        ...currentState.personalInfo,
        [field]: value,
      },
      errors: {
        ...currentState.errors,
        [field]: '',
      },
      status: 'idle',
    }));
  };

  const updateRentalInfo = (field: keyof RentalInfo, value: string) => {
    setState((currentState) => ({
      ...currentState,
      rentalInfo: {
        ...currentState.rentalInfo,
        [field]: value,
      },
      errors: {
        ...currentState.errors,
        [field]: '',
      },
      status: 'idle',
    }));
  };

  const goToStep = (step: WizardStep) => {
    setState((currentState) => ({
      ...currentState,
      currentStep: step,
      errors: {},
      status: 'idle',
    }));

    setAnnouncement(`Đã chuyển sang bước ${step}: ${stepTitles[step]}.`);
  };

  const handleNext = () => {
    const errors = validateCurrentStep(state);
    const visibleErrors = Object.fromEntries(
      Object.entries(errors).filter(([, message]) => Boolean(message)),
    );

    if (Object.keys(visibleErrors).length > 0) {
      firstErrorFieldRef.current = getFirstErrorField(visibleErrors);

      setState((currentState) => ({
        ...currentState,
        errors: visibleErrors,
        status: 'error',
      }));

      setAnnouncement('Biểu mẫu có lỗi. Vui lòng kiểm tra phần tóm tắt lỗi.');
      return;
    }

    if (state.currentStep < 3) {
      goToStep((state.currentStep + 1) as WizardStep);
    }
  };

  const handleBack = () => {
    if (state.currentStep > 1) {
      goToStep((state.currentStep - 1) as WizardStep);
    }
  };

  const handleSubmit = (event: React.FormEvent<HTMLFormElement>) => {
    event.preventDefault();

    const errors = {
      ...validateStepOne(state.personalInfo),
      ...validateStepTwo(state.rentalInfo),
    };

    const visibleErrors = Object.fromEntries(
      Object.entries(errors).filter(([, message]) => Boolean(message)),
    );

    if (Object.keys(visibleErrors).length > 0) {
      firstErrorFieldRef.current = getFirstErrorField(visibleErrors);

      setState((currentState) => ({
        ...currentState,
        errors: visibleErrors,
        status: 'error',
      }));

      setAnnouncement('Không thể gửi biểu mẫu vì còn lỗi cần sửa.');
      return;
    }

    setState((currentState) => ({
      ...currentState,
      errors: {},
      status: 'loading',
    }));

    setAnnouncement('Đang gửi thông tin đăng ký thuê nhà.');

    /**
     * Demo async submit:
     * Trong production, thay setTimeout bằng API call thật.
     */

      setState((currentState) => ({
        ...currentState,
        status: 'success',
      }));

      setAnnouncement('Gửi đăng ký thuê nhà thành công.');
    }, 800);
  };

  const resetWizard = () => {
    setState({
      currentStep: 1,
      personalInfo: {
        fullName: '',
        email: '',
        phone: '',
      },
      rentalInfo: {
        location: '',
        startDate: '',
        duration: '',
      },
      errors: {},
      status: 'idle',
    });

    setAnnouncement(
      'Biểu mẫu đã được đặt lại. Đang ở bước 1: Thông tin cá nhân.',
    );
  };

  const isLoading = state.status === 'loading';
  const isSuccess = state.status === 'success';

  return (

      <style>{styles}</style>

      {/*

        Cho phép keyboard user bỏ qua phần đầu trang và nhảy thẳng vào form.

        aria-live="polite" vì không cần cắt ngang user ngay lập tức.

        className='visually-hidden'
        aria-live='polite'
        aria-atomic='true'
      >
        {announcement}
      </div>

              <button
                className='wizard-button primary-button'

          </section>
        ) : (
          <form
            id='rental-form'

          >
            {/*
```

### Bảng So Sánh — "Ẩn" Element Với Screen Readers

| Method                  | Visual | Screen Reader | Dùng khi nào                                  |
| ----------------------- | ------ | ------------- | --------------------------------------------- |
| `display: none`         | Ẩn     | Ẩn            | Ẩn hoàn toàn (modal đóng, tab inactive)       |
| `visibility: hidden`    | Ẩn     | Ẩn            | Ẩn nhưng giữ layout space                     |
| `opacity: 0`            | Ẩn     | **Đọc được**  | Animation (phải kết hợp pointer-events: none) |
| `aria-hidden="true"`    | Hiện   | Ẩn            | Icon decorative, duplicate text               |
| `.visually-hidden` CSS  | Ẩn     | **Đọc được**  | Screen reader only text                       |
| `hidden` HTML attribute | Ẩn     | Ẩn            | Native hide, tương đương `display:none`       |

---

### Bảng So Sánh — Khi Nào Dùng ARIA Role Nào

| Role       | Dùng khi                                       | Native alternative                    |
| ---------- | ---------------------------------------------- | ------------------------------------- |
| `button`   | Element clickable không phải button            | `<button>` (luôn prefer)              |
| `link`     | Element navigate không phải anchor             | `<a href>` (luôn prefer)              |
| `dialog`   | Modal/popup overlay                            | HTML `<dialog>` (browser support tốt) |
| `alert`    | Error/important message cần đọc ngay           | Không có native                       |
| `status`   | Success/info message không khẩn cấp            | Không có native                       |
| `menu`     | Dropdown menu (actions, không phải navigation) | Không có native                       |
| `listbox`  | Select-like dropdown                           | `<select>`                            |
| `tab`      | Tab trong tablist                              | Không có native                       |
| `tabpanel` | Content của tab                                | Không có native                       |
| `group`    | Group inputs liên quan                         | `<fieldset>`                          |
| `search`   | Search form hoặc region                        | `<search>` (HTML5.3)                  |

> **Quy tắc vàng:** **"First rule of ARIA: Don't use ARIA"** — W3C. Luôn ưu tiên HTML semantic elements. Chỉ dùng ARIA khi HTML không có native solution.

---

### Decision Tree — Cần Thêm ARIA Không?

```
Cần add semantic information cho element?
                │
                ▼
    Có native HTML element không?
    (button, a, input, select...)
         │              │
        CÓ             KHÔNG
         │              │
         ▼              ▼
    Dùng element đó  Cần add ARIA
    KHÔNG add ARIA        │
                          ▼
              Element có visible text không?
                   │              │
                  CÓ             KHÔNG
                   │              │
                   ▼              ▼
            Text đủ rõ?        aria-label
              │       │         (trực tiếp)
             CÓ      KHÔNG
              │       │
              ▼       ▼
           Không cần  aria-labelledby
           thêm gì    (reference element khác)
```

```tsx
// Dynamic content cần aria-live để thông báo changes

function FixedForm() {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');
  const errorId = React.useId();

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    if (!email.includes('@')) {
      setError(
        'Email không hợp lệ. Vui lòng nhập đúng định dạng: ten@domain.com',
      );
    } else {
      setError('');
    }
  };

  return (
    <form
      onSubmit={handleSubmit}
      noValidate
    >
      <label htmlFor='email'>
        Email
        <span aria-hidden='true'> *</span>
        <span className='visually-hidden'> (bắt buộc)</span>
      </label>

        id='email'
        type='email'
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          if (error) setError(''); // Clear error khi user sửa

        role='alert'
        className='error-message'

          <>
            {/* Icon + text, không chỉ màu */}
            <span aria-hidden='true'>⚠ </span>

          </>
        )}
      </div>
```

### A11y Code Review Checklist

```
□ Không có <div> hoặc <span> clickable (phải là <button> hoặc <a>)
□ Tất cả <img> có alt attribute (alt="" cho decorative images)
□ Tất cả form inputs có <label> liên kết (htmlFor + id)
□ Heading hierarchy đúng (không skip levels, 1 h1 per page)
□ Không chỉ dùng màu để convey information
□ Icon buttons có aria-label
□ Dynamic content có aria-live region
□ Modal có role="dialog", aria-modal="true", và focus trap
□ Modal restore focus khi đóng
□ Navigation có aria-label (nếu có nhiều nav)
□ Error messages có role="alert" hoặc aria-live
□ Error inputs có aria-invalid="true" và aria-describedby
□ Expandable elements có aria-expanded
□ Color contrast ≥ 4.5:1 (check bằng Axe hoặc browser devtools)
□ Page hoạt động hoàn toàn chỉ bằng bàn phím
□ Skip link ở đầu page
```

```tsx
interface AccessibleModalProps {
  isOpen: boolean;
  title: string;
  children: React.ReactNode;
  triggerRef: React.RefObject<HTMLButtonElement | null>;
  onClose: () => void;
}

function AccessibleModal({
  isOpen,
  title,
  children,
  triggerRef,
  onClose,
}: AccessibleModalProps) {
  const titleId = React.useId();
  const modalRef = React.useRef<HTMLDivElement | null>(null);
  const closeButtonRef = React.useRef<HTMLButtonElement | null>(null);

  React.useEffect(() => {
    if (!isOpen) {
      return;
    }

    /**
     * Fix:
     * Khi modal mở, đưa focus vào button đóng.
     * Như vậy screen reader user biết modal đã xuất hiện.
     */
    closeButtonRef.current?.focus();

    const handleKeyDown = (event: KeyboardEvent) => {
      if (event.key === 'Escape') {
        onClose();
      }

      /**
       * Focus trap tối giản:
       * Giữ Tab không thoát khỏi modal khi modal đang mở.
       */
      if (event.key === 'Tab' && modalRef.current) {
        const focusableElements =
          modalRef.current.querySelectorAll<HTMLElement>(
            'button, a[href], input, select, textarea, [tabindex]:not([tabindex="-1"])',
          );

        const firstElement = focusableElements[0];
        const lastElement = focusableElements[focusableElements.length - 1];

        if (!firstElement || !lastElement) {
          return;
        }

        if (event.shiftKey && document.activeElement === firstElement) {
          event.preventDefault();
          lastElement.focus();
        }

        if (!event.shiftKey && document.activeElement === lastElement) {
          event.preventDefault();
          firstElement.focus();
        }
      }
    };

    document.addEventListener('keydown', handleKeyDown);

    return () => {
      document.removeEventListener('keydown', handleKeyDown);
    };
  }, [isOpen, onClose]);

  const handleClose = () => {
    onClose();

    /**
     * Fix:
     * Khi modal đóng, trả focus về button đã mở modal.
     */
    window.requestAnimationFrame(() => {
      triggerRef.current?.focus();
    });
  };

  if (!isOpen) {
    return null;
  }

  return (
    <div className='modal-overlay'>
      <div
        ref={modalRef}
        className='modal'

/* =========================================================
 * 4. Fix 2: Accessible Form Error
 * ======================================================= */

function AccessibleEmailForm() {
  const [email, setEmail] = React.useState('');
  const [error, setError] = React.useState('');
  const errorId = React.useId();

  const handleSubmit = (event: React.FormEvent<HTMLFormElement>) => {
    event.preventDefault();

    if (!email.includes('@')) {
      setError(
        'Email không hợp lệ. Vui lòng nhập đúng định dạng: ten@domain.com.',
      );
      return;
    }

    setError('');
  };

  return (
    <form
      onSubmit={handleSubmit}
      noValidate
    >
      <label htmlFor='email'>Email</label>

      <input
        id='email'
        type='email'
        value={email}
        aria-invalid={error ? 'true' : 'false'}
        aria-describedby={error ? errorId : undefined}
        onChange={(event) => {
          setEmail(event.target.value);

          if (error) {
            setError('');
          }
        }}
      />

      {error && (
        <p
          id={errorId}
          role='alert'
        >
          <span aria-hidden='true'>⚠ </span>
          {error}
        </p>
      )}

      <button type='submit'>Gửi</button>
    </form>
  );
}
```

## 💡 SENIOR INSIGHTS

### Cân Nhắc Production

**1. A11y ngay từ đầu, không phải sau cùng:**

Cost của việc fix A11y issues:

- Thiết kế: 1x (rẻ nhất)
- Development: 3x
- QA/Testing: 5x
- Post-launch: 10x+

> Rule of thumb: Nếu component không navigable bằng bàn phím, nó chưa xong.

**2. Automated testing chỉ bắt được ~30-40% A11y issues:**

```tsx
// axe-core bắt được: missing alt text, contrast issues, aria mistakes
// axe-core KHÔNG bắt được: focus order logic sai, confusing screen reader experience
// → Cần manual testing với screen reader
```

**3. Accessibility là SEO:**

Search engine crawlers hoạt động như screen readers. Semantic HTML, alt text, heading hierarchy → tất cả ảnh hưởng đến ranking.

**4. ARIA attribute order không ảnh hưởng đến function:**

```tsx
// Cả hai đều hoạt động như nhau
<button aria-label="Close" aria-expanded={false}>...</button>
<button aria-expanded={false} aria-label="Close">...</button>
// Nhưng nên nhất quán trong codebase
```

### Câu Hỏi Phỏng Vấn

**Junior:**

> "alt text trong thẻ img dùng để làm gì? Khi nào nên để alt rỗng (`alt=""`)?"

_Expected answer:_ alt text là mô tả thay thế cho image, được screen readers đọc và hiển thị khi ảnh không load. alt="" (rỗng) cho images có tính decorative — ảnh chỉ để đẹp, không mang thông tin — để screen readers bỏ qua.

**Mid-level:**

> "Giải thích sự khác biệt giữa aria-label, aria-labelledby, và aria-describedby. Cho ví dụ khi dùng từng cái."

_Expected answer:_ aria-label cung cấp label string trực tiếp (dùng khi không có visible label). aria-labelledby reference ID của element khác đang là label (dùng khi label đã visible trên page). aria-describedby cung cấp additional description, không phải label (dùng cho hints, error messages).

**Senior:**

> "Bạn nhận được yêu cầu thêm một combobox (searchable select) custom vào design system. Hãy mô tả approach của bạn để đảm bảo component này accessible."

_Expected answer:_ Refer đến WAI-ARIA Combobox pattern. Cần: input có role="combobox", aria-expanded, aria-autocomplete, aria-controls trỏ đến listbox; listbox có role="listbox"; options có role="option", aria-selected; keyboard: Arrow keys navigate list, Enter select, Escape close; focus management; aria-activedescendant để indicate focused option. Testing với VoiceOver và NVDA.
