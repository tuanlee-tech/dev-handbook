## 🤔 Kiểm tra đầu vào (5 phút)

1. Sự khác biệt giữa `interface` và `type` là gì? Khi nào bạn chọn cái nào?
   - _Đáp án: `interface` hỗ trợ declaration merging, dùng `extends` để kế thừa, thường dùng cho object shapes (props, state). `type` hỗ trợ union, intersection, tuple, linh hoạt hơn. Chọn `interface` cho props component, `type` cho union/complex types. Quan trọng nhất là nhất quán trong codebase._

2. Tại sao `React.FC` không được khuyến nghị? Bạn thay thế bằng gì?
   - _Đáp án: `React.FC` tự động thêm `children?: ReactNode` ngầm (không cần thiết), ẩn return type, gây khó debug, kém linh hoạt với generic. Từ React 18+ là anti-pattern. Thay thế bằng function declaration thường/ arrow function với explicit props interface. Nếu cần children thì khai báo rõ_

3. `ReactNode` và `ReactElement` khác nhau thế nào?
   - _Đáp án: `ReactNode` rộng hơn — chấp nhận string, number, JSX, null, undefined, array. `ReactElement` chỉ chấp nhận JSX element duy nhất. Dùng `ReactNode` trong 80% trường hợp, `ReactElement` khi cần restrict chỉ JSX._

### 1.2 Mental Model — Generics như "Khuôn Đúc"

```
Analogy: Generic như khuôn đúc bánh.

Khuôn            → Generic Type Parameter <T>
Nguyên liệu      → Type cụ thể (string, number, User...)
Bánh thành phẩm  → Component/function đã được type

Cùng 1 khuôn (generic) → nhiều loại bánh (string select, number select...)
Không cần viết lại khuôn cho từng loại nguyên liệu!
```

```typescript
// Generic function — T là "placeholder" cho type thực tế
function identity<T>(value: T): T {
  return value;
}

identity<string>('hello'); // T = string → returns string
identity<number>(42); // T = number → returns number
identity('hello'); // TypeScript TỰ SUY RA T = string (type inference!)

// Constraint (ràng buộc) dùng để giới hạn
// kiểu dữ liệu được phép truyền vào Generic.

// Generic với constraint — T phải có property 'id'
// T chỉ được phép là object có ít nhất thuộc tính id: string
function getById<T extends { id: string }>(
  items: T[],
  id: string,
): T | undefined {
  return items.find((item) => item.id === id);
}
```

---

### 1.3 Mental Model — Utility Types như "Bộ Công Cụ Biến Đổi"

```
Utility Types = Công cụ biến đổi type có sẵn → type mới

Type gốc: User { id, name, email, password, role, createdAt, bio? }
                               ↓
Partial<User>              → Tất cả fields thành optional   (dùng cho update form)
Required<User>             → Tất cả fields thành required
Pick<User, 'id' | 'name'>  → Chỉ giữ lại id và name
Omit<User, 'password'>     → Bỏ password, giữ lại phần còn lại
Readonly<User>             → Không thể thay đổi sau khi tạo
Record<string, User>       → Object với key là string, value là User
```

---

### 1.4 Hiểu Lầm Phổ Biến

**❌ Hiểu lầm 1: "Generic quá phức tạp, không cần thiết"**

Sai. Bất cứ khi nào bạn copy-paste một component/function chỉ để đổi type → đó là dấu hiệu cần generic.

**❌ Hiểu lầm 2: "Utility types chỉ là syntactic sugar"**

Không đúng. `Omit<User, 'password'>` tạo ra một **type mới** được TypeScript track riêng biệt. Khi `User` thay đổi, `Omit<User, 'password'>` tự động cập nhật theo — không cần sửa tay.

**❌ Hiểu lầm 3: "Cứ dùng `Partial<T>` cho mọi thứ"**

`Partial<T>` làm tất cả fields optional → mất type safety. Chỉ dùng khi thực sự cần (update forms, PATCH operations).

```tsx
interface ListProps<T> {
  items: T[];
  renderItem: (item: T, index: number) => React.ReactNode;
  keyExtractor: (item: T) => string;
  emptyMessage?: string;
}

function List<T>({
  items,
  renderItem,
  keyExtractor,
  emptyMessage = 'Không có dữ liệu',
}: ListProps<T>) {
  if (items.length === 0) {
    return <p className="empty-state">{emptyMessage}</p>;
  }
  return (
    <ul>
      {items.map((item, index) => (
        <li key={keyExtractor(item)}>{renderItem(item, index)}</li>
      ))}
    </ul>
  );
}
```

**Generic Select Component — Ví dụ thực tế nhất:**

```tsx
interface Option<T> {
  value: T;
  label: string;
  disabled?: boolean;
}

interface SelectProps<T> {
  label: string;
  options: Option<T>[];
  value: T | null;
  onChange: (value: T) => void;
  placeholder?: string;
  disabled?: boolean;
  error?: string;
  required?: boolean;
  id: string;
}

// ⚠️ Lưu ý cú pháp: <T,> có dấu phẩy trong .tsx
// để TypeScript không nhầm với JSX opening tag
function Select<T,>({
  label,
  options,
  value,
  onChange,
  placeholder = 'Chọn một giá trị...',
  disabled = false,
  error,
  required = false,
  id,
}: SelectProps<T>) {
  const selectedIndex = options.findIndex(opt => opt.value === value);

  return (
    <div className={`select-wrapper ${error ? 'has-error' : ''}`}>
      <label htmlFor={id}>
        {label}
        {required && (
          <span aria-hidden="true" className="required-mark"> *</span>
        )}
      </label>

      <select
        id={id}
        disabled={disabled}
        aria-required={required}
        aria-invalid={!!error}
        aria-describedby={error ? `${id}-error` : undefined}
        // HTML <select> chỉ làm việc với string — ta map index → value
        value={selectedIndex >= 0 ? selectedIndex.toString() : ''}
        onChange={(e) => {
          const index = parseInt(e.target.value, 10);
          if (!isNaN(index) && options[index] && !options[index].disabled) {
            onChange(options[index].value);
          }
        }}
      >
        <option value="" disabled>{placeholder}</option>
        {options.map((option, index) => (
          <option
            key={index}
            value={index.toString()}
            disabled={option.disabled}
          >
            {option.label}
          </option>
        ))}
      </select>

      {error && (
        <span id={`${id}-error`} className="error-message" role="alert">
          {error}
        </span>
      )}
    </div>
  );
}

// --- Sử dụng với các value types khác nhau ---

// String values
```

### Demo 2: Utility Types Thực Tế ⭐⭐

**6 Utility types bạn sẽ dùng hàng ngày:**

```tsx
// --- Base type ---
interface User {
  id: string;
  name: string;
  email: string;
  password: string;
  role: 'admin' | 'user' | 'seller';
  createdAt: Date;
  bio?: string;
}

// ─────────────────────────────────────────────
// 1. Partial<T> — Mọi fields thành optional
//    Use case: Update/PATCH payload
// ─────────────────────────────────────────────
async function updateUser(
  id: string,
  updates: Partial<Omit<User, 'id' | 'createdAt'>>, // Kết hợp 2 utility types!
): Promise<User> {
  const response = await fetch(`/api/users/${id}`, {
    method: 'PATCH',
    body: JSON.stringify(updates),
    headers: { 'Content-Type': 'application/json' },
  });
  return response.json();
}

// Chỉ update những gì cần — không cần truyền đủ fields
await updateUser('123', { name: 'Tuân Mới', bio: 'Frontend Dev' }); // ✅

// ─────────────────────────────────────────────
// 2. Required<T> — Mọi fields thành bắt buộc
//    Use case: Sau validation, đảm bảo tất cả có giá trị
// ─────────────────────────────────────────────
type ValidatedUser = Required<User>;
// bio không còn optional → bắt buộc phải có

// ─────────────────────────────────────────────
// 3. Pick<T, K> — Chỉ lấy một số fields
//    Use case: Public profile (ẩn password, createdAt)
// ─────────────────────────────────────────────
type UserPublicProfile = Pick<User, 'id' | 'name' | 'bio'>;
// { id: string; name: string; bio?: string }

function PublicProfileCard({ user }: { user: UserPublicProfile }) {
  // TypeScript biết user.password KHÔNG TỒN TẠI ở đây
  // → Không thể vô tình expose password!
  return (
    <div>
      <h2>{user.name}</h2>
      {user.bio && <p>{user.bio}</p>}
    </div>
  );
}

// ─────────────────────────────────────────────
// 4. Omit<T, K> — Bỏ một số fields
//    Use case: Create payload (server tự generate id, createdAt)
// ─────────────────────────────────────────────
type CreateUserPayload = Omit<User, 'id' | 'createdAt'>;
// { name, email, password, role, bio? }

async function createUser(payload: CreateUserPayload): Promise<User> {
  const response = await fetch('/api/users', {
    method: 'POST',
    body: JSON.stringify(payload),
    headers: { 'Content-Type': 'application/json' },
  });
  return response.json();
}

// ─────────────────────────────────────────────
// 5. Record<K, V> — Object với key type K, value type V
//    Use case: Lookup table, cache, dictionary
// ─────────────────────────────────────────────
type UserRole = 'admin' | 'user' | 'seller';
type PermissionMap = Record<UserRole, string[]>;

// TypeScript đảm bảo tất cả keys đều có — không thể bỏ sót
const permissions: PermissionMap = {
  admin: ['read', 'write', 'delete', 'manage-users'],
  user: ['read'],
  seller: ['read', 'write', 'manage-products'],
};

// ─────────────────────────────────────────────
// 6. Readonly<T> — Không thể thay đổi sau khi tạo
//    Use case: Config objects, constants
// ─────────────────────────────────────────────
type AppConfig = Readonly<{
  apiUrl: string;
  timeout: number;
  maxRetries: number;
}>;

const config: AppConfig = {
  apiUrl: 'https://api.example.com',
  timeout: 5000,
  maxRetries: 3,
};

config.apiUrl = 'https://other.com'; // ❌ Lỗi: Cannot assign to 'apiUrl' (readonly)
```

### Demo 3: Typing Hooks Nâng Cao ⭐⭐

**useRef — 3 trường hợp khác nhau:**

```tsx
import { useRef, useEffect } from 'react';

// ─────────────────────────────────────────────
// Trường hợp 1: Ref cho DOM element
// Initial value: null (element chưa mount)
// ─────────────────────────────────────────────
function AutoFocusInput() {
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    inputRef.current?.focus(); // Optional chaining — safe khi chưa mount
  }, []);

  return (
    <input
      ref={inputRef}
      placeholder='Auto-focused!'
    />
  );
}

// ─────────────────────────────────────────────
// Trường hợp 2: Ref lưu mutable value (không trigger re-render)
// ─────────────────────────────────────────────
function Timer() {
  const [seconds, setSeconds] = React.useState(0);
  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);

  const start = () => {
    if (intervalRef.current) clearInterval(intervalRef.current);
    intervalRef.current = setInterval(() => {
      setSeconds((s) => s + 1);
    }, 1000);
  };

  const stop = () => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
  };

  useEffect(() => {
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
    };
  }, []);

  return (
    <div>
      <p>Thời gian: {seconds}s</p>
      <button onClick={start}>Bắt đầu</button>
      <button onClick={stop}>Dừng</button>
    </div>
  );
}

// ─────────────────────────────────────────────
// Trường hợp 3: Generic custom hook — usePrevious<T>
// ─────────────────────────────────────────────
function usePrevious<T>(value: T): T | undefined {
  const previousRef = useRef<T | undefined>(undefined);

  useEffect(() => {
    previousRef.current = value;
  });

  return previousRef.current;
}

// Sử dụng:
function Counter() {
  const [count, setCount] = React.useState(0);
  const previousCount = usePrevious(count); // TypeScript suy ra: number | undefined

  return (
    <div>
      <p>Hiện tại: {count}</p>
      <p>Trước đó: {previousCount ?? 'N/A'}</p>
      <button onClick={() => setCount((c) => c + 1)}>+1</button>
    </div>
  );
}
```

**useFetch — Custom hook generic cho mọi API response:**

```tsx
// Discriminated Union cho fetch state (xem thêm ở Demo 5)
type FetchState<T> =
  | { status: 'idle'; data: null; error: null }
  | { status: 'loading'; data: null; error: null }
  | { status: 'success'; data: T; error: null }
  | { status: 'error'; data: null; error: string };

function useFetch<T>(url: string): FetchState<T> {
  const [state, setState] = React.useState<FetchState<T>>({
    status: 'idle',
    data: null,
    error: null,
  });

  React.useEffect(() => {
    if (!url) return;

    let cancelled = false; // Tránh setState sau unmount

    setState({ status: 'loading', data: null, error: null });

    fetch(url)
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json() as Promise<T>;
      })
      .then((data) => {
        if (!cancelled) setState({ status: 'success', data, error: null });
      })
      .catch((err) => {
        if (!cancelled)
          setState({
            status: 'error',
            data: null,
            error: err instanceof Error ? err.message : 'Unknown error',
          });
      });

    return () => {
      cancelled = true;
    };
  }, [url]);

  return state;
}

// --- Sử dụng ---
interface Product {
  id: string;
  name: string;
  price: number;
}

function ProductList() {
  const state = useFetch<Product[]>('/api/products');
  // TypeScript biết:
  // status === 'success' → data là Product[] (guaranteed, không phải null)
  // status === 'error'   → error là string (guaranteed, không phải null)

  if (state.status === 'loading') return <p>Đang tải...</p>;
  if (state.status === 'error') return <p>Lỗi: {state.error}</p>;
  if (state.status === 'idle') return null;

  // ✅ Ở đây TypeScript biết chắc state.data là Product[]
  return (
    <ul>
      {state.data.map((product) => (
        <li key={product.id}>{product.name}</li>
      ))}
    </ul>
  );
}
```

### Demo 4: Type-safe Context API ⭐⭐⭐

Đây là pattern mà **hầu hết junior devs làm sai** khi kết hợp Context với TypeScript:

```tsx
// ❌ Cách SAI — Buộc phải dùng non-null assertion (!)
interface ThemeContextType {
  theme: 'light' | 'dark';
  toggleTheme: () => void;
}

const ThemeContext = React.createContext<ThemeContextType | undefined>(
  undefined,
);

function useTheme() {
  const context = React.useContext(ThemeContext);
  return context!; // ❌ Nguy hiểm! Nếu dùng ngoài Provider → runtime crash!
}
```

```tsx
// ✅ Cách ĐÚNG — Type-safe với custom error message rõ ràng

interface ThemeContextType {
  theme: 'light' | 'dark';
  toggleTheme: () => void;
}

const ThemeContext = React.createContext<ThemeContextType | undefined>(
  undefined,
);

function useTheme(): ThemeContextType {
  const context = React.useContext(ThemeContext);

  if (context === undefined) {
    // Lỗi này xuất hiện ngay khi dev dùng sai → dễ debug
    throw new Error(
      'useTheme phải được dùng bên trong <ThemeProvider>.\n' +
        'Kiểm tra component tree của bạn.',
    );
  }

  return context; // context: ThemeContextType ✅ (không cần !)
}

// Provider với useMemo + useCallback để tránh re-render không cần thiết
interface ThemeProviderProps {
  children: React.ReactNode;
  defaultTheme?: 'light' | 'dark';
}

function ThemeProvider({
  children,
  defaultTheme = 'light',
}: ThemeProviderProps) {
  const [theme, setTheme] = React.useState<'light' | 'dark'>(defaultTheme);

  const toggleTheme = React.useCallback(() => {
    setTheme((prev) => (prev === 'light' ? 'dark' : 'light'));
  }, []);

  // ⚠️ Quan trọng: memoize value để tránh mọi consumer re-render
  // khi Provider re-render vì lý do khác
  const value = React.useMemo(
    () => ({ theme, toggleTheme }),
    [theme, toggleTheme],
  );

  return (
    <ThemeContext.Provider value={value}>{children}</ThemeContext.Provider>
  );
}
```

```tsx
/**
 * Nhóm mảng items thành object theo key do getKey trả về
 * @template T — kiểu dữ liệu của item
 * @template K — kiểu dữ liệu của key (phải là string)
 */
function groupBy<T, K extends string>(
  items: T[],
  getKey: (item: T) => K,
): Record<K, T[]> {
  return items.reduce(
    (acc, item) => {
      const key = getKey(item);
      if (!acc[key]) {
        acc[key] = [];
      }
      acc[key].push(item);
      return acc;
    },
    {} as Record<K, T[]>,
  );
}
```

```tsx
import React, {
  useState,
  useCallback,
  useRef,
  ChangeEvent,
  FormEvent,
} from 'react';

// ============================================
// GENERIC useForm HOOK
// ============================================

/**
 * Type cho errors — mỗi field có thể có lỗi (string) hoặc không (undefined)
 */
type FormErrors<T> = Partial<Record<keyof T, string>>;

/**
 * Type cho touched — mỗi field có thể đã được chạm (boolean)
 */
type FormTouched<T> = Partial<Record<keyof T, boolean>>;

/**
 * Generic useForm hook — type-safe cho mọi form shape
 * @template T — kiểu dữ liệu form, extends Record<string, string | number | boolean>
 * @param initialValues — giá trị khởi tạo của form
 * @param validate — hàm validate tùy chọn, trả về errors
 */
function useForm<T extends Record<string, string | number | boolean>>(
  initialValues: T,
  validate?: (values: T) => FormErrors<T>,
) {
  // State chính
  const [values, setValues] = useState<T>(initialValues);
  const [errors, setErrors] = useState<FormErrors<T>>({});
  const [touched, setTouched] = useState<FormTouched<T>>({});

  // Lưu initialValues để tính isDirty và reset
  // Dùng useRef để không re-render khi initialValues object reference thay đổi
  const initialRef = useRef<T>(initialValues);

  // ==========================================
  // handleChange — Coerce type dựa trên input type
  // ==========================================
  const handleChange = useCallback(
    (
      event: ChangeEvent<
        HTMLInputElement | HTMLSelectElement | HTMLTextAreaElement
      >,
    ) => {
      const { name, value, type } = event.target as HTMLInputElement;

      // Coerce value sang đúng type dựa trên input type và initial value type
      let coercedValue: string | number | boolean;

      if (type === 'checkbox') {
        // Checkbox → boolean (checked)
        coercedValue = (event.target as HTMLInputElement).checked;
      } else if (
        type === 'number' ||
        typeof initialRef.current[name] === 'number'
      ) {
        // Number input hoặc field ban đầu là number → parse
        const parsed = parseFloat(value);
        coercedValue = isNaN(parsed) ? 0 : parsed;
      } else {
        // Mặc định → string
        coercedValue = value;
      }

      setValues((prev) => ({ ...prev, [name]: coercedValue }));
      setTouched((prev) => ({ ...prev, [name]: true }));

      // Clear error khi user bắt đầu sửa
      if (errors[name as keyof T]) {
        setErrors((prev) => ({ ...prev, [name]: undefined }));
      }
    },
    [errors],
  );

  // ==========================================
  // setFieldValue — Set giá trị trực tiếp (dùng cho custom inputs)
  // ==========================================
  const setFieldValue = useCallback(
    (field: keyof T, value: T[keyof T]) => {
      setValues((prev) => ({ ...prev, [field]: value }));
      setTouched((prev) => ({ ...prev, [field]: true }));

      // Clear error
      if (errors[field]) {
        setErrors((prev) => ({ ...prev, [field]: undefined }));
      }
    },
    [errors],
  );

  // ==========================================
  // handleSubmit — Chạy validate rồi gọi onSubmit
  // ==========================================
  const handleSubmit = useCallback(
    (onSubmit: (values: T) => void | Promise<void>) =>
      async (event: FormEvent<HTMLFormElement>) => {
        event.preventDefault();

        // Validate nếu có
        if (validate) {
          const validationErrors = validate(values);
          const hasErrors = Object.keys(validationErrors).length > 0;

          if (hasErrors) {
            setErrors(validationErrors);
            // Mark tất cả fields là touched để hiển thị errors
            const allTouched = Object.keys(values).reduce(
              (acc, key) => ({ ...acc, [key]: true }),
              {} as FormTouched<T>,
            );
            setTouched(allTouched);
            return;
          }
        }

        await onSubmit(values);
      },
    [values, validate],
  );

  // ==========================================
  // reset — Về initial state
  // ==========================================
  const reset = useCallback(() => {
    setValues(initialRef.current);
    setErrors({});
    setTouched({});
  }, []);

  // isDirty — Form có thay đổi so với initial không?
```

```tsx
// TYPES
// ============================================

/** Constraint: mọi data row phải có id */
interface WithId {
  id: string;
}

  key: keyof T;
  /** Header text */
  header: string;
  /** Custom render cell — value là T[key] exact type */

  /** Cho phép sort cột này */

  width?: string;

  pagination?: PaginationState;

// COMPONENT

function DataTable<T extends WithId>({
  data,
  columns,
  loading = false,
  error,
  emptyMessage = 'Không có dữ liệu',
  sortState,
  onSort,
  selectedIds = [],
  onSelectionChange,
  pagination,
  onPageChange,
  caption,
}: DataTableProps<T>): JSX.Element {
  // --- Sort helper ---
  const handleSort = useCallback(
    (column: keyof T) => {
      if (!onSort) return;
      const newDirection =
        sortState?.column === column && sortState.direction === 'asc'
          ? 'desc'
          : 'asc';
      onSort({ column, direction: newDirection });
    },
    [onSort, sortState],
  );

  // --- Selection helpers ---
  const isAllSelected =
    data.length > 0 && data.every((row) => selectedIds.includes(row.id));
  const isIndeterminate = selectedIds.length > 0 && !isAllSelected;

  const toggleSelectAll = useCallback(() => {
    if (!onSelectionChange) return;
    onSelectionChange(isAllSelected ? [] : data.map((row) => row.id));
  }, [isAllSelected, data, onSelectionChange]);

  const toggleSelectRow = useCallback(
    (id: string) => {
      if (!onSelectionChange) return;
      onSelectionChange(
        selectedIds.includes(id)
          ? selectedIds.filter((s) => s !== id)
          : [...selectedIds, id],
      );
    },
    [selectedIds, onSelectionChange],
  );

  // --- Pagination helpers ---
  const totalPages = pagination
    ? Math.ceil(pagination.total / pagination.pageSize)
    : 1;
  const startItem = pagination
    ? (pagination.page - 1) * pagination.pageSize + 1
    : 1;
  const endItem = pagination
    ? Math.min(pagination.page * pagination.pageSize, pagination.total)
    : data.length;

  // ==========================================
  // RENDER
  // ==========================================

  // Loading state
  if (loading) {
    return (
      <div
        className='data-table-wrapper'
        aria-busy='true'
        aria-live='polite'
      >
        <div className='loading-overlay'>⏳ Đang tải dữ liệu...</div>
      </div>
    );
  }

  // Error state
  if (error) {
    return (
      <div
        className='data-table-wrapper'
        role='alert'
        aria-live='assertive'
      >
        <div className='error-state'>❌ {error}</div>
      </div>
    );
  }

  // Empty state
  if (data.length === 0) {
    return (
      <div className='data-table-wrapper'>
        <p
          role='status'
          className='empty-state'
        >
          {emptyMessage}
        </p>
      </div>
    );
  }

  return (
    <div className='data-table-wrapper'>
      <table
        aria-busy={loading}
        aria-describedby={caption ? 'table-caption' : undefined}
      >
        {caption && <caption id='table-caption'>{caption}</caption>}

        <thead>
          <tr>
            {/* Checkbox "Select All" */}
            {onSelectionChange && (
              <th
                scope='col'
                className='select-col'
              >
                <input
                  type='checkbox'
                  checked={isAllSelected}
                  ref={(el) => {
                    if (el) el.indeterminate = isIndeterminate;
                  }}
                  onChange={toggleSelectAll}
                  aria-label='Chọn tất cả'
                />
              </th>
            )}

            {columns.map((col) => (
              <th
                key={String(col.key)}
                scope='col'
                style={{ width: col.width }}
                aria-sort={
                  sortState?.column === col.key
                    ? sortState.direction === 'asc'
                      ? 'ascending'
                      : 'descending'
                    : undefined
                }
              >
                {col.sortable && onSort ? (
                  <button
                    className='sort-btn'
                    onClick={() => handleSort(col.key)}
                  >
                    {col.header}
                    {sortState?.column === col.key &&
                      (sortState.direction === 'asc' ? ' ▲' : ' ▼')}
                  </button>
                ) : (
                  col.header
                )}
              </th>
            ))}
          </tr>
        </thead>

        <tbody>
          {data.map((row) => (
            <tr
              key={row.id}
              aria-selected={selectedIds.includes(row.id)}
            >
              {/* Row checkbox */}
              {onSelectionChange && (
                <td className='select-col'>
                  <input
                    type='checkbox'
                    checked={selectedIds.includes(row.id)}
                    onChange={() => toggleSelectRow(row.id)}
                    aria-label={`Chọn dòng ${row.id}`}
                  />
                </td>
              )}

              {columns.map((col) => (
                <td key={String(col.key)}>
                  {col.render
                    ? col.render(row[col.key], row)
                    : String(row[col.key])}
                </td>
              ))}
            </tr>
          ))}
        </tbody>
      </table>

      {/* Pagination */}
      {pagination && onPageChange && (
        <div
          className='pagination-bar'
          role='navigation'
          aria-label='Pagination'
        >
          <span>
            Hiển thị {startItem}-{endItem} / {pagination.total}
          </span>
          <div className='page-buttons'>
            <button

            >
              ←
            </button>
            <span aria-current='page'>Trang {pagination.page}</span>
            <button
              onClick={() => onPageChange(pagination.page + 1)}
              disabled={pagination.page >= totalPages}
              aria-label='Trang sau'
            >
              →
            </button>
          </div>
        </div>
      )}
    </div>
  );
}
```

### Bảng So Sánh — Khi Nào Dùng Utility Type Nào

| Situation                                  | Utility Type    | Ví dụ                                      |
| ------------------------------------------ | --------------- | ------------------------------------------ |
| Update/PATCH form — mọi field optional     | `Partial<T>`    | `Partial<Omit<User, 'id'>>`                |
| Sau validation — đảm bảo tất cả có giá trị | `Required<T>`   | `Required<FormData>`                       |
| Chỉ cần một số fields cụ thể               | `Pick<T, K>`    | `Pick<User, 'id' \| 'name'>`               |
| Bỏ fields nhạy cảm / server-managed        | `Omit<T, K>`    | `Omit<User, 'password'>`                   |
| Lookup table / dictionary                  | `Record<K, V>`  | `Record<UserRole, string[]>`               |
| Config / constants immutable               | `Readonly<T>`   | `Readonly<AppConfig>`                      |
| Lấy type từ function return                | `ReturnType<F>` | `ReturnType<typeof createUser>`            |
| Lấy một variant từ union                   | `Extract<T, U>` | `Extract<CartState, {status:'has-items'}>` |
| Bỏ một variant khỏi union                  | `Exclude<T, U>` | `Exclude<Status, 'deleted'>`               |

### Bug 1: Generic Inference Thất Bại — Array vs Tuple

```tsx
// ❌ Code có bug — TypeScript không infer đúng:

function createPair<T, U>(first: T, second: U) {
  return [first, second];
  // TypeScript infer: (T | U)[]
  // Thay vì:          [T, U]
}

const pair = createPair('hello', 42);
// pair[0] → string | number ❌ (phải là string)
// pair[1] → string | number ❌ (phải là number)

// ❓ Tại sao? Có bao nhiêu cách sửa?
```

**Giải thích và fix:**

```tsx
// 🔍 VẤN ĐỀ:
// TypeScript infer array literal là (T | U)[]
// vì arrays có thể có bất kỳ độ dài nào với bất kỳ thứ tự nào
// TypeScript không tự biết bạn muốn fixed-length tuple [T, U]

// ✅ Fix 1: Explicit return type
function createPair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

// ✅ Fix 2: `as const` — return readonly tuple
function createPair<T, U>(first: T, second: U) {
  return [first, second] as const; // readonly [T, U]
}

const pair = createPair('hello', 42);
const s = pair[0]; // string ✅
const n = pair[1]; // number ✅
```

```tsx
      }}

      {children}

// ❓ Vấn đề cụ thể là gì?
// ❓ Cách sửa đúng với TypeScript?
```

**Fix:**

```tsx
// 🔍 VẤN ĐỀ:
// Mỗi render của CounterProvider → {} object mới → mọi consumer re-render
// ngay cả khi count không thay đổi (ví dụ parent re-render vì lý do khác)

// ✅ Fix: useMemo + useCallback
function CounterProvider({ children }: { children: React.ReactNode }) {
  const [count, setCount] = React.useState(0);

  const increment = React.useCallback(
    () => setCount((c) => c + 1),
    [], // increment stable — không bao giờ thay đổi
  );

  const value = React.useMemo<CounterContextType>(
    () => ({ count, increment }),
    [count, increment],
    // value thay đổi CHỈ khi count thay đổi
  );

  return (
    <CounterContext.Provider value={value}>{children}</CounterContext.Provider>
  );
}
```

### Bug 3: Quên Handle Case Mới — Exhaustive Check

```tsx
// ❌ Thiếu exhaustive check — Nguy hiểm khi extend union type:

type Status = 'pending' | 'active' | 'suspended';

function getStatusColor(status: Status): string {
  if (status === 'pending') return 'yellow';
  if (status === 'active') return 'green';
  // Quên 'suspended'! TypeScript KHÔNG báo lỗi
  // → Runtime: trả về undefined → lỗi style!
}

// ❓ Làm sao để TypeScript BẮT BUỘC handle mọi case?
```

**Fix với `assertNever`:**

```tsx
// ✅ assertNever — compile error nếu thiếu case

function assertNever(value: never, message?: string): never {
  throw new Error(message ?? `Unhandled case: ${JSON.stringify(value)}`);
}

function getStatusColor(status: Status): string {
  if (status === 'pending') return 'yellow';
  if (status === 'active') return 'green';
  if (status === 'suspended') return 'red';
  return assertNever(status); // status là `never` ở đây ✅
}

// Khi thêm 'blocked' vào Status:
type Status = 'pending' | 'active' | 'suspended' | 'blocked';
// → TypeScript báo lỗi ngay tại assertNever(status)
// → Buộc dev phải handle 'blocked' trước khi compile!

// ✅ Dùng với switch:
function StatusBadge({ status }: { status: Status }) {
  switch (status) {
    case 'pending':
      return <span className='yellow'>Chờ duyệt</span>;
    case 'active':
      return <span className='green'>Hoạt động</span>;
    case 'suspended':
      return <span className='red'>Đình chỉ</span>;
    default:
      return assertNever(status); // Compile error nếu thiếu ✅
  }
}
```

```tsx
import React, { ElementType, ComponentPropsWithoutRef, ReactNode } from 'react';

// ============================================
// POLYMORPHIC BOX COMPONENT
// ============================================

/**
 * Props riêng của Box — không bao gồm props của element target
 */
interface BoxOwnProps {
  children?: ReactNode;
  className?: string;
}

/**
 * BoxProps = BoxOwnProps + tất cả props của element C (trừ ref)
 *
 * C extends ElementType — C có thể là 'div', 'button', 'a', component, v.v.
 * ComponentPropsWithoutRef<C> — lấy tất cả props của C, không bao gồm ref
 * Omit<..., keyof BoxOwnProps> — loại bỏ props trùng với BoxOwnProps để tránh conflict
 */
type BoxProps<C extends ElementType> = BoxOwnProps &
  Omit<ComponentPropsWithoutRef<C>, keyof BoxOwnProps> & {
    /** Element sẽ render — mặc định 'div' */
    as?: C;
  };

/**
 * Polymorphic Box — render bất kỳ HTML element hoặc component nào
 *
 * @template C — kiểu element (inferred từ prop `as`)
 */
function Box<C extends ElementType = 'div'>({
  as,
  children,
  className,
  ...rest
}: BoxProps<C>): JSX.Element {
  const Component = as || 'div';

  return (
    <Component
      className={className}
      {...rest}
    >
      {children}
    </Component>
  );
}

// ============================================
// TEST
// ============================================

function TestBox(): JSX.Element {
  const handleClick = (event: React.MouseEvent<HTMLButtonElement>) => {
    console.log('Button clicked!', event.currentTarget.textContent);
  };

  const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    console.log('Input value:', event.target.value);
  };

  return (
    <div>
      {/* ✅ Render như <div> — mặc định */}
      <Box>Content thuần</Box>

      {/* ✅ Render như <div> với className */}
      <Box className='container'>Có className</Box>

      {/* ✅ Render như <button> — onClick type đúng */}
      <Box
        as='button'
        onClick={handleClick}
      >
        Click me
      </Box>

      {/* ✅ Render như <a> — href type đúng */}
      <Box
        as='a'
        href='/home'
      >
        Link
      </Box>

      {/* ✅ Render như <input> — onChange type đúng */}
      <Box
        as='input'
        onChange={handleChange}
        placeholder='Nhập...'
      />

      {/* ✅ Render như custom component */}
      <Box
        as='span'
        style={{ color: 'red' }}
      >
        Custom span
      </Box>

      {/* ❌ Compile error — href không tồn tại trên button */}
      {/* 
      <Box as="button" href="/home">
        Sai!
      </Box>
      */}
      {/* _Đáp án: SAI — Property 'href' does not exist on type
          'IntrinsicAttributes & BoxProps<"button">'.
          TypeScript biết `as="button"` → chỉ có props của HTMLButtonElement,
          href là props của HTMLAnchorElement → compile error. */}
    </div>
  );
}
```

### Cân Nhắc Production

**1. Đừng over-generic — biết khi nào dừng:**

```tsx
// ❌ Over-engineered: Generic không cần thiết
function DisplayName<T extends { name: string }>({ item }: { item: T }) {
  return <span>{item.name}</span>;
}

// ✅ Đủ dùng: Interface đơn giản
interface WithName {
  name: string;
}
function DisplayName({ item }: { item: WithName }) {
  return <span>{item.name}</span>;
}

// Quy tắc: Chỉ dùng generic khi return type
// hoặc behavior phụ thuộc vào T
```

```tsx
// satisfies: Validate type mà không làm mất literal type inference
const config = {
  theme: 'dark', // type là 'dark', không bị widening thành 'string'
  fontSize: 16,
} satisfies Partial<AppConfig>;

config.theme; // 'dark' ✅ (literal type được giữ nguyên)
// Khác với `as AppConfig` — satisfies không cast, chỉ validate
```

```
  components/
    Button/
      index.tsx
      Button.types.ts  ← Component-specific types (ButtonProps, variants...)
```

### Câu Hỏi Phỏng Vấn

**Junior:**

> "Utility types như `Partial<T>` và `Omit<T, K>` dùng khi nào?"

_Expected answer:_ `Partial<T>` khi cần form update payload — chỉ truyền fields muốn thay đổi, thường dùng với PATCH. `Omit<T, K>` khi có fields nhạy cảm hoặc server-managed không muốn expose (id, password, createdAt). Ưu điểm lớn: khi base type thay đổi, derived types tự cập nhật — không cần sửa tay nhiều nơi.

**Mid-level:**

> "Giải thích Discriminated Union. Ví dụ thực tế bạn đã áp dụng?"

_Expected answer:_ Discriminated Union là union type trong đó mỗi variant có một field chung (discriminant) với literal value riêng. TypeScript dùng discriminant để tự động narrow type — không cần cast. Ví dụ thực tế: API fetch state (`status: 'idle'|'loading'|'success'|'error'`) — khi check `status === 'success'`, TypeScript biết chắc `data` tồn tại và không phải null. Tôi đã dùng cho shopping cart state machine và alert component variants với props khác nhau theo type.

**Senior:**

> "Tại sao bạn cần `useMemo` và `useCallback` trong Context Provider? Không có thì sao?"

_Expected answer:_ Không có `useMemo` cho value object: mỗi render của Provider tạo `{}` mới → reference change → tất cả consumer re-render kể cả khi data không đổi. Không có `useCallback` cho functions: mỗi render tạo function mới → `useMemo` dependencies thay đổi → value object mới → lại trigger consumer re-render. Kết quả: chain re-renders không cần thiết, performance degradation trong large component trees. Fix: `useCallback` trước cho tất cả functions, rồi `useMemo` bao value object với dependencies là các callback đã stable.

### War Stories

_"Chúng tôi có Notification component dùng boolean props: `isSuccess`, `isError`, `isWarning`, `isInfo`. Một ngày xuất hiện bug kỳ lạ: dev vô tình truyền `isSuccess={true} isError={true}` cùng lúc — component render màu xanh lá với icon ❌, không ai hiểu tại sao._

_Thay vì thêm validation runtime, tôi refactor sang Discriminated Union:_

```tsx
type NotificationProps =
  | { type: 'success'; message: string; onDismiss?: () => void }
  | { type: 'error'; message: string; onRetry: () => void }
  | { type: 'warning'; message: string; onConfirm: () => void }
  | { type: 'info'; message: string };
```

_Sau đó, toàn bộ class bug 'invalid state combination' biến mất. Compiler không cho phép truyền sai — không cần thêm test, không cần documentation, không cần code review checklist._

_Lesson: Thiết kế type system tốt không chỉ giúp DX — nó loại bỏ hoàn toàn một category bugs. Tốt hơn là defensive programming, tốt hơn là unit tests, vì nó hoạt động 100% thời gian ở compile time thay vì catch bugs sau khi đã xảy ra."_

## 🔮 Preview Ngày 60: Accessibility (A11y)

Ngày mai là ngày cuối cùng của Core React Path (60 ngày!). Và là một trong những ngày **quan trọng nhất với góc nhìn professional**.

**Accessibility không phải "nice to have"** — ở nhiều quốc gia đó là legal requirement (ADA tại Mỹ, EN 301 549 tại EU). Các công ty lớn đã phải trả hàng triệu USD tiền phạt vì website không accessible. Với TypeScript bạn đã học, bạn sẽ thấy TS thực sự giúp enforce a11y ở compile time.

Bạn sẽ học:

- **WCAG 2.1 nguyên tắc cốt lõi** — POUR: Perceivable, Operable, Understandable, Robust
- **Semantic HTML** — `<button>` vs `<div onClick>` và tại sao quan trọng cho screen readers
- **ARIA attributes** — `aria-label`, `aria-describedby`, `aria-expanded`, `aria-live`
- **Focus management** — focus trap trong modals, skip links, visible focus indicator
- **Keyboard navigation** — Tab, Enter, Space, Arrow keys, Escape
- **TypeScript + A11y** — TS enforce aria attributes ở compile time

**Chuẩn bị ngay bây giờ:** Cài [axe DevTools](https://www.deque.com/axe/devtools/) cho Chrome. Mở website bạn đang làm → chạy axe scan → bạn sẽ ngạc nhiên với số issues tìm được, ngay cả với code "trông có vẻ đúng".
