Đây chính là giá trị cốt lõi của TypeScript: **bắt lỗi tại thời điểm viết code (compile time), không phải lúc chạy (runtime).**

### 1.2 TypeScript là gì?

TypeScript (TS) là **superset của JavaScript** — nghĩa là mọi code JS hợp lệ đều là TS hợp lệ. TS thêm vào JS một hệ thống **static type** (kiểu tĩnh):

```
JavaScript + Type System = TypeScript
                ↓
           TypeScript Compiler (tsc)
                ↓
           JavaScript thuần (chạy trên browser)
```

**Browser không hiểu TypeScript.** TS phải được compile sang JS. Với React (Vite/CRA), quá trình này xảy ra tự động trong background.

### 1.3 Mental Model: TypeScript như "Hợp Đồng"

Hãy nghĩ TypeScript như một **hợp đồng** giữa các phần của codebase:

```
Component A (người dùng) ←→ Interface/Type ←→ Component B (người cung cấp)
         "Tôi cam kết                              "Tôi cam kết
          truyền đúng                               nhận đúng
          kiểu dữ liệu"                             kiểu này"
```

Khi hợp đồng bị vi phạm → TypeScript báo lỗi ngay lập tức.

```
Analogy thực tế:
TypeScript giống như "form đặt hàng" trong nhà hàng.
- Mục "số lượng": bắt buộc phải là số, không thể ghi "hai"
- Mục "ghi chú": tùy chọn, có thể để trống
- Nếu điền sai → form không submit được (compile error)
- Nếu không có form → nhân viên phải đoán ý khách (runtime bug)
```

### 1.5 `interface` vs `type` — Khi nào dùng cái nào?

Đây là câu hỏi mà **100% senior** sẽ hỏi bạn trong phỏng vấn. Hãy hiểu rõ:

```typescript
// Cách 1: interface
interface UserProps {
  name: string;
  age: number;
}

// Cách 2: type alias
type UserProps = {
  name: string;
  age: number;
};
```

**Một số khái niệm Typescript dùng trong bài giảng**:

**`Omit<T, K>`**

Loại bỏ các key `K` khỏi type `T`.

```typescript
interface User {
  id: string;
  name: string;
  email: string;
  password: string;
}

type PublicUser = Omit<User, 'password'>;
// → { id: string; name: string; email: string }

// Dùng trong React: tạo props mà không cần truyền prop từ cha
type ButtonProps = Omit<
  React.ButtonHTMLAttributes<HTMLButtonElement>,
  'children'
> & {
  label: string;
};
```

**Tuple**

Mảng với **số phần tử cố định** và **kiểu dữ liệu xác định từng vị trí**.

```typescript
type Point = [number, number]; // (x, y)
type User = [string, number]; // [tên, tuổi]
type RGBA = [number, number, number, number];

const p: Point = [10, 20]; // ✅
const p2: Point = [10, '20']; // ❌ Sai kiểu vị trí 1
const p3: Point = [10]; // ❌ Thiếu phần tử

// React: useState trả về tuple
const [count, setCount] = useState(0); // [number, Dispatch<number>]
```

**Declaration merging**

Khai báo `interface` cùng tên nhiều lần → TypeScript **tự động gộp** thành một.

```typescript
interface User {
  name: string;
}

interface User {
  age: number;
}

// → User = { name: string; age: number }

const user: User = { name: 'Tuân', age: 29 }; // ✅
```

**Ứng dụng thực tế:**

```typescript
// Mở rộng type của thư viện bên thứ 3
declare module 'express' {
  interface Request {
    userId?: string; // Thêm field vào Express Request
  }
}
```

**`type` không làm được:**

```typescript
type User = { name: string };
type User = { age: number }; // ❌ Error: Duplicate identifier
```

nhưng dùng `&` (intersection) để **gộp thủ công**.

```typescript
type UserBase = { name: string };
type UserExtra = { age: number };

type User = UserBase & UserExtra;
// → { name: string; age: number }

const user: User = { name: 'Tuân', age: 29 }; // ✅
```

**Cảnh báo intersection:**

⚠️ Tạo kiểu dữ liệu `never` — khó phát hiện

```typescript
type A = { x: string };
type B = { x: number };
type C = A & B;
// → { x: string & number } = { x: never }  // ❌ Không thể tồn tại
```

**Type `never`**:

Trong TypeScript, kiểu dữ liệu `never` đại diện cho những giá trị không bao giờ xuất hiện. Biểu thị _"Giá trị này không bao giờ tồn tại."_.

Khác với `void`, void nghĩa là hàm kết thúc bình thường nhưng không trả về gì. Còn `never` nghĩa là hàm không thể kết thúc bình thường.

**3 Trường hợp sử dụng phổ biến nhất:**

- Hàm luôn ném lỗi (Throw error): Dùng khi một hàm dừng chương trình đột ngột thay vì chạy xong.
  ```ts
  function fail(message: string): never {
    throw new Error(message);
  }
  ```
- Hàm chạy vô tận (Infinite loop): Dùng khi hàm chạy liên tục và không bao giờ thoát ra.
  ```ts
  function startServer(): never {
    while (true) {
      // xử lý request
    }
  }
  ```
- Kiểm tra toàn diện (Exhaustiveness checking): Dùng trong câu lệnh switch để đảm bảo bạn đã xử lý mọi trường hợp của dữ liệu

  ```ts
  function assertNever(value: never): never {
    throw new Error(`Unexpected value: ${value}`);
  }

  type Status = 'loading' | 'success' | 'error';

  function handle(status: Status) {
    switch (status) {
      case 'loading':
        return 'Loading...';

      case 'success':
        return 'Success!';

      case 'error':
        return 'Error!';

      default:
        return assertNever(status);
    }
  }
  /*  Bây giờ nếu thêm cancelled vào
   *   type  Status = | "loading" | "success" | "error" | "cancelled";
   *   thì TypeScript sẽ báo lỗi:
   *   Argument of type '"cancelled"' is not assignable to parameter
   *   of type 'never' vì switch chưa xử lý "cancelled".
   *   Đây gọi là exhaustive switch checking.
   */
  ```

**Khác biệt chính của Interface và Type:**

| Tính năng               | `interface`                            | `type`                                  |
| ----------------------- | -------------------------------------- | --------------------------------------- |
| **Declaration merging** | ✅ Có — khai báo lại cùng tên sẽ merge | ❌ Không — báo lỗi duplicate            |
| **Extend / Kế thừa**    | `extends`                              | `&` (intersection)                      |
| **Union types**         | ❌ Không                               | ✅ Có — `type Status = 'a' \| 'b'`      |
| **Tuple types**         | ❌ Không                               | ✅ Có — `type Point = [number, number]` |
| **Computed properties** | ❌ Không                               | ✅ Có — `type Keys = keyof SomeObj`     |
| **Primitive alias**     | ❌ Không                               | ✅ Có — `type ID = string`              |

> ⚠️ Primitive alias (không phải keyword chính thức trong TypeScript, nhưng là cách gọi phổ biến) nghĩa là:
>
> Dùng type để đặt tên (alias) cho một kiểu primitive như string, number, boolean.
> Ví dụ:
>
> type UserId = string;
>
> type Age = number;
>
> type Flag = boolean;
>
> => Không tạo kiểu mới thật sự
> => Chỉ giúp code rõ nghĩa và dễ maintain hơn

**Quy tắc thực tế trong React (theo React team và phần lớn senior devs):**

```typescript
// ✅ Dùng interface cho Props và State của component
interface ButtonProps {
  label: string;
  onClick: () => void;
}

// ✅ Dùng type cho union, intersection, hoặc type phức tạp
type Status = 'loading' | 'success' | 'error';
type ID = string | number;
```

> **💡 Senior Insight:** Trong thực tế, cả hai đều hoạt động tốt cho props. Quan trọng là **nhất quán trong codebase**. Nhiều team dùng `interface` cho objects, `type` cho primitives/unions. Nếu không có convention, dùng `interface` cho props — đây là convention phổ biến hơn.

```typescript
function processUser(user: any) {
  return user.name.toUpperCase(); // TS không báo lỗi nếu user là null
}

// ✅ Dùng `unknown` nếu thực sự không biết kiểu
function processUser(user: unknown) {
  if (typeof user === 'object' && user !== null && 'name' in user) {
    return (user as { name: string }).name.toUpperCase();
  }
}
```

### Demo 1: Typing Props Cơ Bản ⭐

Bắt đầu với component đơn giản nhất:

```tsx
// ❌ Cách SAI: Không có type
function Greeting({ name, age }) {
  // TypeScript sẽ báo: Parameter 'name' implicitly has an 'any' type
  return <h1>Xin chào {name}, {age} tuổi!</h1>;
}

// ✅ Cách ĐÚNG: Định nghĩa interface cho props
interface GreetingProps {
  name: string;
  age: number;
}

function Greeting({ name, age }: GreetingProps) {
  return <h1>Xin chào {name}, {age} tuổi!</h1>;
}

// Khi dùng:
<Greeting name="Tuân" age={29} />      // ✅ OK
<Greeting name="Tuân" age="29" />      // ❌ Lỗi: age phải là number
<Greeting name="Tuân" />               // ❌ Lỗi: age là bắt buộc
```

**Optional Props — Dùng `?`:**

```tsx
interface UserCardProps {
  name: string;          // Bắt buộc
  email: string;         // Bắt buộc
  avatar?: string;       // Tùy chọn (string | undefined)
  bio?: string;          // Tùy chọn
}

function UserCard({ name, email, avatar, bio }: UserCardProps) {
  return (
    <div>
      {/* avatar có thể undefined → cần fallback */}
      <img src={avatar ?? '/default-avatar.png'} alt={name} />
      <h2>{name}</h2>
      <p>{email}</p>
      {/* Render có điều kiện cho optional prop */}
      {bio && <p className="bio">{bio}</p>}
    </div>
  );
}

// Sử dụng:
<UserCard name="Tuân" email="tuan@example.com" />                    // ✅ OK
<UserCard name="Tuân" email="tuan@example.com" bio="Frontend Dev" /> // ✅ OK
```

### Demo 2: Typing Events ⭐⭐

Events là nơi **nhiều dev React mới học TS bị vấp nhất**. Hãy master phần này:

```tsx
// ❌ Cách SAI: Dùng any hoặc không type event
function SearchInput() {
  const handleChange = (event: any) => {
    // Mất toàn bộ type safety!
    console.log(event.target.value);
  };

  return <input onChange={handleChange} />;
}

// ✅ Cách ĐÚNG: Type đúng event
function SearchInput() {
  // React.ChangeEvent<HTMLInputElement> = event từ <input>
  const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    const value: string = event.target.value; // TypeScript biết đây là string
    console.log(value);
  };

  return <input onChange={handleChange} />;
}
```

**Ví dụ thực tế — Contact Form đầy đủ:**

```tsx
  children: ReactNode;

 * Button dạng <button> HTML
 * Extend từ ButtonHTMLAttributes để thừa hưởng:
 * - type, disabled, onClick, aria-*, data-*, form, name, value, etc.
 * KHÔNG có href — đây là discriminant
 */

  extends
    ButtonBaseProps,
    Omit<ButtonHTMLAttributes<HTMLButtonElement>, 'children'> {
  // href không tồn tại ở đây → TS sẽ báo lỗi nếu truyền href
}

 * Button dạng <a> link
 * Extend từ AnchorHTMLAttributes để thừa hưởng:
 * - href, target, rel, download, aria-*, data-*, etc.
 * href là bắt buộc — đây là discriminant
 */
interface ButtonAsLinkProps
  extends
    ButtonAsButtonProps,
    Omit<AnchorHTMLAttributes<HTMLAnchorElement>, 'children' | 'href'> {

  href: string;

 * Union type: Button chỉ có thể là button HOẶC link, không thể cả hai

function Button(props: ButtonProps): JSX.Element {

    variant = 'primary',
    size = 'md',
    loading = false,
    leftIcon,
    rightIcon,
    children,
    className = '',
    ...rest // Các props còn lại (native attributes)
  } = props;

  // Tạo className từ variant và size
  const baseClasses = `btn btn-${variant} btn-${size}`;
  const disabledClasses =
    loading || ('disabled' in rest && rest.disabled) ? 'btn-disabled' : '';
  const combinedClassName =
    `${baseClasses} ${disabledClasses} ${className}`.trim();

  // Spinner component inline
  const Spinner = () => (
    <span
      className='btn-spinner'
      aria-hidden='true'
    >
      ⏳
    </span>
  );

      {loading && <Spinner />}
      {!loading && leftIcon && (
        <span className='btn-icon-left'>{leftIcon}</span>
      )}
      <span className='btn-text'>{children}</span>
      {!loading && rightIcon && (
        <span className='btn-icon-right'>{rightIcon}</span>
      )}
    </>
  );

  // ==========================================
  // RENDER NHƯ <a> — khi có href

      rest as AnchorHTMLAttributes<HTMLAnchorElement>;

        className={combinedClassName}
        aria-disabled={loading}

    );

    onClick,
    ...buttonRest

      type={type}
      className={combinedClassName}
      disabled={disabled || loading}
      onClick={onClick as (event: MouseEvent<HTMLButtonElement>) => void}
      {...buttonRest}
    >
      {content}
    </button>
  );
```

```tsx
  id: string;
```

```tsx
/**
 * Props cho Select component — single value
 * @template T — kiểu dữ liệu của value (string | number)
 */
interface SelectProps<T extends string | number = string> extends BaseFormProps {
  /** Danh sách options */
  options: Option<T>[];
  /** Giá trị hiện tại (controlled) — '' nghĩa là chưa chọn */
  value: T | '';
  /** Callback khi chọn option mới */
  onChange: (value: T) => void;

/**
 * Select component — dropdown single select
 *
 * Accessibility:
 * - label được liên kết với select qua htmlFor
 * - aria-invalid khi có error
 * - aria-describedby liên kết với error message
 * - aria-required khi bắt buộc
 */
function Select<T extends string | number = string>({
  label,
  id,
  options,
  value,
  onChange,
  placeholder = 'Chọn một mục...',
  required = false,
  disabled = false,
  error,
  className = '',
}: SelectProps<T>): JSX.Element {
  const handleChange = useCallback(
    (event: React.ChangeEvent<<HTMLSelectElement>) => {
      const selectedValue = event.target.value as T;
      onChange(selectedValue);
    },
    [onChange]
  );

  const hasError = !!error;
  const selectId = id;
  const errorId = `${id}-error`;

  return (
    <div className={`form-group ${className}`}>
      <label htmlFor={selectId}>
        {label}
        {required && <span aria-label="bắt buộc"> *</span>}
      </label>

      <select
        id={selectId}
        value={value}
        onChange={handleChange}
        disabled={disabled}
        required={required}
        aria-invalid={hasError}
        aria-describedby={hasError ? errorId : undefined}
        aria-required={required}
        className={hasError ? 'input-error' : ''}
      >
        {/* Placeholder option — disabled, không có value */}
        <option value="" disabled>
          {placeholder}
        </option>

        {/* Render các options */}
        {options.map((option) => (
          <option
            key={String(option.value)}
            value={String(option.value)}
            disabled={option.disabled}
          >
            {option.label}
            {option.disabled ? ' (không khả dụng)' : ''}
          </option>
        ))}
      </select>

      {/* Error message */}
      {hasError && (
        <span id={errorId} className="error-message" role="alert">
          {error}
        </span>
      )}
    </div>
  );
}

 * @template T — kiểu dữ liệu của từng value trong mảng

  /** Danh sách options */
  options: Option<T>[];
  /** Các giá trị đã chọn (controlled) */
  value: T[];
  /** Callback khi thay đổi selection — nhận mảng value mới */
  onChange: (value: T[]) => void;
  /** Hướng dẫn bổ sung */

/**
 * CheckboxGroup component — multi select bằng checkboxes
 *
 * Keyboard accessibility:
 * - Tab: focus qua lại giữa các checkbox
 * - Space: toggle checkbox
 *
 * ARIA:
 * - fieldset + legend cho nhóm checkbox
 * - aria-checked cho trạng thái
 * - aria-disabled cho option bị vô hiệu hóa
 */
function CheckboxGroup<T extends string | number = string>({
  label,
  id,
  options,
  value,
  onChange,
  required = false,
  disabled = false,
  error,
  helperText,
  className = '',
}: CheckboxGroupProps<T>): JSX.Element {
  const groupId = id;
  const errorId = `${id}-error`;
  const helperId = `${id}-helper`;
  const hasError = !!error;

  /** Toggle một option — thêm hoặc xóa khỏi mảng value */
  const toggleOption = useCallback(
    (optionValue: T) => {
      if (disabled) return; // Không cho phép thay đổi khi disabled

      const newValue = value.includes(optionValue)
        ? value.filter((v) => v !== optionValue) // Bỏ chọn
        : [...value, optionValue]; // Chọn thêm

      onChange(newValue);
    },
    [value, onChange, disabled]
  );

  /** Xử lý keyboard — Space để toggle */
  const handleKeyDown = useCallback(
    (event: KeyboardEvent<<HTMLInputElement>, optionValue: T) => {
      if (event.key === ' ') {
        event.preventDefault();
        toggleOption(optionValue);
      }
    },
    [toggleOption]
  );

  return (
    <fieldset
      className={`form-group checkbox-group ${className}`}
      disabled={disabled}
      aria-invalid={hasError}
      aria-describedby={
        `${hasError ? errorId : ''} ${helperText ? helperId : ''}`.trim() || undefined
      }
    >
      <legend>
        {label}
        {required && <span aria-label="bắt buộc"> *</span>}
      </legend>

      {/* Helper text */}
      {helperText && (
        <div id={helperId} className="helper-text">
          {helperText}
        </div>
      )}

      {/* Render các checkbox */}
      <div className="checkbox-list" role="group">
        {options.map((option) => {
          const isChecked = value.includes(option.value);
          const isDisabled = disabled || option.disabled;
          const optionId = `${groupId}-${String(option.value)}`;

          return (
            <div key={String(option.value)} className="checkbox-item">
              <input
                type="checkbox"
                id={optionId}
                name={groupId}
                value={String(option.value)}
                checked={isChecked}
                onChange={() => toggleOption(option.value)}
                onKeyDown={(e) => handleKeyDown(e, option.value)}
                disabled={isDisabled}
                required={required && value.length === 0}
                aria-checked={isChecked}
                aria-disabled={isDisabled}
              />
              <label htmlFor={optionId}>
                {option.label}
              </label>
            </div>
          );
        })}
      </div>

      {/* Empty state */}
      {options.length === 0 && (
        <p className="empty-state">Không có tùy chọn nào</p>
      )}

      {/* Error message */}
      {hasError && (
        <span id={errorId} className="error-message" role="alert">
          {error}
        </span>
      )}
    </fieldset>
  );
}

  value: T | '';

/**
 * RadioGroup component — single select bằng radio buttons
 *
 * Keyboard accessibility:
 * - Tab: focus vào nhóm radio
 * - Arrow keys (Up/Down/Left/Right): chuyển giữa các option
 * - Space: chọn option đang focus
 *
 * ARIA:
 * - radiogroup role
 * - aria-checked cho trạng thái chọn
 * - aria-posinset + aria-setsize cho vị trí
 */
function RadioGroup<T extends string | number = string>({
  label,
  id,
  options,
  value,
  onChange,
  required = false,
  disabled = false,
  error,
  helperText,
  className = '',
}: RadioGroupProps<T>): JSX.Element {
  const groupId = id;
  const errorId = `${id}-error`;
  const helperId = `${id}-helper`;
  const hasError = !!error;

  /** Xử lý chọn option */
  const handleSelect = useCallback(
    (optionValue: T) => {
      if (!disabled) {
        onChange(optionValue);
      }
    },
    [onChange, disabled]
  );

  /** Xử lý keyboard navigation */
  const handleKeyDown = useCallback(
    (event: KeyboardEvent<<HTMLInputElement>, currentIndex: number) => {
      if (disabled) return;

      let nextIndex = currentIndex;

      switch (event.key) {
        case 'ArrowDown':
        case 'ArrowRight':
          event.preventDefault();
          nextIndex = (currentIndex + 1) % options.length;
          break;
        case 'ArrowUp':
        case 'ArrowLeft':
          event.preventDefault();
          nextIndex = (currentIndex - 1 + options.length) % options.length;
          break;
        default:
          return;
      }

      // Focus và chọn option tiếp theo
      const nextOption = options[nextIndex];
      if (!nextOption.disabled) {
        onChange(nextOption.value);
        document.getElementById(`${groupId}-${String(nextOption.value)}`)?.focus();
      }
    },
    [options, onChange, disabled, groupId]
  );

  return (
    <div
      className={`form-group radio-group ${className}`}
      role="radiogroup"
      aria-labelledby={`${groupId}-legend`}
      aria-invalid={hasError}
      aria-describedby={
        `${hasError ? errorId : ''} ${helperText ? helperId : ''}`.trim() || undefined
      }
    >
      <div id={`${groupId}-legend`} className="radio-legend">
        {label}
        {required && <span aria-label="bắt buộc"> *</span>}
      </div>

      {/* Helper text */}
      {helperText && (
        <div id={helperId} className="helper-text">
          {helperText}
        </div>
      )}

      {/* Render các radio buttons */}
      <div className="radio-list">
        {options.map((option, index) => {
          const isChecked = value === option.value;
          const isDisabled = disabled || option.disabled;
          const optionId = `${groupId}-${String(option.value)}`;

          return (
            <div key={String(option.value)} className="radio-item">
              <input
                type="radio"
                id={optionId}
                name={groupId}
                value={String(option.value)}
                checked={isChecked}
                onChange={() => handleSelect(option.value)}
                onKeyDown={(e) => handleKeyDown(e, index)}
                disabled={isDisabled}
                required={required && value === ''}
                aria-checked={isChecked}
                aria-disabled={isDisabled}
                aria-posinset={index + 1}
                aria-setsize={options.length}
              />
              <label htmlFor={optionId}>
                {option.label}
              </label>
            </div>
          );
        })}
      </div>

      {/* Empty state */}
      {options.length === 0 && (
        <p className="empty-state">Không có tùy chọn nào</p>
      )}

      {/* Error message */}
      {hasError && (
        <span id={errorId} className="error-message" role="alert">
          {error}
        </span>
      )}
    </div>
  );
}
```

## 📊 PHẦN 4: SO SÁNH PATTERNS (30 phút)

### Bảng So Sánh — Cách Type Props

| Cách         | Ví dụ                                    | Pros                                          | Cons                                      | Khi dùng                          |
| ------------ | ---------------------------------------- | --------------------------------------------- | ----------------------------------------- | --------------------------------- |
| `interface`  | `interface Props { name: string }`       | Extendable, clean syntax, declaration merging | Không support union                       | Props của component (khuyến nghị) |
| `type` alias | `type Props = { name: string }`          | Support union/intersection, flexible          | Không có declaration merging              | Union types, complex types        |
| Inline type  | `function F({ name }: { name: string })` | Nhanh, không cần đặt tên                      | Không reusable, khó đọc                   | Props 1-2 fields, internal        |
| `React.FC`   | `const F: React.FC<Props>`               | Auto-include displayName                      | Children type implicit, hides return type | **TRÁNH dùng** từ React 18+       |

---

### Bảng So Sánh — Typing Events

| Event     | HTML Element | TypeScript Type                                          |
| --------- | ------------ | -------------------------------------------------------- |
| onClick   | button, div  | `React.MouseEvent<HTMLButtonElement>`                    |
| onChange  | input text   | `React.ChangeEvent<HTMLInputElement>`                    |
| onChange  | select       | `React.ChangeEvent<HTMLSelectElement>`                   |
| onChange  | checkbox     | `React.ChangeEvent<HTMLInputElement>` (check `.checked`) |
| onSubmit  | form         | `React.FormEvent<HTMLFormElement>`                       |
| onKeyDown | input        | `React.KeyboardEvent<HTMLInputElement>`                  |
| onFocus   | any          | `React.FocusEvent<HTMLElement>`                          |

---

### Bảng So Sánh `interface` vs `type`

| Tính năng               | `interface`                            | `type`                                  |
| ----------------------- | -------------------------------------- | --------------------------------------- |
| **Declaration merging** | ✅ Có — khai báo lại cùng tên sẽ merge | ❌ Không — báo lỗi duplicate            |
| **Extend / Kế thừa**    | `extends`                              | `&` (intersection)                      |
| **Union types**         | ❌ Không                               | ✅ Có — `type Status = 'a' \| 'b'`      |
| **Tuple types**         | ❌ Không                               | ✅ Có — `type Point = [number, number]` |
| **Computed properties** | ❌ Không                               | ✅ Có — `type Keys = keyof SomeObj`     |
| **Primitive alias**     | ❌ Không                               | ✅ Có — `type ID = string`              |

**Quy tắc thực tế trong React:**

- Dùng `interface` cho **props và state** của component (vì có thể extend, dễ đọc, declaration merging hữu ích khi augment thư viện)
- Dùng `type` cho **union, tuple, primitive alias, hoặc type phức tạp**

**Ví dụ minh họa:**

```typescript
// interface có thể merge
interface User {
  name: string;
}
interface User {
  age: number;
}
// → User = { name: string; age: number }

// type union — interface không làm được
type Status = 'loading' | 'success' | 'error';
```

**Quan trọng nhất:** Cả hai đều hoạt động tốt cho props. Chọn một và **nhất quán trong codebase**.

---

### Decision Tree — Chọn Type cho Children

```
Cần type gì cho children?
        │
        ▼
Có phải function không? (render props)
   │                            │
  CÓ                           KHÔNG
   │                            │
   ▼                            ▼
(data: T) => ReactNode    Chỉ JSX element?
                             │        │
                            CÓ       KHÔNG
                             │        │
                             ▼        ▼
                        ReactElement  Cần string/number cũng được?
                                         │          │
                                        CÓ         KHÔNG
                                         │          │
                                         ▼          ▼
                                      ReactNode   ReactNode
                                   (hầu hết cases) (mặc định)
```

**Tóm lại: Dùng `ReactNode` trong 80% trường hợp. Chỉ dùng `ReactElement` khi cần restrict.**

```tsx
// 🔍 PHÂN TÍCH:
```

<summary>💡 Solution</summary>

```tsx
import React, { useState, ChangeEvent, FormEvent } from 'react';

// ============================================
// BÀI TẬP VỀ NHÀ: MIGRATION TASK
// ============================================
/**
 * Yêu cầu: Migrate component JavaScript (50+ lines, có props + events)
 * sang TypeScript hoàn chỉnh — không có `any`, tất cả props và events được type.
 *
 * Component gốc (JavaScript): TodoList — quản lý danh sách công việc
 */

// ============================================
// 1. ĐỊNH NGHĨA TYPES
// ============================================

/** Trạng thái của một todo item */
interface Todo {
  /** ID duy nhất, kiểu string (dùng Date.now() hoặc crypto.randomUUID) */
  id: string;
  /** Nội dung công việc */
  text: string;
  /** Đã hoàn thành hay chưa */
  completed: boolean;
  /** Mức độ ưu tiên — union type giới hạn 3 giá trị */
  priority: 'low' | 'medium' | 'high';
  /** Thời gian tạo */
  createdAt: number;
}

/** Props của TodoList component */
interface TodoListProps {
  /** Tiêu đề danh sách — bắt buộc, string */
  title: string;
  /** Danh sách todo ban đầu — tùy chọn, mặc định rỗng */
  initialTodos?: Todo[];
  /** Callback khi thêm todo mới — nhận text và priority, không return */
  onAddTodo?: (text: string, priority: Todo['priority']) => void;
  /** Callback khi xóa todo — nhận id, không return */
  onDeleteTodo?: (id: string) => void;
  /** Callback khi toggle hoàn thành — nhận id và trạng thái mới, không return */
  onToggleComplete?: (id: string, completed: boolean) => void;
  /** Có cho phép sửa todo không — tùy chọn, mặc định true */
  allowEdit?: boolean;
}

/** Trạng thái filter hiển thị — union type */
type FilterStatus = 'all' | 'active' | 'completed';

/** Trạng thái sắp xếp — union type */
type SortBy = 'newest' | 'priority' | 'alphabetical';

// ============================================
// 2. COMPONENT IMPLEMENTATION
// ============================================

/**
 * TodoList — Quản lý danh sách công việc với TypeScript
 *
 * Type chọn:
 * - Todo['priority']: dùng indexed access type để lấy kiểu từ interface Todo
 *   → Đảm bảo đồng bộ, khi thay đổi Todo.priority thì prop tự động cập nhật
 * - React.ChangeEvent<HTMLInputElement>: event từ input text
 * - React.ChangeEvent<HTMLSelectElement>: event từ select dropdown
 * - React.FormEvent<HTMLFormElement>: event từ form submit
 * - React.MouseEvent<HTMLButtonElement>: event từ button click
 */
function TodoList({
  title,
  initialTodos = [], // Default value, TS suy ra kiểu Todo[]
  onAddTodo,
  onDeleteTodo,
  onToggleComplete,
  allowEdit = true, // Default value, TS suy ra boolean
}: TodoListProps): JSX.Element {
  // --- State ---
  // useState với generic<Todo[]> vì initial value là mảng rỗng
  // TS không thể suy ra kiểu phần tử từ [] nên cần generic
  const [todos, setTodos] = useState<Todo[]>(initialTodos);

  // useState tự suy ra kiểu từ initial value string
  const [inputValue, setInputValue] = useState('');

  // useState với generic vì cần union type cụ thể
  const [selectedPriority, setSelectedPriority] =
    useState<Todo['priority']>('medium');

  // useState tự suy ra kiểu từ initial value
  const [filter, setFilter] = useState<FilterStatus>('all');
  const [sortBy, setSortBy] = useState<SortBy>('newest');

  // --- Computed values ---
  // Filter + sort todos — không cần state riêng, dùng useMemo nếu list lớn
  const filteredTodos = todos.filter((todo) => {
    if (filter === 'active') return !todo.completed;
    if (filter === 'completed') return todo.completed;
    return true; // 'all'
  });

  const sortedTodos = [...filteredTodos].sort((a, b) => {
    switch (sortBy) {
      case 'newest':
        return b.createdAt - a.createdAt;
      case 'priority': {
        // Map priority sang số để so sánh
        const priorityMap: Record<Todo['priority'], number> = {
          high: 3,
          medium: 2,
          low: 1,
        };
        return priorityMap[b.priority] - priorityMap[a.priority];
      }
      case 'alphabetical':
        return a.text.localeCompare(b.text);
      default:
        return 0;
    }
  });

  // --- Event Handlers ---

  /**
   * Xử lý thay đổi input text
   * Type: ChangeEvent<HTMLInputElement> — event từ input type="text"
   */
  const handleInputChange = (event: ChangeEvent<HTMLInputElement>) => {
    setInputValue(event.target.value); // event.target.value luôn là string
  };

  /**
   * Xử lý thay đổi priority select
   * Type: ChangeEvent<HTMLSelectElement> — event từ <select>
   */
  const handlePriorityChange = (event: ChangeEvent<HTMLSelectElement>) => {
    // Type assertion `as` cần thiết vì event.target.value là string
    // nhưng state cần kiểu Todo['priority']
    const priority = event.target.value as Todo['priority'];
    setSelectedPriority(priority);
  };

  /**
   * Xử lý thay đổi filter
   * Type: ChangeEvent<HTMLSelectElement>
   */
  const handleFilterChange = (event: ChangeEvent<HTMLSelectElement>) => {
    const newFilter = event.target.value as FilterStatus;
    setFilter(newFilter);
  };

  /**
   * Xử lý thay đổi sort
   * Type: ChangeEvent<HTMLSelectElement>
   */
  const handleSortChange = (event: ChangeEvent<HTMLSelectElement>) => {
    const newSort = event.target.value as SortBy;
    setSortBy(newSort);
  };

  /**
   * Xử lý submit form thêm todo
   * Type: FormEvent<HTMLFormElement> — event từ <form onSubmit>
   */
  const handleSubmit = (event: FormEvent<HTMLFormElement>) => {
    event.preventDefault(); // Ngăn reload trang

    const trimmedText = inputValue.trim();
    if (!trimmedText) return; // Không thêm todo rỗng

    const newTodo: Todo = {
      id: crypto.randomUUID(), // string — khớp với interface Todo
      text: trimmedText,
      completed: false,
      priority: selectedPriority,
      createdAt: Date.now(),
    };

    setTodos((prev) => [...prev, newTodo]);
    setInputValue(''); // Reset input
    setSelectedPriority('medium'); // Reset priority

    // Gọi callback nếu có
    onAddTodo?.(trimmedText, selectedPriority);
  };

  /**
   * Xử lý toggle hoàn thành todo
   * Type: MouseEvent<HTMLButtonElement> — event từ button click
   */
  const handleToggle = (id: string) => {
    setTodos((prev) =>
      prev.map((todo) =>
        todo.id === id ? { ...todo, completed: !todo.completed } : todo,
      ),
    );

    const todo = todos.find((t) => t.id === id);
    if (todo) {
      onToggleComplete?.(id, !todo.completed);
    }
  };

  /**
   * Xử lý xóa todo
   */
  const handleDelete = (id: string) => {
    setTodos((prev) => prev.filter((todo) => todo.id !== id));
    onDeleteTodo?.(id);
  };

  // --- Render helpers ---

  /** Trả về className dựa trên priority — dùng Record type */
  const getPriorityClass = (priority: Todo['priority']): string => {
    const priorityClasses: Record<Todo['priority'], string> = {
      high: 'priority-high',
      medium: 'priority-medium',
      low: 'priority-low',
    };
    return priorityClasses[priority];
  };

  /** Trả về label tiếng Việt cho priority */
  const getPriorityLabel = (priority: Todo['priority']): string => {
    const labels: Record<Todo['priority'], string> = {
      high: 'Cao',
      medium: 'Trung bình',
      low: 'Thấp',
    };
    return labels[priority];
  };

  // ============================================
  // RENDER
  // ============================================

  return (
    <div className='todo-list'>
      <h2>{title}</h2>

      {/* Thống kê */}
      <div className='stats'>
        <span>Tổng: {todos.length}</span>
        <span>Hoàn thành: {todos.filter((t) => t.completed).length}</span>
        <span>Đang làm: {todos.filter((t) => !t.completed).length}</span>
      </div>

      {/* Form thêm todo */}
      <form
        onSubmit={handleSubmit}
        className='add-form'
      >
        <input
          type='text'
          value={inputValue}
          onChange={handleInputChange}
          placeholder='Thêm công việc mới...'
          aria-label='Nội dung công việc'
        />
        <select
          value={selectedPriority}
          onChange={handlePriorityChange}
          aria-label='Mức độ ưu tiên'
        >
          <option value='low'>Thấp</option>
          <option value='medium'>Trung bình</option>
          <option value='high'>Cao</option>
        </select>
        <button
          type='submit'
          disabled={!inputValue.trim()}
        >
          Thêm
        </button>
      </form>

      {/* Controls: Filter + Sort */}
      <div className='controls'>
        <label>
          Lọc:
          <select
            value={filter}
            onChange={handleFilterChange}
          >
            <option value='all'>Tất cả</option>
            <option value='active'>Đang làm</option>
            <option value='completed'>Hoàn thành</option>
          </select>
        </label>

        <label>
          Sắp xếp:
          <select
            value={sortBy}
            onChange={handleSortChange}
          >
            <option value='newest'>Mới nhất</option>
            <option value='priority'>Ưu tiên</option>
            <option value='alphabetical'>A-Z</option>
          </select>
        </label>
      </div>

      {/* Danh sách todo */}
      <ul
        className='todo-items'
        role='list'
      >
        {sortedTodos.length === 0 ? (
          <li className='empty-state'>Không có công việc nào</li>
        ) : (
          sortedTodos.map((todo) => (
            <li
              key={todo.id}
              className={`todo-item ${todo.completed ? 'completed' : ''} ${getPriorityClass(todo.priority)}`}
            >
              <input
                type='checkbox'
                checked={todo.completed}
                onChange={() => handleToggle(todo.id)}
                aria-label={`Đánh dấu ${todo.text} là ${todo.completed ? 'chưa hoàn thành' : 'hoàn thành'}`}
              />

              <div className='todo-content'>
                <span className={todo.completed ? 'text-completed' : ''}>
                  {todo.text}
                </span>
                <span
                  className={`priority-badge ${getPriorityClass(todo.priority)}`}
                >
                  {getPriorityLabel(todo.priority)}
                </span>
              </div>

              {allowEdit && (
                <button
                  onClick={() => handleDelete(todo.id)}
                  aria-label={`Xóa ${todo.text}`}
                  className='delete-btn'
                >
                  🗑️
                </button>
              )}
            </li>
          ))
        )}
      </ul>
    </div>
  );
}
```

```tsx
import React, { useState, ChangeEvent } from 'react';

// ============================================
// FILE: types/components.ts — Shared Types
// ============================================

/** Props cơ bản mọi component nên có */
interface BaseProps {
  /** ID duy nhất cho element */
  id?: string;
  /** Class CSS tùy chỉnh */
  className?: string;
  /** Data attribute cho testing */
  'data-testid'?: string;
}

/** Props cơ bản cho mọi form input */
interface InputProps extends BaseProps {
  /** Label hiển thị */
  label: string;
  /** Giá trị hiện tại (controlled) */
  value: string;
  /** Callback khi giá trị thay đổi */
  onChange: (value: string) => void;
  /** Placeholder */
  placeholder?: string;
  /** Có bắt buộc không */
  required?: boolean;
  /** Có bị disabled không */
  disabled?: boolean;
  /** Error message */
  error?: string;
  /** Tên field (dùng cho form data) */
  name?: string;
}

/** Trạng thái async operation */
type LoadingState = 'idle' | 'loading' | 'success' | 'error';

/** Props cho pagination */
interface PaginationProps extends BaseProps {
  /** Trang hiện tại (1-based) */
  currentPage: number;
  /** Tổng số trang */
  totalPages: number;
  /** Callback khi đổi trang */
  onPageChange: (page: number) => void;
  /** Số nút hiển thị xung quanh trang hiện tại */
  siblingCount?: number;
}

// ============================================
// COMPONENT: TextInput
// ============================================

/**
 * TextInput — Input text cơ bản
 * Extend từ InputProps, không cần thêm gì
 */
function TextInput({
  id,
  label,
  value,
  onChange,
  placeholder,
  required,
  disabled,
  error,
  name,
  className = '',
  'data-testid': testId,
}: InputProps): JSX.Element {
  const handleChange = (event: ChangeEvent<HTMLInputElement>) => {
    onChange(event.target.value);
  };

  const inputId = id || `text-${name || label}`;
  const errorId = `${inputId}-error`;

  return (
    <div className={`form-group ${className}`}>
      <label htmlFor={inputId}>
        {label}
        {required && <span aria-label='bắt buộc'> *</span>}
      </label>
      <input
        id={inputId}
        type='text'
        name={name}
        value={value}
        onChange={handleChange}
        placeholder={placeholder}
        disabled={disabled}
        required={required}

// COMPONENT: Pagination (dùng PaginationProps)
// ============================================

function Pagination({
  currentPage,
  totalPages,
  onPageChange,
  siblingCount = 1,
  className = '',
  'data-testid': testId,
}: PaginationProps): JSX.Element {
  /** Tính range các trang hiển thị */
  const getPageRange = (): (number | string)[] => {
    const totalNumbers = siblingCount * 2 + 5;
    // first + last + current + 2*dots + 2*siblings = siblingCount * 2 + 5

    const totalBlocks = totalNumbers + 2;

    if (totalPages <= totalBlocks) {
      // Case 1: TotalPage: 5 , totalBlocks : 7
      // return [1] 2 3 4 5
      return Array.from({ length: totalPages }, (_, i) => i + 1);
    }

    const leftSiblingIndex = Math.max(currentPage - siblingCount, 1);
    const rightSiblingIndex = Math.min(currentPage + siblingCount, totalPages);

    const showLeftDots = leftSiblingIndex > 2;
    const showRightDots = rightSiblingIndex < totalPages - 1;

    if (!showLeftDots && showRightDots) {
      // Case 2: Total page: 6, current: 4, sibling: 1
      // Show left từ first đến trước right dots + right dots + last page

      const leftItemCount = 3 + 2 * siblingCount;

      // 1 2 3 [4] 5 [...] 6
      return [
        ...Array.from({ length: leftItemCount }, (_, i) => i + 1),
        '...',
        totalPages,
      ];
    }

    if (showLeftDots && !showRightDots) {
      // Case 3: Total page: 7, current: 6, sibling: 1
      // Show first + left dots + 2 sibling + current

      const rightItemCount = 3 + 2 * siblingCount;
      // 1 [...] 5 [6] 7
      return [
        1,
        '...',
        ...Array.from(
          { length: rightItemCount },
          (_, i) => totalPages - rightItemCount + i + 1, // Lấy N số cuối cùng = rightItemCount
        ),
      ];
    }
    // Case 4: current:6; totalPages = 20; siblingCount = 1
    // 1 [...] 5 [6] 7 [...] 20
    return [
      1,
      '...',
      ...Array.from(
        { length: rightSiblingIndex - leftSiblingIndex + 1 },
        (_, i) => leftSiblingIndex + i,
      ),
      '...',
      totalPages,
    ];
  };

  const pages = getPageRange();

  return (
    <nav
      className={`pagination ${className}`}
      aria-label='Pagination'
      data-testid={testId}
    >
      <button
        onClick={() => onPageChange(currentPage - 1)}
        disabled={currentPage === 1}
        aria-label='Trang trước'
      >
        ←
      </button>

      {pages.map((page, index) =>
        page === '...' ? (
          <span
            key={`dots-${index}`}
            className='dots'
          >
            …
          </span>
        ) : (
          <button
            key={page}
            onClick={() => onPageChange(page as number)}
            aria-current={currentPage === page ? 'page' : undefined}
            className={currentPage === page ? 'active' : ''}
          >
            {page}
          </button>
        ),
      )}

      <button
        onClick={() => onPageChange(currentPage + 1)}
        disabled={currentPage === totalPages}
        aria-label='Trang sau'
      >
        →
      </button>
    </nav>
  );
}
```

### Cân Nhắc Production

**1. Strict Mode là bắt buộc trong team:**

```json
// tsconfig.json

  "compilerOptions": {
```

```
  components/
    Button/
```

**3. Đừng over-engineer types:**

```tsx
// ❌ Over-engineered
type ButtonVariant = 'primary' | 'secondary';

type ButtonConfig = {
  variant: ButtonVariant;
  size: ButtonSize;
  state: ButtonState;
};

// ✅ Vừa đủ — Types phục vụ code, không phải ngược lại
interface ButtonProps {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
  disabled?: boolean;
}
```

### Câu Hỏi Phỏng Vấn

**Junior:**

> "Sự khác biệt giữa `interface` và `type` trong TypeScript là gì?"

_Expected answer:_ Interface hỗ trợ declaration merging và thường được dùng cho object shapes. Type alias linh hoạt hơn, hỗ trợ unions, intersections, tuples. Trong thực tế, cả hai đều dùng được cho props — quan trọng là nhất quán.

**Mid-level:**

> "Tại sao `React.FC` không được khuyến nghị trong React 18+? Bạn thay thế bằng gì?"

_Expected answer:_ `React.FC` tự động thêm `children?: ReactNode` vào mọi component (không cần thiết), ẩn return type (có thể return `null` mà TS không bắt), và gây khó debug. Thay thế: function declaration bình thường với explicit return type `JSX.Element` hoặc `ReactNode`.

**Senior:**

> "Hãy thiết kế type system cho một polymorphic (đa hình) component (component render element HTML khác nhau tùy theo prop, như `as` prop trong Chakra UI)."

_Expected answer:_ Đây là bài toán về generic components với discriminated unions và `React.ElementType`. Cần xử lý: props của element target phải merge với component props, `ref` forwarding, và TypeScript inference. (Đây chính là content của Ngày 59!)

> Discriminated Union (union phân biệt) là cách kết hợp nhiều object type, trong đó mỗi type có một “field chung” dùng để phân biệt chúng. Field đó gọi là discriminant (thường là `type`, `kind`, `status`, ...).

```tsx
import React from 'react';

type PolymorphicProps<E extends React.ElementType> = {
  as?: E;
  children: React.ReactNode;
} & React.ComponentPropsWithoutRef<E>;
// Option 1: “Lấy tất cả props đúng của element E, nhưng không lấy ref”

// Option 2: Nếu cần forward ref:
// type Props = React.ComponentPropsWithRef<"input">;

// const Input = React.forwardRef<HTMLInputElement, Props>(
//   (props, ref) => {
//     return <input {...props} ref={ref} />;
//   }
// );

// Dùng : <Input ref={ref} placeholder="Enter name" />
// Lưu ý: React.forwardRef() đã xử lý ref rồi
// Nên type KHÔNG bắt buộc dùng phải dùng WithRef.
// Dùng WithoutRef vẫn chạy đúng 100%

function Text<E extends React.ElementType = 'span'>({
  as,
  children,
  ...props
}: PolymorphicProps<E>) {
  const Component = as || 'span';

  return <Component {...props}>{children}</Component>;
}
// Cách dùng:
/*
 * 1. Render default (span)
 * <Text>Hello</Text>
 * HTML → <span>Hello</span>
 *
 * 2. Render thành button
 * <Text as="button" onClick={() => alert("click")}>
 *  Click me
 * </Text>
 * HTML →  <button>Click me</button>
 *
 * 3. Render thành thẻ <a>
 * <Text as="a" href="https://google.com">
 *  Google
 * </Text>
 * HTML →  <a href="https://google.com">Google</a>
 *
 *
 * Nếu viết <Text as="button" href="abc" />
 * → ❌ TypeScript báo lỗi (vì button không có href)
 */
```

### War Stories

**Câu chuyện thật từ production:**

_"Chúng tôi có component `<DataTable>` đã chạy production 6 tháng. Một ngày, một dev thêm column mới bằng cách truyền `columns={newColumns}` — nhưng `newColumns` có field `sortKey` là `number` trong khi codebase cũ expect `string`. Bug xuất hiện ở runtime khi user click sort._

_Sau sự cố, team quyết định migrate toàn bộ `<DataTable>` và các components liên quan sang TypeScript strict mode. Quá trình migration tìm ra thêm 12 type mismatch bugs tiềm ẩn khác — chưa có bug nào được report nhưng tất cả đều là time bomb._

_Lesson learned: TypeScript không chỉ là 'nice to have' cho developer experience. Nó là safety net ngăn những bug như thế này reach production."_
