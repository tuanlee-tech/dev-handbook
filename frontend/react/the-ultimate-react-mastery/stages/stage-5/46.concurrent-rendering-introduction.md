## 🤔 Kiểm tra đầu vào (5 phút)

**Câu 1:** State update trong React trigger re-render như thế nào? Có đồng bộ hay bất đồng bộ?

- _Đáp án: State update trong React được xử lý bất đồng bộ (asynchronous). Khi gọi `setState`, React không re-render ngay lập tức mà schedule một re-render trong tương lai, có thể batch nhiều updates thành một lần render duy nhất._

**Câu 2:** Khi bạn gọi `setState` nhiều lần liên tiếp, React xử lý như thế nào?

- _Đáp án: React sẽ batch (gom nhóm) các state updates lại và thực hiện re-render một lần duy nhất với state cuối cùng. React 17 chỉ batch trong event handlers, React 18 automatic batching hoạt động ở mọi nơi (setTimeout, promises, fetch)._

**Câu 3:** Trong app có danh sách 10,000 items, user typing vào search box bị lag. Tại sao và bạn xử lý thế nào?

- _Đáp án: Tại sao: React 17 render đồng bộ (synchronous), mỗi keystroke trigger re-render toàn bộ 10,000 items, block main thread làm input freeze. Xử lý: Dùng React 18 concurrent features (`useTransition`, `useDeferredValue`) để đánh dấu search results là non-urgent, dùng `useMemo` cache filtered results, tách input và list thành components riêng, hoặc dùng virtualization (`react-window`) chỉ render items visible._

Tưởng tượng bạn đang build một search interface:


**Vấn đề:** Mỗi keystroke → UI freezes vài trăm ms → Trải nghiệm tệ!

**Tại sao?** React 17 render **đồng bộ (synchronous)**:

1. User types "a"
2. React bắt đầu render từ đầu đến cuối
3. PHẢI hoàn thành 100% trước khi handle input tiếp theo
4. Nếu render takes 200ms → Input bị block 200ms

### 1.2 Giải Pháp: Concurrent Rendering

React 18 giới thiệu **Concurrent Rendering** - khả năng:

- **Interruptible rendering**: Tạm dừng render để xử lý urgent updates
- **Prioritized updates**: Updates có độ ưu tiên khác nhau
- **Automatic batching**: Batch nhiều state updates thành 1 render

**Mental Model:**

```
REACT 17 (Synchronous):
User Input → [==========RENDER==========] → UI Update
              (blocked, không thể interrupt)

REACT 18 (Concurrent):
User Input → [==RENDER==] ← New Input!
              (pause render, handle input first)
             → [==RENDER URGENT==] → UI Update (fast!)
             → [==RESUME RENDER==] → Background Update
```

### 1.3 Mental Model

Hãy nghĩ về Concurrent Rendering như **multitasking trên computer**:

**Synchronous (React 17):**

```
Task A [=============================] Task B
       Must finish A before start B
```

**Concurrent (React 18):**

```
Task A [====]     [====]     [====]
Task B      [====]     [====]     [====]
       Switch between tasks, prioritize urgent ones
```

**Analogy:**

- **Synchronous:** Như đọc sách từ đầu đến cuối, không được dừng
- **Concurrent:** Như đọc sách nhưng có thể bookmark, xử lý điện thoại, rồi quay lại đọc tiếp

### 1.4 Hiểu Lầm Phổ Biến

❌ **Hiểu lầm 1:** "Concurrent = Parallel (đa luồng)"

- ✅ **Sự thật:** Vẫn single-threaded! Chỉ là **time-slicing** thông minh

❌ **Hiểu lầm 2:** "React 18 tự động làm mọi thứ nhanh hơn"

- ✅ **Sự thật:** Cần **opt-in** vào concurrent features (useTransition, useDeferredValue)

❌ **Hiểu lầm 3:** "Phải rewrite toàn bộ app để dùng React 18"

- ✅ **Sự thật:** Backward compatible 100%! Chỉ cần upgrade version

❌ **Hiểu lầm 4:** "Automatic batching làm app chậm hơn"

- ✅ **Sự thật:** Ngược lại - ít render hơn = nhanh hơn!

### Demo 1: Automatic Batching - Before vs After ⭐

**React 17 (No Batching ngoài event handlers):**

```jsx
/**
 * Demo: React 17 batching behavior
 * Batching CHỈ hoạt động trong event handlers
 */
function Counter17() {
  const [count, setCount] = useState(0);
  const [flag, setFlag] = useState(false);

  console.log('Render'); // Log để đếm số lần render

  // ✅ Event handler: Batched (1 render)
  const handleClick = () => {
    setCount((c) => c + 1);
    setFlag((f) => !f);
    // React 17: Batches these → 1 render
  };

  // ❌ setTimeout: NOT batched (2 renders)
  const handleAsync = () => {
    setTimeout(() => {
      setCount((c) => c + 1); // Render 1
      setFlag((f) => !f); // Render 2
      // React 17: NO batching → 2 renders!
    }, 1000);
  };

  // ❌ fetch: NOT batched (2 renders)
  const handleFetch = () => {
    fetch('/api/data').then(() => {
      setCount((c) => c + 1); // Render 1
      setFlag((f) => !f); // Render 2
      // React 17: NO batching → 2 renders!
    });
  };

  return (
    <div>
      <p>
        Count: {count}, Flag: {flag.toString()}
      </p>
      <button onClick={handleClick}>Sync Update (1 render)</button>
      <button onClick={handleAsync}>Async Update (2 renders)</button>
      <button onClick={handleFetch}>Fetch Update (2 renders)</button>
    </div>
  );
}

// Console output khi click "Async Update":
// Render
// Render
// → 2 renders! Performance issue!
```

```jsx
  const handleClick = () => {
    setCount((c) => c + 1);
    setFlag((f) => !f);
    // React 18: Batched → 1 render
  };

  // ✅ setTimeout: Batched! (NEW in React 18)
  const handleAsync = () => {
    setTimeout(() => {
      setCount((c) => c + 1);
      setFlag((f) => !f);
      // React 18: Batched → 1 render ✨
    }, 1000);

    const data = await fetch('/api/data');

  return (
    <div>
      <p>
```

```jsx
import { flushSync } from 'react-dom';

function OptOut() {
```

```jsx
/**
 * Search Performance Comparison
 * Compares different filtering approaches
 */
function SearchPerformance() {
  const allItems = useMemo(
    () =>
      Array.from({ length: 10000 }, (_, i) => ({
        id: i,
        name: `Item ${i}`,
        category: ['Electronics', 'Clothing', 'Food'][i % 3],

  const [approach, setApproach] = useState('A');
  const [times, setTimes] = useState({ A: [], B: [], C: [] });

  const onRender = useCallback((id, phase, actualDuration) => {
    if (phase === 'update') {
      setTimes((prev) => ({
        ...prev,
        [id]: [...prev[id].slice(-4), actualDuration],
      }));
    }
  }, []);

    arr.length
      ? (arr.reduce((a, b) => a + b, 0) / arr.length).toFixed(2)

 * Approach A: Filter inline (WORST)
 * Re-filters on EVERY render, even unrelated updates

  // ❌ Filters on every render!
  const filtered = items.filter((item) =>
    item.name.toLowerCase().includes(query.toLowerCase()),

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder='Search...'
      />
      <button onClick={() => setCount((c) => c + 1)}>

      </button>
      <p>Found: {filtered.length} items</p>
    </div>
  );
}

/**
 * Approach B: Filter in handler (BETTER)
 * Only filters when query changes
 */
function ApproachB({ items }) {
  const [query, setQuery] = useState('');
  const [filtered, setFiltered] = useState(items);
  const [count, setCount] = useState(0);

  const handleSearch = (e) => {
    const value = e.target.value;
    setQuery(value);

    // Filter and update state
    const results = items.filter((item) =>
      item.name.toLowerCase().includes(value.toLowerCase()),
    );
    setFiltered(results);
  };

  return (
    <div>
      <input
        value={query}
        onChange={handleSearch}
        placeholder='Search...'
      />
      <button onClick={() => setCount((c) => c + 1)}>
        Unrelated update: {count}
      </button>
      <p>Found: {filtered.length} items</p>
    </div>
  );
}

 * Approach C: useMemo (BEST)
 * Only recalculates when dependencies change
 */
function ApproachC({ items }) {

  const [count, setCount] = useState(0);

  // ✅ Only recalculates when query or items change
  const filtered = useMemo(() => {
    console.log('Filtering...'); // Log để verify
    return items.filter((item) =>

    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder='Search...'
      />
      <button onClick={() => setCount((c) => c + 1)}>
        Unrelated update: {count}
      </button>
      <p>Found: {filtered.length} items</p>
    </div>
  );
```
```jsx
}

function calculateAverage(samples) {
  if (samples.length === 0) return 0;

  return samples.reduce((sum, value) => sum + value, 0) / samples.length;

   External Store

function createPerformanceStore() {
  const metricsMap = new Map();
  const samplesMap = new Map();
  const listeners = new Set();

  let version = 0;
  let cachedSnapshot = {
    version: 0,
    metricsMap: new Map(),
  };

  function emit() {
    version += 1;

    cachedSnapshot = {
      version,
      metricsMap: new Map(metricsMap),
    };

    listeners.forEach((listener) => listener());
  }

  function subscribe(listener) {
    listeners.add(listener);

    return () => {
      listeners.delete(listener);
    };
  }

  function getSnapshot() {
    return cachedSnapshot;
  }

  function getServerSnapshot() {
    return {
      version: 0,
      metricsMap: new Map(),
    };
  }

  function recordRender(componentName, payload, options = {}) {
    const {
      slowThreshold = DEFAULT_SLOW_THRESHOLD,
      maxSamples = DEFAULT_MAX_SAMPLES,
    } = options;

    const { phase, actualDuration } = payload;

    const renderTime = Number.isFinite(actualDuration) ? actualDuration : 0;

    const prevMetrics =
      metricsMap.get(componentName) ?? createInitialMetrics(componentName);

    const prevSamples = samplesMap.get(componentName) ?? [];
    const nextSamples = [...prevSamples, renderTime];

    if (nextSamples.length > maxSamples) {
      nextSamples.shift();
    }

    const isSlow = renderTime > slowThreshold;

    const nextMetrics = {
      componentName,
      renderCount: prevMetrics.renderCount + 1,
      lastRenderTime: renderTime,
      averageRenderTime: calculateAverage(nextSamples),
      maxRenderTime: Math.max(prevMetrics.maxRenderTime, renderTime),
      slowRenderCount: prevMetrics.slowRenderCount + (isSlow ? 1 : 0),
      isSlow,
      phase,
    };

    samplesMap.set(componentName, nextSamples);
    metricsMap.set(componentName, nextMetrics);

    emit();
  }

  function reset(componentName) {
    samplesMap.delete(componentName);
    metricsMap.set(componentName, createInitialMetrics(componentName));
    emit();
  }

  return {
    subscribe,
    getSnapshot,
    getServerSnapshot,
    recordRender,
    reset,
  };
}

  const {
    enabled = IS_DEV,
    slowThreshold = DEFAULT_SLOW_THRESHOLD,
    maxSamples = DEFAULT_MAX_SAMPLES,

    (id, phase, actualDuration, baseDuration, startTime, commitTime) => {

        componentName,

          id,
          phase,
          actualDuration,
          baseDuration,
          startTime,
          commitTime,
        },
        {
          slowThreshold,
          maxSamples,
```

```jsx
    const root = [];

    const nodeMap = new Map();

    for (const component of componentList) {
      const parts = component.path.split(' > ');
      let currentPath = '';

      parts.forEach((part, index) => {
        currentPath = index === 0 ? part : `${currentPath} > ${part}`;

        if (!nodeMap.has(currentPath)) {
          const node = {
            id: currentPath,
            name: part,
            path: currentPath,
            children: [],
            metrics: null,
          };

          nodeMap.set(currentPath, node);

          if (index === 0) {
            root.push(node);
          } else {
            const parentPath = parts.slice(0, index).join(' > ');
            const parent = nodeMap.get(parentPath);
            parent?.children.push(node);
          }

      }

    return root;
  }

  function rebuildSnapshot() {
    const componentList = Array.from(components.values()).sort(
      (a, b) => b.avgDuration - a.avgDuration,
    );

    const totalRenders = componentList.reduce(
      (sum, component) => sum + component.renderCount,
      0,
    );

    const slowRenderCount = componentList.reduce(
      (sum, component) => sum + component.slowRenderCount,
      0,
    );

    const totalDuration = componentList.reduce(
      (sum, component) => sum + component.totalDuration,
      0,
    );

    const maxRenderTime = componentList.reduce(
      (max, component) => Math.max(max, component.maxDuration),
      0,
    );

    const alerts = componentList
      .filter((component) => component.lastDuration > SLOW_RENDER_THRESHOLD)
      .slice(0, 10)
      .map((component) => ({
        id: `${component.path}-${component.updatedAt}`,
        path: component.path,
        duration: component.lastDuration,
        message: `${component.path} rendered slowly: ${component.lastDuration.toFixed(
          2,
        )}ms`,
        timestamp: component.updatedAt,
      }));

    cachedSnapshot = {
      version,
      components: componentList,
      hierarchy: buildHierarchy(componentList),
      alerts,
      timeline: [...dataPoints],
      summary: {
        componentCount: componentList.length,
        totalRenders,
        slowRenderCount,
        averageRenderTime:
          totalRenders === 0 ? 0 : totalDuration / totalRenders,
        maxRenderTime,
      },
    };
  }

  function emitNow() {
    version += 1;
    rebuildSnapshot();
    listeners.forEach((listener) => listener());
  }

  function scheduleEmit() {
    if (emitTimer !== null) return;

    emitTimer = window.setTimeout(() => {
      emitTimer = null;
      emitNow();
    }, STORE_DEBOUNCE_MS);
  }

  function subscribe(listener) {
    listeners.add(listener);

    return () => {
      listeners.delete(listener);
    };
  }

  function getSnapshot() {
    return cachedSnapshot;
  }

  function getServerSnapshot() {
    return {
      version: 0,
      components: [],
      hierarchy: [],
      alerts: [],
      timeline: [],
      summary: {
        componentCount: 0,
        totalRenders: 0,
        slowRenderCount: 0,
        averageRenderTime: 0,
        maxRenderTime: 0,
      },
    };
  }

  function recordRender({
    id,
    phase,
    actualDuration,
    baseDuration,
    startTime,
    commitTime,
  }) {
    try {
      const safeDuration = Number.isFinite(actualDuration) ? actualDuration : 0;

      const existing = components.get(id);

      const nextRenderCount = existing ? existing.renderCount + 1 : 1;
      const nextTotalDuration = (existing?.totalDuration ?? 0) + safeDuration;

      const nextSlowRenderCount =
        (existing?.slowRenderCount ?? 0) +
        (safeDuration > SLOW_RENDER_THRESHOLD ? 1 : 0);

      const samples = existing ? [...existing.samples] : [];

      samples.push({
        duration: safeDuration,
        phase,
        timestamp: Date.now(),
      });

      if (samples.length > 100) {
        samples.shift();
      }

      components.set(id, {
        id,
        name: id.split(' > ').at(-1) ?? id,
        path: id,
        phase,
        renderCount: nextRenderCount,
        totalDuration: nextTotalDuration,
        avgDuration: nextTotalDuration / nextRenderCount,
        lastDuration: safeDuration,
        maxDuration: Math.max(existing?.maxDuration ?? 0, safeDuration),
        baseDuration: Number.isFinite(baseDuration) ? baseDuration : 0,
        startTime: Number.isFinite(startTime) ? startTime : 0,
        commitTime: Number.isFinite(commitTime) ? commitTime : 0,
        slowRenderCount: nextSlowRenderCount,
        samples,
        updatedAt: Date.now(),
      });

      dataPoints.push({
        component: id,
        duration: safeDuration,
        phase,
        timestamp: Date.now(),
      });

      while (dataPoints.length > MAX_DATA_POINTS) {
        dataPoints.shift();
      }

      if (components.size > MAX_COMPONENTS) {
        const oldest = Array.from(components.values()).sort(
          (a, b) => a.updatedAt - b.updatedAt,
        )[0];

        if (oldest) {
          components.delete(oldest.id);
        }
      }

      scheduleEmit();
    } catch (error) {
      console.error('[PerformanceDashboard] Failed to record render:', error);
    }
  }

  function clear() {
    components.clear();
    dataPoints.length = 0;
    emitNow();
  }

  function exportCSV() {
    const rows = [
      [
        'Component',
        'Render Count',
        'Average Duration',
        'Last Duration',
        'Max Duration',
        'Slow Render Count',
        'Last Phase',
      ],
    ];

    for (const component of components.values()) {
      rows.push([
        component.path,
        component.renderCount,
        component.avgDuration.toFixed(2),
        component.lastDuration.toFixed(2),
        component.maxDuration.toFixed(2),
        component.slowRenderCount,
        component.phase,
      ]);
    }

    const csv = rows
      .map((row) =>
        row.map((cell) => `"${String(cell).replaceAll('"', '""')}"`).join(','),
      )
      .join('\n');

    downloadTextFile(`react-performance-${Date.now()}.csv`, csv, 'text/csv');
  }

  return {
    subscribe,
    getSnapshot,
    getServerSnapshot,
    recordRender,
    clear,
    exportCSV,
  };
}

const PerformanceStoreContext = createContext(null);
const PerformancePathContext = createContext('');

function usePerformanceStore() {
  const store = useContext(PerformanceStoreContext);

  if (!store) {
    throw new Error(
      'usePerformanceStore must be used inside <PerformanceProvider>',
    );
  }

  return store;
}

/* =========================================================
   Provider
========================================================= */

export function PerformanceProvider({ children, enabled = IS_DEV }) {
  // Dùng lazy useState để không bị : Cannot access refs during render
  const [store] = useState(() => createPerformanceStore());
  const [isDashboardOpen, setIsDashboardOpen] = useState(true);

  useEffect(() => {
    const handleKeyDown = (event) => {
      const isShortcut =
        (event.metaKey || event.ctrlKey) &&
        event.shiftKey &&
        event.key.toLowerCase() === 'p';

      if (!isShortcut) return;

      event.preventDefault();
      setIsDashboardOpen((prev) => !prev);
    };

    window.addEventListener('keydown', handleKeyDown);

    return () => {
      window.removeEventListener('keydown', handleKeyDown);
    };
  }, []);
  if (!enabled) {
    return <>{children}</>;
  }

  return (
    <PerformanceStoreContext.Provider value={store}>
      <PerformancePathContext.Provider value=''>
        {children}
      </PerformancePathContext.Provider>

      {isDashboardOpen && (
        <PerformanceDashboard onClose={() => setIsDashboardOpen(false)} />
      )}
    </PerformanceStoreContext.Provider>
  );
}

/* =========================================================
   usePerformanceMonitor Hook
========================================================= */

function usePerformanceMonitor() {
  const store = usePerformanceStore();

  const markRef = useRef(new Map());

  const mark = useCallback((name) => {
    markRef.current.set(name, safeNow());
  }, []);

  const measure = useCallback(
    (name) => {
      const start = markRef.current.get(name);

      if (start === undefined) {
        return null;
      }

      const duration = safeNow() - start;

      store.recordRender({
        id: `custom > ${name}`,
        phase: 'custom',
        actualDuration: duration,
        baseDuration: duration,
        startTime: start,
        commitTime: safeNow(),
      });

      markRef.current.delete(name);

      return duration;
    },
    [store],
  );

  const clearMetrics = useCallback(() => {
    store.clear();
  }, [store]);

  const exportMetrics = useCallback(() => {
    store.exportCSV();
  }, [store]);

  return {
    mark,
    measure,
    clearMetrics,
    exportMetrics,
  };
}

/* =========================================================
   PerformanceMonitor Component
========================================================= */

export function PerformanceMonitor({ name, children }) {
  const store = useContext(PerformanceStoreContext);
  const parentPath = useContext(PerformancePathContext);

  if (!store || !IS_DEV) {
    return <>{children}</>;
  }

  const fullPath = parentPath ? `${parentPath} > ${name}` : name;

  const handleRender = (
    id,
    phase,
    actualDuration,
    baseDuration,
    startTime,
    commitTime,
  ) => {
    store.recordRender({
      id,
      phase,
      actualDuration,
      baseDuration,
      startTime,
      commitTime,
    });
  };

  return (
    <PerformancePathContext.Provider value={fullPath}>
      <Profiler
        id={fullPath}
        onRender={handleRender}
      >
        {children}
      </Profiler>
    </PerformancePathContext.Provider>
  );
}

function PerformanceDashboard({ onClose }) {
  const store = usePerformanceStore();

  const snapshot = useSyncExternalStore(
    store.subscribe,
    store.getSnapshot,
    store.getServerSnapshot,
  );

  const [initialDashboardState] = useState(readDashboardState);

  const [position, setPosition] = useState(
    initialDashboardState.position ?? DEFAULT_DASHBOARD_POSITION,
  );

  const [activeTab, setActiveTab] = useState(
    initialDashboardState.tab ?? DEFAULT_DASHBOARD_TAB,
  );

  const [isDragging, setIsDragging] = useState(false);
  const dragStartRef = useRef({ x: 0, y: 0 });

  useEffect(() => {
    safeLocalStorageSet(
      DASHBOARD_STORAGE_KEY,
      JSON.stringify({
        position,
        tab: activeTab,
      }),
    );
  }, [position, activeTab]);

  const handleMouseDown = useCallback(
    (event) => {
      const header = event.target.closest('[data-dashboard-header]');

      if (!header) return;

      setIsDragging(true);

      dragStartRef.current = {
        x: event.clientX - position.x,
        y: event.clientY - position.y,
      };
    },
    [position.x, position.y],
  );

  useEffect(() => {
    if (!isDragging) return;

    const handleMouseMove = (event) => {
      setPosition({
        x: event.clientX - dragStartRef.current.x,
        y: event.clientY - dragStartRef.current.y,
      });
    };

    const handleMouseUp = () => {
      setIsDragging(false);
    };

    window.addEventListener('mousemove', handleMouseMove);
    window.addEventListener('mouseup', handleMouseUp);

    return () => {
      window.removeEventListener('mousemove', handleMouseMove);
      window.removeEventListener('mouseup', handleMouseUp);
    };
  }, [isDragging]);

  return (
    <div
      style={{
        position: 'fixed',
        left: position.x,
        top: position.y,
        width: 760,
        maxHeight: '85vh',
        background: '#ffffff',
        color: '#111827',
        border: '1px solid #d1d5db',
        borderRadius: 10,
        boxShadow: '0 10px 35px rgba(0,0,0,0.25)',
        zIndex: 999999,
        overflow: 'hidden',
        fontFamily:
          'ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace',
        fontSize: 12,
      }}
      onMouseDown={handleMouseDown}
    >
      <DashboardHeader
        onClose={onClose}
        isDragging={isDragging}
      />

      <DashboardTabs
        activeTab={activeTab}
        onChange={setActiveTab}
      />

      <div style={{ padding: 12, overflowY: 'auto', maxHeight: '65vh' }}>
        {activeTab === 'overview' && (
          <OverviewPanel
            snapshot={snapshot}
            store={store}
          />
        )}

        {activeTab === 'components' && (
          <ComponentsPanel components={snapshot.components} />
        )}

        {activeTab === 'hierarchy' && (
          <HierarchyPanel hierarchy={snapshot.hierarchy} />
        )}

        {activeTab === 'timeline' && (
          <TimelinePanel timeline={snapshot.timeline} />
        )}

        {activeTab === 'alerts' && <AlertsPanel alerts={snapshot.alerts} />}
      </div>

      <div
        style={{
          padding: 8,
          borderTop: '1px solid #e5e7eb',
          background: '#f9fafb',
          color: '#6b7280',
          textAlign: 'center',
          fontSize: 11,
        }}
      >
        Cmd/Ctrl + Shift + P to toggle • Drag header to move
      </div>
    </div>
  );
}

function MetricsChart({ data }) {
  if (data.length === 0) {
    return null;
  }

  const width = 700;
  const height = 140;
  const max = Math.max(...data.map((point) => point.duration), 1);

  const bars = data.map((point, index) => {
    const barWidth = width / data.length;
    const barHeight = Math.max((point.duration / max) * height, 1);

    return {
      x: index * barWidth,
      y: height - barHeight,
      width: Math.max(barWidth - 1, 1),
      height: barHeight,
      slow: point.duration > SLOW_RENDER_THRESHOLD,
    };
  });

  return (
    <svg
      width='100%'
      viewBox={`0 0 ${width} ${height}`}
      style={{
        border: '1px solid #e5e7eb',
        borderRadius: 8,
        background: '#ffffff',
      }}
    >
      {bars.map((bar, index) => (
        <rect
          key={index}
          x={bar.x}
          y={bar.y}
          width={bar.width}
          height={bar.height}
          fill={bar.slow ? '#ef4444' : '#10b981'}
        />
      ))}
    </svg>
  );
}
```

### Bảng So Sánh: React 17 vs React 18

| Feature       | React 17                  | React 18                    | Impact                       |
| ------------- | ------------------------- | --------------------------- | ---------------------------- |
| **Batching**  | Chỉ trong event handlers  | Mọi nơi (automatic)         | ⬆️ Performance               |
| **Rendering** | Synchronous               | Concurrent (opt-in)         | ⬆️ Responsiveness            |
| **APIs**      | Legacy                    | Modern (useTransition, etc) | ⬆️ Developer Experience      |
| **SSR**       | Client hydration blocking | Streaming SSR               | ⬆️ TTI (Time to Interactive) |
| **Suspense**  | Limited                   | Full support                | ⬆️ Loading UX                |

### Trade-offs Matrix

| Approach     | Pros                                                                                            | Cons                                                                         | When to Use                                                |
| ------------ | ----------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **React 17** | ✅ Stable<br>✅ Well-tested<br>✅ Simple model                                                  | ❌ No automatic batching<br>❌ Blocking renders<br>❌ No concurrent features | Legacy apps<br>Simple UIs<br>Low interactivity             |
| **React 18** | ✅ Better performance<br>✅ Concurrent rendering<br>✅ Automatic batching<br>✅ Modern features | ❌ Learning curve<br>❌ Migration effort<br>❌ Some breaking changes         | New projects<br>High interactivity<br>Performance-critical |

### Decision Tree: Upgrade to React 18?

```
START: Should I upgrade to React 18?
│
├─ App cần Concurrent features? (useTransition, Suspense)
│  ├─ YES → ✅ Upgrade
│  └─ NO → Continue
│
├─ Performance issues với frequent state updates?
│  ├─ YES → ✅ Upgrade (automatic batching helps)
│  └─ NO → Continue
│
├─ Using Suspense for data fetching?
│  ├─ YES → ✅ Upgrade (better support)
│  └─ NO → Continue
│
├─ App is stable, no issues?
│  ├─ YES → ⚠️ Optional (low risk, minor benefits)
│  └─ NO → Continue
│
└─ Legacy codebase, high risk?
   └─ NO → ⚠️ Test thoroughly before upgrade
```

### Bug 2: Infinite Renders sau Upgrade

```jsx
/**
 * ❌ BUG: Component re-render vô hạn sau upgrade React 18
 *
 * Symptom: Browser freeze, console đầy "Render"
 * Root cause: useEffect dependency issue amplified bởi automatic batching
 */
function BuggyComponent() {
  const [data, setData] = useState([]);
  const [filter, setFilter] = useState('all');

  console.log('Render');

  useEffect(() => {
    // ❌ Creates new array reference every time
    const filtered = data.filter(
      (item) => filter === 'all' || item.type === filter,
    );
    setData(filtered); // This triggers re-render!
  }, [data, filter]); // data changes → effect runs → setData → data changes...

  return <div>{data.length} items</div>;
}

// ❓ CÂU HỎI:
// 1. Tại sao infinite loop xảy ra?
// 2. React 17 có bug này không? Tại sao React 18 amplify?
// 3. Fix như thế nào?
```

**💡 Giải thích:**

1. **Tại sao infinite loop:**
   - `useEffect` depends on `data`
   - Effect runs → `setData(filtered)` → `data` changes
   - `data` changes → effect runs again → infinite loop!

2. **React 17 vs 18:**
   - React 17: Bug vẫn có nhưng có thể ít obvious hơn
   - React 18: Automatic batching làm loop nhanh hơn → dễ phát hiện

3. **Fix:**

   ```jsx
   // ✅ Fix 1: Remove data từ dependencies
   useEffect(() => {
     setData((prev) =>
       prev.filter((item) => filter === 'all' || item.type === filter),
     );
   }, [filter]); // Chỉ depend on filter

   // ✅ Fix 2: useMemo thay vì useEffect
   const filteredData = useMemo(
     () =>
       originalData.filter((item) => filter === 'all' || item.type === filter),
     [filter],
   );

   // ✅ Fix 3: Separate filtered state
   const [rawData, setRawData] = useState([]);
   const [filteredData, setFilteredData] = useState([]);

   useEffect(() => {
     setFilteredData(
       rawData.filter((item) => filter === 'all' || item.type === filter),
     );
   }, [rawData, filter]); // Safe - rawData không update trong effect
   ```

### Bug 3: flushSync Misuse

```jsx
/**
 * ❌ BUG: Overusing flushSync causing performance regression
 *
 * Symptom: React 18 chậm hơn React 17!
 * Root cause: Opt-out khỏi batching unnecessarily
 */
import { flushSync } from 'react-dom';

function OverFlushSync() {
  const [items, setItems] = useState([]);
  const [loading, setLoading] = useState(false);

  const handleLoad = async () => {
    // ❌ Unnecessary flushSync
    flushSync(() => {
      setLoading(true);
    });

    const data = await fetchData();

    // ❌ Another unnecessary flushSync

      setItems(data);
    });

    // ❌ And another!
    flushSync(() => {
      setLoading(false);
    });
  };

  return (

// 1. Tại sao code này slow?
// 2. Khi nào NÊN dùng flushSync?
// 3. Fix như thế nào?
```

**💡 Giải thích:**

1. **Tại sao slow:**

   - Mất đi benefit của automatic batching!

2. **Khi NÊN dùng flushSync:**

   ```jsx
   // ✅ Use case 1: Cần DOM measurement ngay
   flushSync(() => {

   scrollToPosition(savedPosition);
   ```

## 💡 SENIOR INSIGHTS

### Cân Nhắc Production

**1. Migration Strategy:**

```
PHASED APPROACH:
Week 1: Upgrade dependencies, test builds
Week 2: Test critical paths, fix breaking changes
Week 3: Performance monitoring
Week 4: Gradual rollout (10% → 50% → 100% users)
```

**2. Monitoring:**

- Track render times before/after upgrade bằng React DevTools Profiler.
- Watch for performance regressions
- Monitor error rates : qua Sentry/LogRocket
- Set up alerts cho slow renders (>100ms). Theo dõi memory usage và user feedback về lag/freeze.

**3. Rollback Plan:**

- Keep React 17 build ready
- Feature flags để toggle concurrent features
- Incremental adoption strategy
- Rollback khi: checkout lỗi, app đơ trên mobile, error rate tăng gấp 3, khách hàng phàn nàn liên tục. Cách rollback: revert commit → deploy bản cũ → clear CDN cache → investigate lỗi trên branch R18.

### Câu Hỏi Phỏng Vấn

**Junior Level:**

- Q: React 18 có gì mới so với React 17?
- A: Automatic batching everywhere, concurrent rendering opt-in, new APIs (useTransition, useDeferredValue), Suspense improvements

**Mid Level:**

- Q: Giải thích Automatic Batching và tại sao nó improve performance?
- A: React 18 batches multiple state updates thành 1 render, ngay cả trong async callbacks (setTimeout, promises). Ít renders hơn = better performance. React 17 chỉ batch trong event handlers.

**Senior Level:**

- Q: Khi nào nên migrate lên React 18? Trade-offs là gì?
- A:
  - Migrate khi: Cần concurrent features, performance issues với frequent updates, modern SSR
  - Trade-offs: Learning curve, migration effort, potential breaking changes
  - Strategy: Test thoroughly, gradual rollout, monitor metrics, rollback plan ready

**Architect Level:**

- Q: Design strategy để migrate large legacy app lên React 18 without downtime
- A:
  1. Audit codebase (breaking changes, third-party deps)
  2. Phased migration (dev → staging → prod)
  3. Feature flags để toggle concurrent features
  4. Comprehensive testing (unit, integration, E2E, perf)
  5. Monitoring setup (render times, error rates)
  6. Gradual rollout (percentage-based)
  7. Rollback plan (React 17 build ready)
  8. Post-migration optimization (adopt new patterns)

### War Stories

**Story 1: "Automatic Batching Surprise"**

```
Scenario: Upgraded to React 18, performance đột nhiên WORSE!

Root cause: Code rely on synchronous renders để trigger effects
useEffect(() => {
  // Expects setA to trigger render before setB
}, [a]);

setA(1);  // React 17: Render now
setB(2);  // React 17: Render now

setA(1);  // React 18: Batched
setB(2);  // React 18: Batched → 1 render → effect timing changed!

Fix: Review useEffect dependencies, use flushSync nếu cần sync behavior
```

**Story 2: "The flushSync Trap"**

```
Scenario: Developer đọc về flushSync, nghĩ nó "optimize" performance

Reality: Wrapped mọi setState trong flushSync
→ Opt-out khỏi batching completely
→ React 18 chậm hơn React 17!

Lesson: flushSync là escape hatch cho edge cases, không phải default
```

**Story 3: "Profiler in Production"**

```
Mistake: Ship Profiler component to production

Impact:
- Bundle size increase
- Runtime overhead (callbacks firing constantly)
- Memory leaks (storing metrics)

Fix:
if (process.env.NODE_ENV === 'development') {
  return <Profiler onRender={callback}>{children}</Profiler>;
}
return children;
```
