## 🤔 Kiểm tra đầu vào (5 phút)

1. **React Hook Form Basics** (Ngày 41): `useForm`, `register`, `handleSubmit` hoạt động như thế nào?
   - _Đáp án: `useForm()` khởi tạo form instance và quản lý state nội bộ. `register(name)` trả về `ref`, `onChange`, `onBlur` để kết nối input với RHF mà không cần controlled component. `handleSubmit(onValid, onInvalid)` chặn submit mặc định, chạy validate, rồi gọi callback với dữ liệu sạch._

2. **Context API** (Ngày 36-38): Làm sao share state giữa components?
   - _Đáp án: `createContext()` tạo context object → `Provider` ở component cha truyền value xuống cây component → `useContext()` ở component con/cháu bất kỳ để đọc value. Tránh prop drilling qua nhiều cấp, nhưng chỉ dùng cho data thực sự "global" trong nhánh đó._

3. **Performance** (Ngày 32-34): `useMemo`, `useCallback`, `React.memo` dùng để làm gì?
   - _Đáp án: `useMemo` cache kết quả tính toán đắt đỏ giữa các render. `useCallback` cache function reference để không tạo hàm mới mỗi lần render. `React.memo` bọc component để bỏ qua re-render nếu props không đổi. Cả ba dùng để tối ưu re-render, nhưng không dùng bừa bãi vì có overhead so sánh._

### 1.2 Core Advanced APIs

#### 1. useFieldArray

**Purpose:** Quản lý array of fields (add/remove/reorder)

```jsx
import { useFieldArray } from 'react-hook-form';

const { fields, append, remove, insert, update, move } = useFieldArray({
  control, // from useForm()
  name: 'items', // array field name
});
```

**Key Features:**

- Dynamic add/remove items
- Stable unique IDs (field.id)
- Automatic re-indexing
- Integrated validation

#### 2. useFormContext

**Purpose:** Access form state từ deeply nested components

```jsx
// Parent
<FormProvider {...methods}>
  <form>
    <NestedComponent /> {/* Can access form */}
  </form>
</FormProvider>;

// Child (anywhere in tree)
const { register, formState } = useFormContext();
```

**Benefits:**

- No prop drilling
- Clean component APIs
- Easier composition

#### 3. useWatch

**Purpose:** Optimized field subscription (isolated re-render)

```jsx
// watch()
// Component chứa watch() sẽ re-render khi field change
const value = watch('fieldName');

// useWatch()
// Chỉ component subscribe bị re-render (field-level subscription)
const value = useWatch({
  control,
  name: 'fieldName',
});
```

---

##### 📌 Cách dùng & nơi dùng

```txt
watch()
- Dùng trong: Parent component / form nhỏ
- Khi: cần đọc value trực tiếp trong render
- Không ưu tiên tối ưu performance
```

```txt
useWatch()
- Dùng trong: Child component / component tách riêng
- Khi: form lớn, nested UI, dynamic fields
- Khi: cần tránh re-render parent
- Khi: cần isolate subscription theo field
```

---

##### ⚙️ Behavior

```txt
watch():
Component chứa watch()
    ↓ re-render khi field change
Children của component đó
    ↓ re-render theo React reconciliation
```

```txt
useWatch():
Component chứa useWatch()
    ↓ isolated re-render theo field subscription

Parent không bị ảnh hưởng
Sibling components không bị ảnh hưởng
```

---

##### 🧠 Mental Model

```txt
watch()
= component-level subscription (gắn vào render cycle)

useWatch()
= field-level subscription (observer độc lập theo field)
```

### 1.4 Hiểu Lầm Phổ Biến

❌ **"useFieldArray chỉ dùng cho simple arrays"**
→ Sai! Có thể nest arrays, objects phức tạp, conditional fields.

❌ **"useFormContext thay thế props drilling trong mọi trường hợp"**
→ Không! Chỉ dùng cho form-related data. Other props vẫn drill normally.

❌ **"useWatch và watch() giống nhau"**
→ Sai! useWatch tối ưu hơn, không re-render parent component.

❌ **"Field array index là stable ID"**
→ SAI NGHIÊM TRỌNG! Luôn dùng `field.id`, không dùng array index làm key.

```jsx
/**
 * Checkout Form - useFormContext Demo
 *
 * Shows how to:
 * - Share form across multiple components
 * - Avoid prop drilling
 * - Keep components clean and focused
 */

import { useForm, FormProvider, useFormContext } from 'react-hook-form';

// ============================================
// MAIN FORM COMPONENT (Parent)
// ============================================
function CheckoutForm() {
  const methods = useForm({
    mode: 'onBlur',
    defaultValues: {
      // Personal Info
      firstName: '',
      lastName: '',
      email: '',

      // Shipping
      address: '',
      city: '',
      zipCode: '',

      // Payment
      cardNumber: '',
      expiryDate: '',
      cvv: '',
    },
  });

  const onSubmit = (data) => {
    console.log('Checkout data:', data);
    alert('Order placed successfully!');
  };

  return (
    <FormProvider {...methods}>
      {/* FormProvider wraps form - children can access context */}
      <form onSubmit={methods.handleSubmit(onSubmit)}>
        <div style={{ maxWidth: '800px', padding: '20px' }}>
          <h1>Checkout</h1>

          {/* Nested components - NO PROPS NEEDED! */}
          <PersonalInfoSection />
          <ShippingSection />
          <PaymentSection />

          <SubmitButton />
        </div>
      </form>
    </FormProvider>
  );
}

// REUSABLE FORM FIELD COMPONENT
// ============================================
function FormField({ label, name, type = 'text', validation, placeholder }) {
  const {
    register,
    formState: { errors },
  } = useFormContext(); // ✅ Access form context

  return (
    <div style={{ marginBottom: '16px' }}>
      <label
        style={{ display: 'block', marginBottom: '4px', fontWeight: 'bold' }}

        type={type}
        {...register(name, validation)}

}
```

```jsx
 * ======================================================
 *
 * watch():
 *
 *   subscription nằm ở parent
 *
 *   field change
 *      -> parent render
 *          -> child render
 *
 *

 *

 *      -> child render

 *
 *

 *   ✅ performance gần giống useWatch()
 *
 *
 * ======================================================
 * BEST PRACTICE
 * ======================================================
 *
 * Form nhỏ:
 *   ✅ watch()
 *
 * Form lớn / nested:
 *   ✅ useWatch()
```

```jsx
function SurveyForm() {
  const methods = useForm({
    mode: 'onBlur',
    defaultValues: {
      // Demographics
      age: '',
      gender: '',
      occupation: '',

      // Preferences
      favoriteColor: '',
      hobbies: [],
      newsletter: '',

      // Feedback
      rating: '',
      comments: '',
```

```jsx
function calculateGPA(courses) {
  if (!courses || courses.length === 0) return 0;

  const gradePoints = {
    A: 4.0,
    B: 3.0,
    C: 2.0,
    D: 1.0,
    F: 0.0,
  };

  let totalPoints = 0;
  let totalCredits = 0;

  courses.forEach((course) => {
    const credits = Number(course.credits) || 0;
    const points = gradePoints[course.grade] || 0;

    totalPoints += points * credits;
    totalCredits += credits;
  });

  return totalCredits > 0 ? totalPoints / totalCredits : 0;
}
```

```jsx
// Auto-save hook
function useAutoSave(formValues, delay = 30000) {
  const [lastSaved, setLastSaved] = useState(null);
  const [isSaving, setIsSaving] = useState(false);

  useEffect(() => {
    setIsSaving(true);
    const timeoutId = setTimeout(() => {
      try {
        localStorage.setItem(DRAFT_KEY, JSON.stringify(formValues));
        setLastSaved(new Date());
        setIsSaving(false);
      } catch (error) {
        console.error('Auto-save failed:', error);
        setIsSaving(false);
      }
    }, 1000); // Debounce 1s

    return () => clearTimeout(timeoutId);
  }, [formValues, delay]);

  return { lastSaved, isSaving };
}

    localStorage.removeItem(DRAFT_KEY);
    methods.reset(getDefaultValues());
  };

  const clearDraft = () => {
    if (window.confirm('Clear draft and start over?')) {
      localStorage.removeItem(DRAFT_KEY);
      methods.reset(getDefaultValues());
    }
  };

  const duplicateJob = () => {
    const currentValues = methods.getValues();
    methods.reset({
      ...currentValues,
      title: `${currentValues.title} (Copy)`,
      // Reset arrays to avoid reference issues
      skills: [...currentValues.skills],
      questions: [...currentValues.questions],
    });
  };

  if (showPreview) {
    return (
      <JobPreview
        data={formValues}
        onBack={() => setShowPreview(false)}
      />
    );
  }

  return (
    <FormProvider {...methods}>
      <form onSubmit={methods.handleSubmit(onSubmit)}>
        <div style={{ maxWidth: '1000px', margin: '0 auto', padding: '20px' }}>
          {/* Header */}
          <div
            style={{
              display: 'flex',
              justifyContent: 'space-between',

              marginBottom: '24px',
            }}

                  {isSaving
                    ? '💾 Saving...'
                    : `✓ Saved ${formatTime(lastSaved)}`}
                </span>
              )}
              <button
                type='button'
                onClick={() => setShowPreview(true)}
                style={{
                  padding: '8px 16px',
                  backgroundColor: '#2196f3',
                  color: 'white',
                  border: 'none',
                  borderRadius: '4px',
                  cursor: 'pointer',
                }}
              >
                👁️ Preview
              </button>
              <button
                type='button'
                onClick={duplicateJob}
                style={{
                  padding: '8px 16px',
                  backgroundColor: '#9c27b0',
                  color: 'white',
                  border: 'none',
                  borderRadius: '4px',
                  cursor: 'pointer',
                }}
              >
                📋 Duplicate
              </button>
              <button
                type='button'
                onClick={clearDraft}
                style={{
                  padding: '8px 16px',
                  backgroundColor: '#f44336',
                  color: 'white',
                  border: 'none',
                  borderRadius: '4px',
                  cursor: 'pointer',
                }}
              >
                🗑️ Clear
              </button>
            </div>
          </div>

          {/* Sections */}
          <BasicInfoSection />
          <JobDescriptionSection />
          <RequirementsSection />
          <SalaryBenefitsSection />
          <ScreeningQuestionsSection />

          {/* Submit */}
          <div
            style={{
              display: 'flex',
              gap: '12px',
              marginTop: '32px',
            }}
          >
            <button
              type='submit'
              disabled={
                methods.formState.isSubmitting || !methods.formState.isValid
              }
              style={{
                flex: 1,
                padding: '16px',
                backgroundColor:
                  methods.formState.isSubmitting || !methods.formState.isValid
                    ? '#ccc'
                    : '#4caf50',
                color: 'white',
                border: 'none',
                borderRadius: '4px',
                fontSize: '18px',
                fontWeight: 'bold',
                cursor:
                  methods.formState.isSubmitting || !methods.formState.isValid
                    ? 'not-allowed'
                    : 'pointer',
              }}
            >
              {methods.formState.isSubmitting ? 'Posting...' : 'Post Job'}

          </div>
        </div>

function formatTime(date) {
  const now = new Date();
  const diff = Math.floor((now - date) / 1000); // seconds

  if (diff < 60) return 'just now';
  if (diff < 3600) return `${Math.floor(diff / 60)}m ago`;
  if (diff < 86400) return `${Math.floor(diff / 3600)}h ago`;
  return date.toLocaleDateString();
```

```jsx
    const newField = {
      id: `field_${Date.now()}`,
      type: 'text',
      label: 'New Field',
      required: false,
    };

    updateConfig({
      fields: [...config.fields, newField],
    });
  };

  const removeField = (index) => {
    updateConfig({
      fields: config.fields.filter((_, i) => i !== index),
    });
  };

  const updateField = (index, updates) => {
    const newFields = [...config.fields];
    newFields[index] = { ...newFields[index], ...updates };
    updateConfig({ fields: newFields });

    const newFields = [...config.fields];
    const targetIndex = direction === 'up' ? index - 1 : index + 1;

    if (targetIndex < 0 || targetIndex >= newFields.length) return;

      newFields[targetIndex],
      newFields[index],
    ];

function evaluateCondition(value, condition) {
  switch (condition.operator) {
    case 'equals':
      return value === condition.value;
    case 'not_equals':
      return value !== condition.value;
    case 'in':
      return Array.isArray(condition.value) && condition.value.includes(value);
    case 'not_in':
      return Array.isArray(condition.value) && !condition.value.includes(value);
    default:
      return true;
  }
}
```

## 📊 PHẦN 4: SO SÁNH PATTERNS (30 phút)

### Bảng So Sánh: Advanced RHF Patterns

| **Pattern**        | **Use Case**                             | **Pros**                                                 | **Cons**                                      | **When to Use**               |
| ------------------ | ---------------------------------------- | -------------------------------------------------------- | --------------------------------------------- | ----------------------------- |
| **useFieldArray**  | Dynamic lists (cart items, skills, etc.) | ✅ Built-in re-indexing<br>✅ Stable IDs<br>✅ Easy CRUD | ❌ Learning curve<br>❌ Nested complexity     | Lists with add/remove         |
| **useFormContext** | Nested form sections                     | ✅ No prop drilling<br>✅ Clean code<br>✅ Easy refactor | ❌ Hidden dependencies<br>❌ Debugging harder | Large multi-section forms     |
| **useWatch**       | Optimized watching                       | ✅ Performance<br>✅ Isolated re-renders                 | ❌ More verbose<br>❌ Need control prop       | Performance-critical watching |
| **watch()**        | Simple watching                          | ✅ Simple API<br>✅ Quick to use                         | ❌ Re-renders parent<br>❌ Performance issues | Small forms, simple cases     |
| **Custom Hooks**   | Reusable logic                           | ✅ DRY<br>✅ Testable<br>✅ Composable                   | ❌ Initial setup<br>❌ Abstraction overhead   | Common patterns repeated      |

### Decision Tree: Which Pattern?

```
Need dynamic array of fields?
├─ YES → useFieldArray
└─ NO ↓

Form has nested components?
├─ YES → useFormContext (wrap with FormProvider)
└─ NO ↓

Need to watch field values?
├─ YES ↓
│   └─ Performance critical? (large form, many watchers)
│       ├─ YES → useWatch
│       └─ NO → watch()
└─ NO ↓

Have repetitive form logic?
├─ YES → Custom Hooks
└─ NO → Use basic RHF (register, handleSubmit)
```

#### `watch(callback)` và re-render behavior

Ngoài việc dùng `watch()` để render UI reactive, `react-hook-form` còn hỗ trợ subscription callback:

```tsx
watch((value, info) => {
  console.log(value);
  console.log(info.name);
});
```

Pattern này thường được dùng cho:

- Analytics / tracking
- Autosave form
- Sync với external state
- Logging
- Side effects toàn form

---

#### Khác biệt quan trọng

#### `watch('field')`

```tsx
const email = watch('email');
```

- Component sẽ re-render khi `email` thay đổi
- Phù hợp để render UI reactive

---

#### `watch(callback)`

```tsx
watch((value) => {
  console.log(value);
});
```

- Callback sẽ chạy khi form thay đổi
- **Không trigger React re-render**
- Hoạt động giống một subscription/event listener

Điều này giúp xử lý side effects mà không làm UI render liên tục khi user typing.

---

#### Khi nào nên dùng subscription callback?

#### Analytics / tracking

```tsx
useEffect(() => {
  const subscription = watch((value, { name, type }) => {
    console.log('Tracking:', {
      field: name,
      eventType: type,
      value: value[name as keyof typeof value],
    });

    // analytics.track(...)
  });

  return () => subscription.unsubscribe();
}, [watch]);
```

---

#### Autosave form

```tsx
useEffect(() => {
  const autosave = debounce((value) => {
    console.log('Saving draft...', value);

    // await api.saveDraft(value)
  }, 1000);

  const subscription = watch((value) => {
    autosave(value);
  });

  return () => {
    subscription.unsubscribe();
    autosave.cancel();
  };
}, [watch]);
```

---

#### Sync với external state

```tsx
// Sync với Zustand
const setDraft = useDraftStore((state) => state.setDraft);
useEffect(() => {
  const subscription = watch((value) => {
    setDraft(value);
  });

  return () => subscription.unsubscribe();
}, [watch, setDraft]);

// Sync với localStorage
useEffect(() => {
  const subscription = watch((value) => {
    localStorage.setItem('draft-form', JSON.stringify(value));
  });

  return () => subscription.unsubscribe();
}, [watch]);
```

---

#### Lưu ý quan trọng

`watch(callback)` sẽ không re-render component **trừ khi bạn update React state bên trong callback**.

Ví dụ:

```tsx
watch((value) => {
  setPreview(value.title);
});
```

Trong trường hợp này:

- `setPreview()` sẽ trigger re-render
- nguyên nhân là React state update
- không phải do `react-hook-form`

---

#### Rule of thumb

| Use case                          | Recommended API   |
| --------------------------------- | ----------------- |
| Render UI reactive                | `watch()`         |
| Optimized reactive UI             | `useWatch()`      |
| Analytics / autosave / sync state | `watch(callback)` |

### Bug 1: useFieldArray key issue ❌

```jsx
// ❌ Code bị lỗi
function BuggyList() {
  const { fields, append } = useFieldArray({ control, name: 'items' });

  return (
    <div>
      {fields.map((field, index) => (
        <input
          key={index} // ← BUG: Using index as key!
          {...register(`items.${index}.name`)}
        />
      ))}
    </div>
  );
}

// ❓ Khi remove item, UI bị mess up. Tại sao?
```

<details>
<summary>💡 Giải thích & Fix</summary>

**Vấn đề:**

- Dùng `index` làm key
- Khi remove item → indexes thay đổi
- React reuses components incorrectly
- UI shows wrong data

**Tại sao xảy ra:**

```
Initial: [item0, item1, item2]
Keys:    [0,     1,     2]

Remove index 1:
New:     [item0, item2]
Keys:    [0,     1]     ← Index 1 now points to different item!

React thinks: Keep item at index 0, update item at index 1
Reality: Should remove middle item, keep last item
```

**Fix:**

```jsx
// ✅ Cách đúng - Use field.id
{
  fields.map((field, index) => (
    <input
      key={field.id} // ← CORRECT: Use stable ID from useFieldArray
      {...register(`items.${index}.name`)}
    />
  ));
}
```

**Nguyên tắc:**

- **NEVER** use array index as key in useFieldArray
- **ALWAYS** use `field.id` (generated by RHF, stable across reorders)
- field.id remains same even when item moves position

</details>

### Bug 2: useFormContext không hoạt động ❌

```jsx
// ❌ Code bị lỗi
function ParentForm() {
  const methods = useForm();

  return (
    <form onSubmit={methods.handleSubmit(onSubmit)}>
      <ChildComponent /> {/* ← BUG: No FormProvider! */}
```

<details>
<summary>💡 Giải thích & Fix</summary>

**Vấn đề:**

- useFormContext cần FormProvider ở parent
- Không wrap → no context available
- Component crashes với "cannot read properties of undefined"

**Fix:**

```jsx
// ✅ Cách đúng
import { FormProvider } from 'react-hook-form';

function ParentForm() {
  const methods = useForm();

  return (
    <FormProvider {...methods}>
      {' '}
      {/* ← Wrap với FormProvider */}
      <form onSubmit={methods.handleSubmit(onSubmit)}>
        <ChildComponent /> {/* Now can access context */}
      </form>
    </FormProvider>
  );
}

function ChildComponent() {
  const { register } = useFormContext(); // ✅ Works!
  return <input {...register('field')} />;
}
```

**Nguyên tắc:**

- FormProvider phải wrap tất cả components cần access form
- Chỉ cần 1 FormProvider ở top-level
- All children/grandchildren có thể dùng useFormContext

**Pattern:**

```jsx

  <form>
    <Section1 /> {/* Can use useFormContext */}
```

### Bug 3: Nested `useFieldArray` gây re-render không cần thiết ❌

```jsx
// ❌ Code dễ gây performance issues
function EducationForm() {
  const { control, register } = useFormContext();

  const { fields } = useFieldArray({
    control,
    name: 'education',
  });

  return fields.map((edu, eduIndex) => (
    <div key={edu.id}>
      <CoursesSection
        control={control}
        register={register}
        eduIndex={eduIndex}
      />
    </div>
  ));
}

function CoursesSection({ control, register, eduIndex }) {
  // ❌ Nested field array
  const { fields: courses } = useFieldArray({
    control,
    name: `education.${eduIndex}.courses`,
  });

  return courses.map((course, i) => (
    <input
      key={course.id}
      {...register(`education.${eduIndex}.courses.${i}.name`)}
    />
  ));
}

// ❓ Thêm/xóa education item
// → tất cả các component CoursesSection phải re-render
// → Các subscriptions lồng nhau phải re-evaluate
// → Có thể  mất focus + lag trong form lớn
```

<details>
<summary>💡 Giải thích & Fix</summary>

**Vấn đề**

Khi parent array (`education`) thay đổi:

- `EducationForm` re-render
- toàn bộ `CoursesSection` re-render theo
- nested `useFieldArray` subscriptions bị re-evaluate
- dễ gây:
  - mất focus
  - lag khi typing
  - render không cần thiết trong large forms

Issue này thường rõ hơn khi:

- nested arrays nhiều cấp
- form lớn
- dynamic append/remove liên tục

---

**✅ Fix: Memoize nested component**

```jsx
import { memo } from 'react';

const CoursesSection = memo(function CoursesSection({
  control,
  register,
  eduIndex,
}) {
  const { fields: courses } = useFieldArray({
    control,
    name: `education.${eduIndex}.courses`,
  });

  return courses.map((course, i) => (
    <input
      key={course.id}
      {...register(`education.${eduIndex}.courses.${i}.name`)}
    />
  ));
});
```

**Tại sao hiệu quả hơn?**

- Parent re-render không kéo toàn bộ nested sections render theo
- Giảm reconciliation không cần thiết
- Giữ input stable hơn
- Hạn chế focus loss

---

**Best practices cho nested `useFieldArray`**

**✅ Dùng `field.id` làm key**

```jsx
key={field.id}
```

Không dùng index:

```jsx
key = { index }; // ❌
```

---

**✅ Tách nested arrays thành component riêng**

Giúp isolate render tốt hơn.

---

**✅ Memoize nested sections**

```jsx
memo(CoursesSection);
```

Đặc biệt quan trọng với large dynamic forms.

---

**✅ Hạn chế `watch()` ở parent component**

Prefer:

```jsx
useWatch();
```

thay vì:

```jsx
watch();
```

để tránh parent re-render lan rộng.

### Tổng kết React Hook Form (đã học + cần biết thêm)

Dưới đây là tổng hợp nhanh các API quan trọng trong React Hook Form, chia theo mức độ đã học và mở rộng thêm để bạn có cái nhìn hệ thống hơn.

---

**Những phần đã học quan trọng**

**1. `useForm`**

Hook cốt lõi để khởi tạo form

```tsx
const { register, handleSubmit, control, watch, formState } = useForm();
```

- Quản lý toàn bộ form state
- Tối ưu re-render mặc định
- Trung tâm của mọi API khác

---

**2. `useFormContext`**

Truy cập form từ component con

```tsx
const { register } = useFormContext();
```

- Tránh prop drilling
- Dùng cho form lớn / nhiều layer
- Kết hợp với `<FormProvider />`

---

**3. `useWatch`**

Theo dõi field thay đổi một cách tối ưu

```tsx
const value = useWatch({
  control,
  name: 'email',
});
```

- Chỉ re-render component đang subscribe
- Tốt hơn `watch()` trong form lớn
- Dùng nhiều trong UI reactive

---

**4. `useFieldArray`**

Quản lý mảng field động

```tsx
const { fields, append, remove } = useFieldArray({
  control,
  name: 'items',
});
```

- Add / remove / reorder fields
- Dùng cho dynamic form (CRUD)
- Quan trọng trong nested form

---

**🚀 Những phần chưa học (nâng cao)**

**1. `useLens`**

- API nâng cao cho type-safe form composition
- Dùng để tạo reusable form field component
- Hữu ích trong design system / form library nội bộ

👉 Khi nào dùng:

- build form system lớn
- muốn abstraction field mạnh + type-safe

<details>
<summary>📌 useLens là gì? (giải thích ngắn gọn)</summary>

`useLens` là một API nâng cao trong React Hook Form dùng để tạo **“lens” (thấu kính)** vào một phần cụ thể của form state, giúp bạn:

- làm việc với **một nhánh nhỏ của form**
- xây dựng **component tái sử dụng (reusable form fields)**
- đảm bảo **type-safe + scoped state**
- giảm phụ thuộc vào path string dài (`a.b.c.d`)

👉 Nói đơn giản:

> `useLens` = cách “nhìn thẳng vào một field/subtree của form” một cách gọn, an toàn và tái sử dụng được

---

**🧠 Vấn đề khi không dùng lens**

Trong form phức tạp:

```tsx
register('user.profile.address.city');
```

❌ Vấn đề:

- path dài, dễ sai
- khó reuse component
- khó refactor
- dễ typo

---

**✅ Ý tưởng của `useLens`**

Thay vì truyền string path:

```tsx
const cityLens = useLens(control, 'user.profile.address.city');
```

Bạn làm việc trực tiếp với “field scoped”:

```tsx
cityLens.value;
cityLens.onChange;
```

---

**📌 Ví dụ cơ bản**

```tsx
const cityLens = useLens({
  control,
  name: 'address.city',
});

return (
  <input
    value={cityLens.value}
    onChange={cityLens.onChange}
  />
);
```

---

**🧩 Use case quan trọng**

**1. Reusable form components**

```tsx
function InputField({ lens }) {
  return (
    <input
      value={lens.value}
      onChange={lens.onChange}
    />
  );
}

// dùng lại
<InputField lens={useLens(control, "email")} />
<InputField lens={useLens(control, "username")} />
```

---

**2. Nested form component sạch hơn**

```tsx
const addressLens = useLens(control, 'address');

return (
  <>
    <input
      value={addressLens.city.value}
      onChange={addressLens.city.onChange}
    />
  </>
);
```

---

**3. Type-safe form architecture**

- giảm string path
- dễ refactor
- phù hợp design system / form library nội bộ

---

**⚠️ Khi nào KHÔNG nên dùng?**

❌ Không cần dùng nếu:

- form đơn giản
- không có reusable field system
- chỉ dùng `useForm` + `register`

👉 Vì:

- tăng abstraction
- không cần thiết cho CRUD đơn giản

---

**🔥 So sánh nhanh**

| API               | Mục đích                             |
| ----------------- | ------------------------------------ |
| `register()`      | input đơn giản                       |
| `useController()` | controlled input                     |
| `useLens()`       | reusable + type-safe field component |

---

**📌 Tóm lại**

`useLens` = cách xây dựng **form component reusable + type-safe**, bằng cách “trỏ trực tiếp” vào một phần của form state thay vì dùng string path.

👉 Nếu:

- `register` = dùng nhanh
- `useController` = controlled input
- `useLens` = architecture reusable form system

</details>

---

**2. `createFormControl`**

- Tạo form control ngoài React component
- Dùng form logic ở service layer

👉 Khi nào dùng:

- form state global
- form orchestration phức tạp
- non-React integration

<details>
<summary>📌 createFormControl là gì? (giải thích ngắn gọn)</summary>

`createFormControl` là API dùng để **tạo form control object bên ngoài React component**, giúp bạn:

- quản lý form state **không phụ thuộc React lifecycle**
- gọi các method của form (getValues, setValue, trigger...) từ service layer
- xây dựng form logic ở mức “vanilla JS” hoặc architecture nâng cao

👉 Nói đơn giản:

> `createFormControl` = tạo “engine form” độc lập, không cần hook React

---

**🧠 Khác gì với `useForm`?**

**❌ `useForm` (gắn với React)**

```tsx
const form = useForm();
```

- chỉ dùng trong React component
- phụ thuộc render lifecycle
- phù hợp UI form thông thường

---

**✅ `createFormControl` (tách khỏi React)**

```tsx
const form = createFormControl();
```

- không cần React component
- có thể dùng trong:
  - service layer
  - state manager
  - external logic (API, workflow engine)

---

**📌 Ví dụ cơ bản**

```tsx
import { createFormControl } from 'react-hook-form';

const form = createFormControl({
  defaultValues: {
    email: '',
    name: '',
  },
});

// set value từ bên ngoài React
form.setValue('email', 'test@gmail.com');

// lấy value
console.log(form.getValues());
```

---

**🧩 Use case thực tế**

**1. Form logic nằm ngoài UI**

```tsx
const form = createFormControl();

export function saveDraft(data) {
  form.setValue('draft', data);
}
```

---

**2. Workflow / wizard engine**

```tsx
const form = createFormControl();

function nextStep() {
  const values = form.getValues();

  if (values.email) {
    goToStep(2);
  }
}
```

---

**3. Sync với external system (Redux/Zustand/API)**

```tsx
const form = createFormControl();

form.watch((value) => {
  store.setState({ form: value });
});
```

---

**⚠️ Khi nào KHÔNG nên dùng?**

❌ Không nên dùng nếu:

- form đơn giản trong UI
- chỉ cần `useForm`
- không có architecture phức tạp

👉 Vì:

- tăng complexity không cần thiết
- khó maintain nếu dùng sai level

---

**🔥 So sánh nhanh**

| API                 | Context         | Use case                   |
| ------------------- | --------------- | -------------------------- |
| `useForm`           | React component | form UI bình thường        |
| `createFormControl` | outside React   | form engine / architecture |
| `useFormContext`    | nested React    | share form trong tree      |

---

**📌 Tóm lại**

`createFormControl` = cách tạo **form engine độc lập React**, dùng khi bạn muốn:

- tách logic form khỏi UI
- build architecture lớn (wizard, workflow, CMS form engine)
- sync form với hệ thống bên ngoài React

👉 Nếu `useForm` là “UI form”, thì `createFormControl` là “form backend engine”.

</details>

---

**3. `useFormState`**

Theo dõi trạng thái form (errors, dirty, touched...)

```tsx
const { errors, isDirty } = useFormState({
  control,
});
```

- Tách re-render khỏi `useForm`
- Giảm render không cần thiết
- Chỉ subscribe state cần thiết

<details>
<summary>📌 useFormState là gì? (giải thích ngắn gọn)</summary>

`useFormState` là hook của React Hook Form dùng để **theo dõi trạng thái của form** (form state) như:

- `errors` (lỗi validate)
- `isDirty` (form đã bị thay đổi chưa)
- `isValid` (form hợp lệ không)
- `isSubmitting` (đang submit)
- `touchedFields`

Điểm quan trọng là:
👉 nó giúp **tách việc subscribe form state ra khỏi `useForm`**, tránh re-render không cần thiết.

---

**❌ Vấn đề khi dùng sai**

```tsx
const { formState } = useForm();
```

Cách này có thể gây:

- re-render toàn component khi formState thay đổi
- performance kém trong form lớn
- khó kiểm soát render boundary

---

**✅ Cách dùng đúng: `useFormState`**

```tsx
import { useForm, useFormState } from 'react-hook-form';

function MyForm() {
  const { control, register, handleSubmit } = useForm();

  const { errors, isDirty, isValid } = useFormState({
    control,
  });

  return (
    <form onSubmit={handleSubmit(console.log)}>
      <input
        {...register('email', {
          required: 'Email là bắt buộc',
        })}
      />

      {errors.email && <p>{errors.email.message}</p>}

      <button disabled={!isDirty || !isValid}>Submit</button>
    </form>
  );
}
```

---

**🧠 Cơ chế hoạt động**

- `useForm` giữ toàn bộ form state
- `useFormState` chỉ subscribe vào phần state bạn cần
- khi state thay đổi → chỉ component dùng `useFormState` re-render

👉 giúp giảm re-render lan rộng trong form lớn

---

**📌 Khi nào nên dùng?**

**✅ Nên dùng khi:**

- Hiển thị error (`errors`)
- Enable/disable button (`isDirty`, `isValid`)
- Loading submit (`isSubmitting`)
- UI phụ thuộc trạng thái form

---

**❌ Không nên dùng khi:**

- Lấy value input → dùng `useWatch`
- Xử lý logic field cụ thể
- Thay thế `watch()`

---

**🔥 So sánh nhanh**

| Hook                        | Dùng cho        | Re-render         |
| --------------------------- | --------------- | ----------------- |
| `useFormState`              | trạng thái form | scoped            |
| `useWatch`                  | value field     | scoped theo field |
| `formState` (trong useForm) | tất cả          | dễ lan rộng       |

---

**📌 Tóm lại**

`useFormState` = cách tối ưu để theo dõi **trạng thái form** (error, dirty, valid...) mà không làm UI re-render không cần thiết.

👉 Nếu `useWatch` là theo dõi **value**, thì `useFormState` là theo dõi **status của form**.

</details>

---

**4. `useController`**

Controlled component bridge

```tsx
const { field } = useController({
  name: 'email',
  control,
});
```

- Dùng cho input custom (MUI, AntD…)
- Giảm manual wiring
- Giữ controlled input sync với RHF

<details>
<summary>📌 useController là gì? (giải thích ngắn gọn)</summary>

`useController` là hook dùng để **kết nối (bridge) giữa React Hook Form và controlled components**.

Nó giúp bạn:

- điều khiển input “controlled” (có `value`, `onChange`)
- vẫn giữ được tối ưu re-render của React Hook Form
- tránh phải tự viết `value + onChange + register` thủ công

👉 Nói đơn giản:

> `useController` = cách “wrap” input custom để RHF quản lý như native input

---

**❌ Khi không dùng `useController`**

Với controlled component, bạn phải tự wiring:

```tsx
function MyInput({ value, onChange }) {
  return (
    <input
      value={value}
      onChange={(e) => onChange(e.target.value)}
    />
  );
}
```

Và trong form:

```tsx
<input
  value={value}
  onChange={(e) => setValue('email', e.target.value)}
/>
```

👉 Vấn đề:

- boilerplate nhiều
- dễ sai sync state
- khó scale với UI library (MUI, AntD, custom components)

---

**✅ Cách dùng `useController`**

```tsx
import { useController, useForm } from 'react-hook-form';

function MyForm() {
  const { control } = useForm();

  const { field } = useController({
    name: 'email',
    control,
    defaultValue: '',
  });

  return (
    <input
      value={field.value}
      onChange={field.onChange}
      onBlur={field.onBlur}
    />
  );
}
```

---

**🧠 Cơ chế hoạt động**

`useController`:

- tạo **controlled field binding**
- tự quản lý:
  - `value`
  - `onChange`
  - `onBlur`

- subscribe riêng cho field đó
- giúp isolate re-render (không kéo cả form)

---

**📌 Khi nào nên dùng?**

**✅ Nên dùng khi:**

- UI library controlled input (MUI, Ant Design, Chakra...)
- custom input component (DatePicker, Select, RichTextEditor)
- component không hỗ trợ `register`

---

**❌ Không nên dùng khi:**

- input HTML đơn giản (`input`, `select`, `textarea`)
  → dùng `register()` là đủ

```tsx
<input {...register('email')} />
```

---

**🔥 So sánh nhanh**

| Cách              | Dùng cho           | Re-render   |
| ----------------- | ------------------ | ----------- |
| `register()`      | native input       | tối ưu nhất |
| `useController()` | controlled input   | scoped      |
| manual state      | custom React state | dễ lag      |

---

**📌 Ví dụ thực tế (UI library)**

```tsx
const { field } = useController({
  name: 'country',
  control,
});

<Select
  value={field.value}
  onChange={field.onChange}
  options={[
    { label: 'VN', value: 'vn' },
    { label: 'US', value: 'us' },
  ]}
/>;
```

---

**📌 Tóm lại**

`useController` = cầu nối giữa React Hook Form và controlled components.

👉 Nếu:

- `register()` = input HTML đơn giản
- `useController()` = input “xịn” / custom / UI library

</details>

---

**📌 Tóm tắt kiến trúc tư duy**

React Hook Form có thể hiểu theo 3 lớp:

**1. Core form engine**

- `useForm`

**2. State subscription layer**

- `useWatch`
- `useFormState`

**3. UI binding layer**

- `useController`
- `useFieldArray`
- `useFormContext`

---

**📚 Tài liệu tham khảo chính thức**

👉 [https://react-hook-form.com/docs](https://react-hook-form.com/docs)

---

**Kết luận nhanh**

- `useForm` → lõi hệ thống
- `useFieldArray` → xử lý form động
- `useWatch` / `useFormState` → tối ưu re-render
- `useController` → custom input
- `useFormContext` → scale form lớn

Nếu nắm vững 5 hook đã học, bạn đã đủ để build hầu hết form production phức tạp.

```jsx
            marginBottom: '24px',
            padding: '20px',
            backgroundColor: '#f9f9f9',
            borderRadius: '8px',
          }}
        >
          <h2>📋 Basic Information</h2>

              style={{
                display: 'block',
                marginBottom: '4px',
                fontWeight: 'bold',

              Recipe Name *

              {...register('name', { required: 'Recipe name is required' })}

                width: '100%',
                padding: '8px',
                border: errors.name ? '2px solid red' : '1px solid #ccc',
                borderRadius: '4px',
              }}
            />
            {errors.name && (
              <span style={{ color: 'red', fontSize: '12px' }}>
                {errors.name.message}
              </span>
            )}
          </div>

                width: '100%',
```

```jsx
 * - Expense item (level 3) → description, amount, date

function useBudgetCalculations(control, monthIndex) {
  // Watch income and expenses for specific month
  const income = useWatch({
    control,
    name: `months.${monthIndex}.incomeSources`,
  });
  const categories = useWatch({
    control,
    name: `months.${monthIndex}.expenseCategories`,
  });

  const totalIncome = useMemo(() => {
    if (!income) return 0;
    return income.reduce((sum, item) => sum + (Number(item.amount) || 0), 0);
  }, [income]);

  const totalExpenses = useMemo(() => {
    if (!categories) return 0;
    return categories.reduce((sum, cat) => {
      const catExpenses = cat.expenses || [];
      return (
        sum +
        catExpenses.reduce(
          (catSum, exp) => catSum + (Number(exp.amount) || 0),
          0,
        )
      );
    }, 0);
  }, [categories]);

  const balance = totalIncome - totalExpenses;
  const isOverBudget = totalExpenses > totalIncome && totalIncome > 0;

  return { totalIncome, totalExpenses, balance, isOverBudget };
}

function useOverallSummary(control) {
  const months = useWatch({ control, name: 'months' });

  return useMemo(() => {
    if (!months) return { totalIncome: 0, totalExpenses: 0, totalBalance: 0 };

    let totalIncome = 0;
    let totalExpenses = 0;

    months.forEach((month) => {
      (month.incomeSources || []).forEach((inc) => {
        totalIncome += Number(inc.amount) || 0;
      });
      (month.expenseCategories || []).forEach((cat) => {
        (cat.expenses || []).forEach((exp) => {
          totalExpenses += Number(exp.amount) || 0;
        });
      });
    });

    return {
      totalIncome,
      totalExpenses,
      totalBalance: totalIncome - totalExpenses,
      monthCount: months.length,
    };
  }, [months]);
}
```

### Cân Nhắc Production

**1. useFieldArray Performance**

```jsx
// ❌ Tránh: Re-initialize nested arrays
{fields.map((field, index) => (
  <NestedComponent index={index} /> // Re-creates useFieldArray each time
))}

// ✅ Tốt hơn: Memoize hoặc inline
const NestedComponent = memo(({ index }) => {
  const { fields } = useFieldArray({ name: `parent.${index}.children` });
  // ...
});

// ✅ Hoặc inline (simpler)
{fields.map((field, index) => {
  const NestedFields = () => {
    const { fields: children } = useFieldArray({ name: `parent.${index}.children` });
    return children.map(...);
  };
  return <NestedFields key={field.id} />;
})}
```

**2. useFormContext Best Practices**

```jsx
// ❌ Tránh: FormProvider everywhere
<FormProvider {...methods1}>

// ✅ Tốt hơn: One form per FormProvider
<FormProvider {...methods}>

// If need multiple forms, separate components:

  <Form1 />
</FormProvider>
<FormProvider {...form2}>
```

### Câu Hỏi Phỏng Vấn

**Mid Level:**

1. **Q:** useFieldArray vs manual array state management?
   **A:** useFieldArray provides: stable IDs (field.id), automatic re-indexing, built-in validation support, better performance. Manual state requires handling all this yourself.

2. **Q:** Khi nào dùng useFormContext?
   **A:** Dùng khi form có nhiều nested components và muốn tránh prop drilling. NOT for passing non-form data (use normal Context/props).

**Senior Level:**

3. **Q:** Design form architecture cho ứng dụng với 50+ different forms?
   **A:**
   - Shared field component library
   - Form config driven (JSON schema)
   - Reusable validation rules
   - Custom hooks for common patterns
   - FormBuilder for non-developers
   - Centralized error handling
   - Analytics integration

4. **Q:** Optimize performance cho form với 100+ fields và nested arrays?
   **A:**
   - useWatch thay vì watch()
   - React.memo cho field components
   - Debounce validation
   - Code splitting cho large sections
   - Virtual scrolling nếu cần
   - Lazy load field components

5. **Q:** Handle concurrent updates trong collaborative forms (multiple users)?
   **A:**
   - Optimistic updates
   - Conflict resolution strategy
   - Last-write-wins vs merge
   - Field-level locking
   - Real-time sync with debouncing
   - Show who's editing what

### War Stories

**Story 1: The Field Array Index Disaster**

```
Situation: E-commerce cart với 50 items, dùng index làm key
Problem: Remove item → UI shows wrong products!
Root cause: Index as key → React reuses wrong components
Solution: Switch to field.id → Problem solved
Lesson: ALWAYS use field.id, NEVER index
Time wasted: 4 hours debugging
```

**Story 2: The FormProvider Confusion**

```
Situation: Large form với 10 sections, mỗi section là separate component
Problem: Passing register, errors, etc. qua 10 components
Solution: useFormContext → no more prop drilling
Benefit: 500 lines of props removed, much cleaner
Lesson: FormProvider = game changer for complex forms
```

**Story 3: The Nested Array Performance Hell**

```
Situation: Education form với courses (nested array)
Problem: Adding 1 course → ALL education entries re-render
Root cause: Nested useFieldArray không memoized
Solution: React.memo on nested component
Result: Performance improvement 10x
Lesson: Always memoize components với nested useFieldArray
```
